1. pyproject.toml

pyproject.toml is the modern single-file standard for Python project configuration (PEP 517/518). It replaces the old setup.py + setup.cfg + MANIFEST.in trio. Think of it as pom.xml for Python.

Java — pom.xml (excerpt) 
<project>
  <groupId>com.example</groupId>
  <artifactId>taskmanager</artifactId>
  <version>1.0.0</version>

  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter</artifactId>
    </dependency>
  </dependencies>
</project>
Python — pyproject.toml 
[build-system]
requires = ["setuptools>=68", "wheel"]
build-backend = "setuptools.backends.legacy:build"

[project]
name = "taskmanager"
version = "1.0.0"
description = "A task management library and CLI"
readme = "README.md"
requires-python = ">=3.10"
license = {text = "MIT"}
authors = [{name = "Vipin Sharma", email = "v@example.com"}]

dependencies = [
    "fastapi>=0.115",
    "sqlalchemy>=2.0",
    "pydantic-settings>=2.0",
    "click>=8.0",
]

[project.optional-dependencies]
dev = ["pytest", "httpx", "mypy", "ruff"]

[project.scripts]
taskmanager = "taskmanager.cli:main"  # entry point

Package Layout

file layout 
taskmanager/               ← project root
  pyproject.toml
  README.md
  src/
    taskmanager/           ← the actual package (src layout is recommended)
      __init__.py
      models.py
      storage.py
      cli.py
  tests/
    test_models.py
src layout vs flat layout: Placing your package under src/ prevents accidentally importing from the project root instead of the installed package — a common source of "it works locally but fails in CI" bugs. Most new projects use the src layout.

2. Editable Installs

An editable install (pip install -e .) installs your package in development mode — changes to your source files take effect immediately without reinstalling. This is the Python equivalent of mvn install to a local repo, but better — there's no reinstall step at all.

$ python -m venv .venv && source .venv/bin/activate
$ # Install the package + dev dependencies
$ pip install -e ".[dev]"

$ # Now you can import your package from anywhere
$ python -c "from taskmanager.models import Task; print(Task)"

$ # And run tests
$ pytest tests/

$ # Edit src/taskmanager/models.py — changes take effect immediately

3. Click — CLI Framework

Click is Python's standard for building command-line interfaces. It uses decorators to declare arguments, options, and commands — far cleaner than argparse, and the Python equivalent of picocli.

$ pip install click

Basic Commands and Options

Python — cli.py 
import click
from taskmanager.storage import JsonStorage
from taskmanager.models import Task

storage = JsonStorage("tasks.json")

@click.group()
@click.version_option(version="1.0.0")
def main():
    """Task Manager — a CLI for managing your tasks."""
    pass

@main.command()
@click.argument("title")
@click.option("--priority", "-p",
              type=click.Choice(["low", "medium", "high"]),
              default="medium",
              show_default=True,
              help="Task priority level.")
@click.option("--due", "-d", type=str, default=None, help="Due date (YYYY-MM-DD).")
def add(title: str, priority: str, due: str | None):
    """Add a new task."""
    task = Task(title=title, priority=priority, due_date=due)
    storage.add(task)
    click.echo(click.style(f"✓ Added: {title}", fg="green"))

@main.command("list")
@click.option("--status", type=click.Choice(["all", "pending", "done"]), default="all")
def list_tasks(status: str):
    """List all tasks."""
    tasks = storage.all()
    if status != "all":
        tasks = [t for t in tasks if t.status == status]
    if not tasks:
        click.echo("No tasks found.")
        return
    for t in tasks:
        icon = "✓" if t.status == "done" else "○"
        color = "green" if t.status == "done" else "yellow"
        click.echo(click.style(f"{icon} [{t.id}] {t.title}", fg=color))

@main.command()
@click.argument("task_id", type=int)
def done(task_id: int):
    """Mark a task as done."""
    task = storage.get(task_id)
    if not task:
        click.echo(click.style(f"Task {task_id} not found.", fg="red"), err=True)
        raise SystemExit(1)
    task.status = "done"
    storage.update(task)
    click.echo(click.style(f"✓ Completed: {task.title}", fg="green"))

@main.command()
@click.argument("task_id", type=int)
@click.confirmation_option(prompt="Are you sure you want to delete this task?")
def delete(task_id: int):
    """Delete a task permanently."""
    storage.delete(task_id)
    click.echo(f"Deleted task {task_id}.")

Click Utilities

Python 
import click

# Styled output
click.echo(click.style("Error!", fg="red", bold=True))
click.echo(click.style("Success!", fg="green"))
click.echo(click.style("Warning", fg="yellow"))

# Write to stderr
click.echo("Error details", err=True)

# Prompt for input
name = click.prompt("Your name", default="World")
confirmed = click.confirm("Are you sure?", default=False)

# Progress bar
import time
with click.progressbar(range(100), label="Processing") as bar:
    for item in bar:
        time.sleep(0.01)

# Pager — for long output
help_text = "\n".join(f"Line {i}" for i in range(200))
click.echo_via_pager(help_text)

# File argument (handles opening/closing automatically)
@click.command()
@click.argument("input", type=click.File("r"))
@click.argument("output", type=click.File("w"))
def transform(input, output):
    for line in input:
        output.write(line.upper())

Sub-command Groups

Python — nested commands

def main():
    """Task Manager CLI."""

# Nested group: taskmanager config set KEY VALUE
@main.group()
def config():
    """Manage configuration."""

@config.command("set")
@click.argument("key")
@click.argument("value")

    """Set a configuration value."""
    click.echo(f"Set {key}={value}")

@config.command("get")
@click.argument("key")
def config_get(key: str):
    """Get a configuration value."""
    click.echo(f"{key} = ...")

4. Entry Points

Entry points tell pip to create a system command that calls your function. After pip install, users can just type taskmanager in their terminal.

pyproject.toml 
[project.scripts]
# command-name = "package.module:function"
taskmanager = "taskmanager.cli:main"
tm = "taskmanager.cli:main"   # short alias
$ pip install -e .
$ taskmanager --help
$ taskmanager add "Buy milk" --priority high
$ taskmanager list --status pending
$ taskmanager done 1
$ tm list # short alias works too

5. Building and Publishing

$ pip install build twine

$ # Build the distribution packages
$ python -m build
# Creates dist/taskmanager-1.0.0.tar.gz (sdist)
# Creates dist/taskmanager-1.0.0-py3-none-any.whl (wheel)

$ # Upload to TestPyPI first (safe for testing)
$ twine upload --repository testpypi dist/*

$ # Install from TestPyPI to verify
$ pip install --index-url https://test.pypi.org/simple/ taskmanager

$ # Upload to real PyPI
$ twine upload dist/*
API tokens: PyPI uses token-based authentication. Create a token at pypi.org/manage/account/token/ and store it in ~/.pypirc or as a CI/CD secret. Never use your password directly.

Wrap-up

  • pyproject.toml is the single source of truth for a Python project — metadata, dependencies, build config, tool config (mypy, ruff, pytest). No more scattered config files.
  • pip install -e . is the right way to develop a package. You get real import paths and real entry points without any reinstall-on-change cycle.
  • Click's decorator approach makes CLIs readable and testable. The @click.group() + @main.command() pattern maps naturally to nested sub-commands like git commit, docker container run.
  • Entry points in [project.scripts] are how pip install creates terminal commands. The pattern "package.module:function" is just a dotted import path and a function name.
  • Always publish to TestPyPI before the real PyPI. There's no delete on PyPI — once a version is published, it's permanent.

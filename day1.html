1. The Python Toolchain

As a Java developer, you're used to the Maven/Gradle + JDK ecosystem. Python has its own equivalent trio:

Java Python Purpose
JDK python / python3 The runtime + standard library
Maven Central PyPI Central package repository
mvn / gradle pip Package installer
pom.xml / build.gradle venv + requirements.txt Project-scoped deps

Installing Python

On macOS use Homebrew (recommended). On Linux use your package manager. On Windows use the official installer.

Shell 
# macOS
brew install python

# Verify
python3 --version   # Python 3.12.x
pip3 --version

Virtual Environments (venv)

In Maven, each project declares its deps in pom.xml and Maven resolves them into a local repo (~/.m2) shared across projects. Python's pip installs to a global location by default, which causes version conflicts. venv creates a self-contained Python environment per project.

Rule of thumb: Always create a venv before starting any Python project. It takes 3 seconds and prevents dependency hell.
Shell 
# Create venv (creates a .venv/ folder in your project)
python3 -m venv .venv

# Activate it (must do this every new terminal session)
source .venv/bin/activate          # macOS/Linux
.venv\Scripts\activate             # Windows

# You'll see (.venv) prefix in your prompt
# Now install packages — they go into .venv/, not global
pip install requests

# Freeze deps (like Maven's pom.xml dependency section)
pip freeze > requirements.txt

# Deactivate when done
deactivate

Running Python code

Python is interpreted. No separate compile step. The interpreter reads your .py file and executes it directly.

Java 
// Two steps:
// 1. Compile
javac Hello.java   // → Hello.class

// 2. Run
java Hello
Python 
# One step:
python3 hello.py   # That's it

# Or interactively:
python3            # Opens REPL
>>> print("hello")

The Python REPL (Read-Eval-Print Loop) is like a live scratchpad. Use it constantly as a Java developer — it's faster than writing a main method to test a snippet.

2. Variables and Dynamic Typing

Python uses dynamic typing. The type is attached to the value, not the variable. The variable is just a name pointing to an object.

Java — static typing 
String name = "Alice";
int age = 30;
double pi = 3.14;
boolean active = true;

// Type mismatch → compile error
name = 42;  // ERROR
Python — dynamic typing 
name = "Alice"    # str
age = 30          # int
pi = 3.14         # float
active = True     # bool (capital T/F!)

# Variable can be rebound to any type
name = 42         # now it's an int — valid
Important: Dynamic typing doesn't mean no types exist. Every value has a type — type(42) returns <class 'int'>. The difference is that the type is checked at runtime, not compile time. This is why Python has type hints (Day 13) — they restore tooling-level safety without changing the runtime behavior.

Naming conventions

Java conventions 
// camelCase for variables/methods
String userName = "alice";
int maxRetryCount = 3;

// PascalCase for classes
class UserAccount { }

// SCREAMING_SNAKE for constants
static final int MAX_SIZE = 100;
Python conventions (PEP 8) 
# snake_case for variables/functions
user_name = "alice"
max_retry_count = 3

# PascalCase for classes (same as Java)
class UserAccount: pass

# SCREAMING_SNAKE for constants (same)
MAX_SIZE = 100

# Python has no const keyword —
# it's just a convention

Multiple assignment

Python 
# Assign multiple variables at once
x, y, z = 1, 2, 3

# Swap without temp variable (tuple unpacking)
x, y = y, x

# Assign same value to multiple names
a = b = c = 0

# Check a variable's type at runtime
print(type(name))    # <class 'str'>
print(type(age))     # <class 'int'>

3. Built-in Types

int — arbitrary precision integers

Java 
int x = 42;
long big = 9_999_999_999L;
int a = 10 / 3;   // = 3 (integer division)
double b = 10.0 / 3;  // = 3.333...
Python 
x = 42
big = 9_999_999_999   # underscores ok, no suffix
a = 10 // 3    # = 3  (floor division, use //)
b = 10 / 3     # = 3.333...  (/ always returns float)
r = 10 % 3     # = 1  (remainder)
p = 2 ** 10    # = 1024  (exponentiation, not ^)
Gotcha: In Python, / always returns a float. To get integer division use //. In Java, 10 / 3 already gives 3.

float

Python 
pi = 3.14159
sci = 1.5e10       # 15000000000.0  (scientific notation)

# Same float precision gotcha as Java
print(0.1 + 0.2)   # 0.30000000000000004

# For money/precision: use decimal module
from decimal import Decimal
price = Decimal("9.99")   # exact decimal representation

str — immutable strings

Java 
String s1 = "hello";
String s2 = 'h';      // char, not String
// Multi-line (Java 15+):
String text = """
    line one
    line two
    """;
// String is an object
s1.length()           // 5
s1.toUpperCase()      // "HELLO"
Python 
s1 = "hello"
s2 = 'hello'    # single or double, identical

# No separate char type — it's just len-1 str

# Multi-line (built-in):
text = """
line one
line two
"""

# Strings are objects with methods
len(s1)              # 5  (function, not method)
s1.upper()           # "HELLO"
s1[0]                # 'h'  (indexing)
s1[-1]               # 'o'  (last char — no ArrayIndexOutOfBounds)

bool

Java 
boolean active = true;
boolean closed = false;

// Null check
if (obj != null) { }
Python 
active = True     # capital T
closed = False    # capital F

# None is Python's null
obj = None
if obj is not None:   # prefer 'is not' for None checks
    pass

# Truthiness — Python is flexible:
bool(0)       # False
bool("")      # False
bool([])      # False  (empty list)
bool(None)    # False
bool(1)       # True
bool("hi")    # True
Key insight: In Python, many things are "falsy" beyond just False. Zero, empty strings, empty collections, and None all evaluate to False in a boolean context. This is used heavily in idiomatic Python: if items: instead of if items.size() > 0.

4. f-strings

f-strings (formatted string literals) are Python's way to embed expressions inside strings. Prefix a string with f and wrap any expression in {}.

Java — multiple approaches 
String name = "Alice";
int age = 30;

// Old: verbose
"Hello, " + name + "!"

// String.format
String.format("Hello, %s! Age: %d", name, age)

// Java 21 text block interpolation (preview)
STR."Hello \{name}!"

// MessageFormat
MessageFormat.format(
    "Hello, {0}!", name)
Python — f-strings (Python 3.6+) 
name = "Alice"
age = 30

# Simple variable
f"Hello, {name}!"

# Expression — anything goes inside {}
f"Next year I'll be {age + 1}"

# Method calls
f"Upper: {name.upper()}"

# Formatting numbers
pi = 3.14159
f"Pi is {pi:.2f}"          # "Pi is 3.14"
f"Hex: {255:#x}"           # "Hex: 0xff"
f"Thousands: {1000000:,}"  # "Thousands: 1,000,000"

# Multi-line f-string
msg = (
    f"Name: {name}\n"
    f"Age:  {age}"
)
f-string debugging trick (Python 3.8+): Use f"{variable=}" to print both the name and value — great for quick debugging:
Python 
x = 42
name = "Alice"
pi = 3.14159

print(f"{x=}")        # x=42
print(f"{name=}")     # name='Alice'
print(f"{pi=:.2f}")   # pi=3.14

5. print() and input()

Java 
System.out.println("Hello");
System.out.print("No newline");
System.out.printf("Pi: %.2f%n", 3.14);

Scanner sc = new Scanner(System.in);
String input = sc.nextLine();
Python 
print("Hello")                  # adds newline
print("No newline", end="")     # override line ending
print("Pi:", 3.14)              # auto-inserts space
print("a", "b", "c", sep=", ") # "a, b, c"

# Reading user input (always returns str)
name = input("Enter your name: ")
age = int(input("Enter age: "))  # cast to int manually
Note: input() always returns a string. If you want a number, you must cast it explicitly with int() or float(). No automatic conversion like some languages do.

Putting it all together — a first real program

Python — hello.py 
# hello.py — run with: python3 hello.py

def main():
    name = input("What's your name? ")
    age = int(input("How old are you? "))

    greeting = f"Hello, {name}!"
    years_to_100 = 100 - age

    print(greeting)
    print(f"You'll turn 100 in {years_to_100} years.")
    print(f"At that point, it'll be {2024 + years_to_100}.")

main()

Wrap-up

  • Python is interpreted — python3 file.py runs directly, no compile step.
  • Always use venv for project-scoped dependencies — treat it like Maven's project-scoped deps.
  • Dynamic typing means the variable is just a label; the type lives on the value. Python doesn't enforce types at compile time — but type hints (Day 13) bring back tooling safety.
  • / always returns a float; use // for integer division. ** is power, not ^.
  • f-strings are the idiomatic way to format strings — faster and clearer than String.format().

FastAPI Basics

FastAPI is Python's answer to Spring Boot — type-safe routes, automatic validation, and interactive docs at /docs out of the box. One decorator replaces @RestController, @RequestMapping, and @ResponseBody.

Plan for the Day

What you'll cover

  • Why FastAPI over Flask — speed, type safety, auto docs
  • Creating a FastAPI app and running with uvicorn
  • Path operations: @app.get, @app.post, @app.put, @app.delete
  • Path parameters and query parameters
  • Request bodies with Pydantic models
  • Response models and status codes
  • Automatic OpenAPI docs at /docs

Java parallels

  • FastAPI() → @SpringBootApplication
  • @app.get("/path") → @GetMapping("/path")
  • @app.post("/path") → @PostMapping("/path")
  • Path param {item_id} → @PathVariable
  • Query param ?q=value → @RequestParam
  • Pydantic model in body → @RequestBody + validation annotations
  • uvicorn → embedded Tomcat / Netty
  • Auto /docs (Swagger UI) → springdoc-openapi

Path Parameters and Query Parameters

Python 
from fastapi import FastAPI, Query, Path
from typing import Optional

app = FastAPI()

# Path parameters — declared as function parameters matching {name}
@app.get("/users/{user_id}/posts/{post_id}")
def get_post(
    user_id: int,    # FastAPI validates and converts automatically
    post_id: int,
):
    return {"user_id": user_id, "post_id": post_id}

# Query parameters — any param NOT in path becomes a query param
# GET /items?skip=0&limit=10&q=widget
@app.get("/items")
def list_items(
    skip:  int = 0,          # default value makes it optional
    limit: int = 10,
    q:     str | None = None,  # optional string query
):
    items = ["Widget A", "Widget B", "Gadget X"]
    filtered = [i for i in items if q is None or q.lower() in i.lower()]
    return filtered[skip : skip + limit]

# Validation with Query() and Path()
@app.get("/items/validated")
def list_validated(
    limit: int = Query(default=10, ge=1, le=100,
                       description="Items to return (1-100)"),
    q:     str = Query(default=None, min_length=2,
                       description="Search term (min 2 chars)"),
):
    return {"limit": limit, "q": q}

@app.get("/items/{item_id}/validated")
def get_validated(
    item_id: int = Path(ge=1, description="Positive item ID"),
):
    return {"item_id": item_id}

Request Bodies with Pydantic

FastAPI uses Pydantic models for request and response bodies. Define a class, and FastAPI validates the incoming JSON against it automatically — field types, required fields, and custom validators all work out of the box.

Java 
public class CreateItemRequest {
    @NotNull @Size(min=1)
    private String name;
    @Positive
    private double price;
    // getters/setters or use record...
}

@PostMapping("/items")
@ResponseStatus(HttpStatus.CREATED)
public Item createItem(@Valid @RequestBody CreateItemRequest req) {
    return itemService.create(req);
}
Python 
from fastapi import FastAPI
from pydantic import BaseModel, Field
from typing import Optional

app = FastAPI()

class CreateItemRequest(BaseModel):
    name:        str     = Field(min_length=1, description="Item name")
    price:       float   = Field(gt=0, description="Price in USD")
    description: str | None = None
    in_stock:    bool    = True

class ItemResponse(BaseModel):
    id:          int
    name:        str
    price:       float
    description: str | None
    in_stock:    bool

# Fake DB
_db: dict[int, dict] = {}
_next_id = 1

@app.post("/items", response_model=ItemResponse, status_code=201)
def create_item(body: CreateItemRequest):
    global _next_id
    item = {"id": _next_id, **body.model_dump()}
    _db[_next_id] = item
    _next_id += 1
    return item

@app.get("/items/{item_id}", response_model=ItemResponse)
def get_item(item_id: int):
    if item_id not in _db:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")
    return _db[item_id]
response_model: When you specify response_model=ItemResponse, FastAPI filters the response to only include fields defined in the model — even if your function returns extra fields. This is the Python equivalent of Jackson's @JsonView.

A Complete CRUD API

Python — main.py 
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field
from datetime import datetime, timezone

app = FastAPI(title="Task API", version="1.0.0")


# ── Schemas ──────────────────────────────────────────────────────────────────

class TaskCreate(BaseModel):
    title:       str     = Field(min_length=1, max_length=200)
    description: str     = ""
    priority:    int     = Field(default=3, ge=1, le=5)

class TaskUpdate(BaseModel):
    title:       str | None = Field(default=None, min_length=1)
    description: str | None = None
    done:        bool | None = None

class TaskResponse(BaseModel):
    id:          int
    title:       str
    description: str
    priority:    int
    done:        bool
    created_at:  datetime


# ── In-memory storage ─────────────────────────────────────────────────────────

_tasks: dict[int, dict] = {}
_counter = 0


# ── Routes ────────────────────────────────────────────────────────────────────

@app.get("/tasks", response_model=list[TaskResponse])
def list_tasks(done: bool | None = None):
    """List all tasks, optionally filtered by done status."""
    tasks = list(_tasks.values())
    if done is not None:
        tasks = [t for t in tasks if t["done"] == done]
    return tasks


@app.post("/tasks", response_model=TaskResponse, status_code=status.HTTP_201_CREATED)
def create_task(body: TaskCreate):
    """Create a new task."""
    global _counter
    _counter += 1
    task = {
        "id": _counter,
        **body.model_dump(),
        "done": False,
        "created_at": datetime.now(timezone.utc),
    }
    _tasks[_counter] = task
    return task


@app.get("/tasks/{task_id}", response_model=TaskResponse)
def get_task(task_id: int):
    """Get a single task by ID."""
    if task_id not in _tasks:
        raise HTTPException(status_code=404, detail=f"Task {task_id} not found")
    return _tasks[task_id]


@app.patch("/tasks/{task_id}", response_model=TaskResponse)
def update_task(task_id: int, body: TaskUpdate):
    """Partially update a task."""
    if task_id not in _tasks:
        raise HTTPException(status_code=404, detail=f"Task {task_id} not found")
    updates = body.model_dump(exclude_none=True)   # only provided fields
    _tasks[task_id].update(updates)
    return _tasks[task_id]


@app.delete("/tasks/{task_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_task(task_id: int):
    """Delete a task."""
    if task_id not in _tasks:
        raise HTTPException(status_code=404, detail=f"Task {task_id} not found")
    del _tasks[task_id]
Try it: Run uvicorn main:app --reload then open http://localhost:8000/docs for interactive Swagger UI — create tasks, list them, update, delete — all in the browser, no Postman needed.

Quick Reference Card

Create app
app = FastAPI(title="My API")

Run with: uvicorn main:app --reload

Route decorator
@app.get("/path") / @app.post / @app.put / @app.delete

Method + path in one decorator. Function return value becomes the response.

Path parameter
@app.get("/items/{id}") / def fn(id: int): ...

Type annotation triggers automatic conversion and validation.

Query parameter
def fn(skip: int = 0, q: str | None = None): ...

Any parameter not in path becomes a query param. Defaults make it optional.

Request body
def fn(body: MyModel): ...

Pydantic model parameter → parsed + validated from request JSON body.

HTTP error
raise HTTPException(status_code=404, detail="Not found")

FastAPI converts this to a JSON error response automatically.

Response model
@app.get("/", response_model=MySchema)

Filters and validates the response. Extra fields are excluded.

Partial update
body.model_dump(exclude_none=True)

Only update fields that were actually provided. Essential for PATCH endpoints.

Wrap-up

  • FastAPI is a single decorator per endpoint — @app.get("/path") replaces @RestController + @RequestMapping + @GetMapping. Type hints in the function signature drive parameter parsing, validation, and documentation automatically.
  • Path parameters ({id} in the path) are declared as function arguments. Query parameters are any argument not in the path. Both are automatically typed, validated, and documented from the type annotation.
  • Request body validation is handled by Pydantic models. FastAPI validates the incoming JSON, returns a 422 with detailed error messages on failure, and gives you a typed Python object in the function — no manual parsing.
  • response_model is both a filter and a validator. It strips extra fields from your response (security — never accidentally return passwords or internal IDs) and validates the output shape.
  • Visit /docs for Swagger UI and /redoc for ReDoc — both generated automatically from your type hints. No separate spec file, no annotation processing step.

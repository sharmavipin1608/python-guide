LangChain: Agents + Tools

Chains follow a fixed path. Agents are different — the LLM decides at runtime which tools to call and in what order. Today you'll build agents that use built-in and custom tools, with both OpenAI and Ollama backends.

  • Agent memory → conversation state, like a session-scoped Spring bean
  • Tool schema → OpenAPI spec that tells the LLM how to call your function

1. What Are Agents?

A chain is like a hardcoded workflow — step A always goes to step B. An agent is different: given a goal and a set of tools, the LLM reasons about which tools to call and in what order to reach the answer. The LLM is the control flow.

Chain: input → prompt → LLM → output parser → result   Agent: input → LLM decides → tool call → LLM observes result ↑ ↓ └──── (repeat until done) ────┘ → final answer
ReAct pattern (Reason + Act): The LLM alternates between thinking about what to do next and acting by calling a tool. It sees the tool output and reasons again. This continues until the LLM decides it has enough information to give a final answer.

2. Creating Custom Tools with @tool

A tool is a Python function decorated with @tool. The docstring becomes the tool's description — the LLM uses it to decide when to call the tool. The type hints define the input schema.

Python 
from langchain_core.tools import tool
from datetime import datetime
import httpx

@tool
def get_current_time() -> str:
    """Returns the current date and time. Use when the user asks about time or dates."""
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

@tool
def calculate(expression: str) -> str:
    """
    Evaluates a mathematical expression and returns the result.
    Input must be a valid Python math expression like '2 + 2' or '100 * 0.15'.
    """
    try:
        # Safe eval for math expressions only
        result = eval(expression, {"__builtins__": {}}, {})
        return str(result)
    except Exception as e:
        return f"Error: {e}"

@tool
def get_python_docs(query: str) -> str:
    """
    Searches the Python documentation for information about a module or function.
    Use when the user asks about Python stdlib functions or modules.
    """
    # Simplified — real implementation would call docs.python.org API
    response = httpx.get(
        f"https://docs.python.org/3/search.html",
        params={"q": query, "check_keywords": "yes", "area": "default"},
    )
    return f"Search results for '{query}': {response.url}"

# Inspect the tool schema the LLM will see
print(calculate.name)         # "calculate"
print(calculate.description)  # the docstring
print(calculate.args_schema.schema())  # JSON schema of inputs

3. Building a ReAct Agent

Python 
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain import hub
from langchain_core.tools import tool

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# Define the tools
tools = [get_current_time, calculate, get_python_docs]

# Pull the standard ReAct prompt from LangChain hub
# It instructs the LLM on the Thought/Action/Observation format
prompt = hub.pull("hwchase17/react")

# Create the agent (the LLM + tools + prompt wired together)
agent = create_react_agent(llm, tools, prompt)

# AgentExecutor handles the ReAct loop
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,      # prints each Thought/Action/Observation step
    max_iterations=5,  # prevent infinite loops
    handle_parsing_errors=True,
)

# Run it
result = executor.invoke({
    "input": "What is 15% of 847, and what time is it now?"
})
print(result["output"])

What verbose=True Shows

Observation: 127.05
Thought: Now I need the current time.
Action: get_current_time
Action Input: (no input needed)
Observation: 2026-04-25 14:32:10
Thought: I have both answers now.
Final Answer: 15% of 847 is 127.05. The current time is 2:32 PM.

4. Tool Binding — The Modern Approach

For OpenAI models, bind_tools is the cleaner, more reliable approach. It uses OpenAI's native function calling instead of text parsing.

Python 
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
tools = [calculate, get_current_time]

# Bind tools to the LLM
llm_with_tools = llm.bind_tools(tools)

# Tool lookup map for dispatching calls
tool_map = {t.name: t for t in tools}

def run_agent(user_input: str) -> str:
    messages = [HumanMessage(user_input)]

    while True:
        response = llm_with_tools.invoke(messages)
        messages.append(response)

        # If no tool calls, we have the final answer
        if not response.tool_calls:
            return response.content

        # Execute each tool call and append results
        for tool_call in response.tool_calls:
            tool_fn = tool_map[tool_call["name"]]
            result = tool_fn.invoke(tool_call["args"])
            messages.append(ToolMessage(
                content=str(result),
                tool_call_id=tool_call["id"],
            ))

result = run_agent("What is 2^10 and what time is it?")
print(result)

5. Agent Memory

By default, each agent invocation is stateless — it doesn't remember previous questions. Add memory to maintain conversation context across multiple turns.

Python 
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
llm_with_tools = llm.bind_tools([calculate, get_current_time])

# Session store — keyed by session_id
store: dict[str, InMemoryChatMessageHistory] = {}

def get_session_history(session_id: str) -> InMemoryChatMessageHistory:
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant with access to tools."),
    MessagesPlaceholder(variable_name="history"),   # inject chat history
    ("human", "{input}"),
])

chain = prompt | llm_with_tools

# Wrap with message history
chain_with_history = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history",
)

config = {"configurable": {"session_id": "user-123"}}

# Turn 1
r1 = chain_with_history.invoke({"input": "My name is Vipin."}, config=config)
print(r1.content)

# Turn 2 — it remembers the name from turn 1
r2 = chain_with_history.invoke({"input": "What's my name?"}, config=config)
print(r2.content)   # "Your name is Vipin."

6. Agents with Ollama (Local)

Not all local models support tool calling, but modern ones (llama3.2, mistral-nemo, qwen2.5) do. Use ChatOllama as a drop-in replacement.

Python 
from langchain_ollama import ChatOllama
from langchain_core.tools import tool

# Requires Ollama running locally: ollama pull llama3.2
llm = ChatOllama(model="llama3.2", temperature=0)

@tool
def word_count(text: str) -> int:
    """Counts the number of words in a text string."""
    return len(text.split())

tools = [word_count, calculate]
llm_with_tools = llm.bind_tools(tools)

# Same agent logic as above — the only change is the LLM
result = run_agent("How many words are in 'the quick brown fox'?")
print(result)

# Zero cost, zero latency to OpenAI, full privacy
Model capability varies: Smaller local models (7B parameters) are less reliable at tool selection than GPT-4o. Test with your specific use case. llama3.2:3b handles simple single-tool tasks well; complex multi-step reasoning benefits from larger models.

Wrap-up

  • The docstring of a @tool function is its description to the LLM. Write it clearly — poor descriptions lead to wrong tool selection. Treat it like an API contract.
  • The ReAct loop: the LLM reasons, picks a tool, sees the result, reasons again. Your job is to provide good tools and let the LLM drive the control flow.
  • bind_tools + a manual while loop is more reliable than AgentExecutor for production code — you control the loop, error handling, and retries explicitly.
  • Memory requires you to store and inject chat_history on each invocation. RunnableWithMessageHistory handles this automatically when keyed by session ID.
  • Local Ollama agents work for simple tasks at zero cost. For complex multi-step reasoning, larger models are significantly more reliable.

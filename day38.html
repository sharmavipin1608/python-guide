ChromaDB + Vector Stores

Embeddings turn text into vectors. Vector databases find similar vectors fast. Together they power RAG — the pattern that lets an LLM answer questions about your private documents without fine-tuning.

Java Parallels

  • Embedding → a feature vector, like a hash but semantically meaningful
  • ChromaDB collection → a table where similarity (not equality) is the primary query
  • query_texts() → a "SELECT ... ORDER BY semantic_distance" that doesn't exist in SQL
  • RAG → like a search engine (Elasticsearch) feeding context into a prompt instead of a UI
  • Document chunking → like paginating a large ResultSet before passing it to the next layer

1. What Are Embeddings?

An embedding is a list of numbers (a vector) that represents a piece of text. The key property: texts with similar meaning have similar vectors — their distance in vector space is small. This is fundamentally different from text search, which matches keywords.

Python — generating embeddings 
from openai import OpenAI

client = OpenAI()

def embed(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text,
    )
    return response.data[0].embedding

v1 = embed("Python dictionary")
v2 = embed("Java HashMap")
v3 = embed("chocolate cake recipe")

# v1 and v2 are close (both about key-value data structures)
# v3 is far from both

print(len(v1))       # 1536 — the vector has 1536 dimensions
print(v1[:5])        # [-0.023, 0.041, -0.013, ...]
Why not just use string similarity? "Python dictionary" and "Java HashMap" share zero words, but they're semantically identical. Embeddings capture meaning, not characters. That's the fundamental advantage.

2. ChromaDB Setup

Python — ChromaDB basics 
import chromadb
from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction

# In-memory client (for development and testing)
client = chromadb.Client()

# Persistent client — saves to disk
client = chromadb.PersistentClient(path="./chroma_db")

# Embedding function — called automatically when you add/query
embedding_fn = OpenAIEmbeddingFunction(
    api_key="your-api-key",
    model_name="text-embedding-3-small",
)

# Create a collection (like a table for vectors)
collection = client.get_or_create_collection(
    name="python_docs",
    embedding_function=embedding_fn,
    metadata={"hnsw:space": "cosine"},   # cosine similarity (default)
)

# Find the 3 most relevant documents for a query
results = collection.query(
    query_texts=["how do I run tasks at the same time in Python?"],
    n_results=3,
)

# Results are sorted by similarity (closest first)
for doc, distance, metadata in zip(
    results["documents"][0],
    results["distances"][0],
    results["metadatas"][0],

4. Free Embeddings with sentence-transformers

OpenAI embeddings cost money per token. For local development or open-source projects, sentence-transformers runs embedding models locally — free, private, no API key needed.

$ pip install sentence-transformers
Python 
from chromadb.utils.embedding_functions import SentenceTransformerEmbeddingFunction

# Downloads model on first use (~90MB for all-MiniLM-L6-v2)
embedding_fn = SentenceTransformerEmbeddingFunction(
    model_name="all-MiniLM-L6-v2"   # good balance: fast, decent quality
)

collection = client.get_or_create_collection(
    name="local_docs",
    embedding_function=embedding_fn,
)

# Everything else is identical — the embedding_fn is swappable
collection.add(
    documents=["Python uses duck typing instead of interfaces."],
    ids=["doc-1"],
)

5. The RAG Pattern End-to-End

RAG = Retrieve → Augment → Generate. Find relevant chunks from your vector store, inject them into the prompt as context, then let the LLM answer based on that context. This is how you give an LLM knowledge of your private documents.

Python — complete RAG pipeline 
import chromadb
from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# Setup
client = chromadb.PersistentClient(path="./chroma_db")
embedding_fn = OpenAIEmbeddingFunction(
    api_key="your-api-key",
    model_name="text-embedding-3-small",
)
collection = client.get_or_create_collection("python_docs", embedding_function=embedding_fn)
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

RAG_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """You are a Python expert helping a Java developer learn Python.
Answer the question using ONLY the provided context.
If the context doesn't contain the answer, say so honestly.

Context:
{context}"""),
    ("human", "{question}"),
])

def rag_query(question: str, n_results: int = 3) -> str:
    # Step 1: RETRIEVE — find relevant chunks
    results = collection.query(
        query_texts=[question],
        n_results=n_results,
    )
    retrieved_docs = results["documents"][0]

    # Step 2: AUGMENT — build context string
    context = "\n\n".join(
        f"[Source {i+1}]: {doc}"
        for i, doc in enumerate(retrieved_docs)
    )

    # Step 3: GENERATE — LLM answers using the context
    chain = RAG_PROMPT | llm
    response = chain.invoke({"context": context, "question": question})
    return response.content

# Use it
answer = rag_query("How do I run multiple things concurrently in Python?")
print(answer)

Chunking Strategy

Python — chunking a large document 
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,         # characters per chunk
    chunk_overlap=50,       # overlap between chunks (preserves context at boundaries)
    separators=["\n\n", "\n", ". ", " "],  # split preference order
)

with open("python_tutorial.txt") as f:
    text = f.read()

chunks = splitter.split_text(text)

# Add all chunks to ChromaDB
collection.add(
    documents=chunks,
    ids=[f"chunk-{i}" for i in range(len(chunks))],
)

print(f"Indexed {len(chunks)} chunks from the document")

Wrap-up

  • Embeddings capture semantic meaning, not keywords. "Python dict" and "Java HashMap" have similar vectors even though they share no words.
  • ChromaDB's get_or_create_collection is idempotent — safe to call on every app startup. Use PersistentClient for anything beyond a quick demo.
  • The RAG pattern is three steps: retrieve relevant chunks, inject them as context in the prompt, let the LLM generate from that context. Keep each step explicit so you can debug which step fails.
  • Chunk size matters. Too large: you waste tokens on irrelevant content. Too small: you lose context. 300–600 characters with 50-character overlap is a reasonable starting point for prose.
  • Swap OpenAIEmbeddingFunction for SentenceTransformerEmbeddingFunction for zero-cost local embeddings — useful during development and for private data.

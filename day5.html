1. String Methods

Python strings are objects with a rich set of built-in methods. Unlike Java, most return a new string (since strings are immutable) — no StringBuilder needed for simple operations.

Splitting and joining

Java 
String csv = "alice,bob,carol";
String[] parts = csv.split(",");   // ["alice","bob","carol"]
String joined = String.join(", ", parts);  // "alice, bob, carol"

// Split with limit
"a.b.c.d".split("\\.", 2);  // ["a", "b.c.d"]
Python 
csv = "alice,bob,carol"
parts = csv.split(",")          # ["alice", "bob", "carol"]
joined = ", ".join(parts)       # "alice, bob, carol"

# join is called ON the separator, not the list
# This feels backward at first but makes sense:
# the separator is a str, and join is a str method

# Split with limit
"a.b.c.d".split(".", 1)        # ["a", "b.c.d"]

# Split on whitespace (default) — handles multiple spaces
"  hello   world  ".split()    # ["hello", "world"]

Stripping, searching, replacing

Python 
# Strip whitespace (or specified chars)
"  hello  ".strip()       # "hello"
"  hello  ".lstrip()      # "hello  "
"  hello  ".rstrip()      # "  hello"
"###hello###".strip("#")  # "hello"

# Case conversion
"Hello World".upper()     # "HELLO WORLD"
"Hello World".lower()     # "hello world"
"hello world".title()     # "Hello World"
"hello world".capitalize()# "Hello world" (only first char)

# Checking content
"hello".startswith("hel")   # True
"hello".endswith("llo")     # True
"hello123".isalnum()        # True
"12345".isdigit()           # True
"hello".isalpha()           # True

# Finding and replacing
"hello world".find("world")     # 6  (index, or -1 if not found)
"hello world".index("world")    # 6  (ValueError if not found)
"hello world".count("l")        # 3
"hello world".replace("world", "Python")  # "hello Python"
"aabbcc".replace("b", "", 1)    # "aabbcc" → replaces first only

Chaining methods

Python 
# Methods return new strings, so you can chain them
raw = "  Hello, WORLD!  "
clean = raw.strip().lower().replace("!", "")
# "hello, world"

# Practical: normalize user input
email = "  Alice@Example.COM  ".strip().lower()
# "alice@example.com"

2. String Slicing

String slicing works exactly like list slicing (Day 4) since strings are sequences too. The syntax is s[start:stop:step].

Java 
String s = "Hello, World!";

s.substring(0, 5);    // "Hello"
s.substring(7);       // "World!"
s.charAt(0);          // 'H'

// Reverse a string (verbose)
new StringBuilder(s).reverse().toString();
Python 
s = "Hello, World!"

s[0:5]       # "Hello"
s[7:]        # "World!"
s[0]         # "H"
s[-1]        # "!"  (last char)
s[-6:]       # "orld!"

# Step
s[::2]       # "Hlo ol!"  (every 2nd char)

# Reverse a string — one line
s[::-1]      # "!dlroW ,olleH"

# Check palindrome
word = "racecar"
word == word[::-1]   # True

4. Regular Expressions

Python's re module provides regex support. Unlike Java, you don't need to create a Pattern object first — though you can compile for repeated use.

Core regex functions

Java 
import java.util.regex.*;

String text = "Call 555-1234 or 555-5678";

// Compile pattern
Pattern p = Pattern.compile("\\d{3}-\\d{4}");
Matcher m = p.matcher(text);

// Find first match
if (m.find()) {
    m.group();   // "555-1234"
}

// Find all matches
List<String> all = new ArrayList<>();
while (m.find()) {
    all.add(m.group());
}
Python 
import re

text = "Call 555-1234 or 555-5678"

# search — find first match anywhere in string
match = re.search(r"\d{3}-\d{4}", text)
if match:
    match.group()   # "555-1234"

# findall — find ALL matches, return as list
phones = re.findall(r"\d{3}-\d{4}", text)
# ["555-1234", "555-5678"]

# match — only matches at START of string
re.match(r"\d+", "123abc")   # match object
re.match(r"\d+", "abc123")   # None (didn't start with digits)
Use raw strings! Always prefix regex patterns with r (like r"\d+"). Without it, Python interprets backslashes as escape sequences before the regex engine sees them. r"\d+" ensures \d reaches the regex engine intact.

Substitution and groups

Python 
# sub — search and replace with regex
text = "Hello World"
re.sub(r"World", "Python", text)    # "Hello Python"

# Replace with pattern
phone = "Call (555) 123-4567"
clean = re.sub(r"[^\d]", "", phone)  # "5551234567" (digits only)

# Groups — capture parts of the match
match = re.search(r"(\d{3})-(\d{4})", "Call 555-1234")
match.group(0)   # "555-1234"  (full match)
match.group(1)   # "555"       (first group)
match.group(2)   # "1234"      (second group)

# Named groups
match = re.search(r"(?P<area>\d{3})-(?P<number>\d{4})", "Call 555-1234")
match.group("area")     # "555"
match.group("number")   # "1234"

# Compile for reuse (like Java's Pattern.compile)
email_pattern = re.compile(r"[\w.+-]+@[\w-]+\.[\w.]+")
emails = email_pattern.findall("Contact alice@example.com or bob@test.org")
# ["alice@example.com", "bob@test.org"]

5. File I/O with open()

Python's file I/O is dramatically simpler than Java's. The open() function combined with the with statement handles everything — no BufferedReader, FileInputStream, or try/finally boilerplate.

Reading files

Java 
// Read entire file (Java 11+)
String content = Files.readString(
    Path.of("data.txt"));

// Read lines
List<String> lines = Files.readAllLines(
    Path.of("data.txt"));

// Buffered reading
try (var reader = Files.newBufferedReader(
        Path.of("data.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
}
Python 
# Read entire file
with open("data.txt") as f:
    content = f.read()     # entire file as one string

# Read all lines into a list
with open("data.txt") as f:
    lines = f.readlines()  # ["line1\n", "line2\n", ...]

# Iterate line by line (memory-efficient for large files)
with open("data.txt") as f:
    for line in f:
        print(line.strip())  # strip removes trailing \n

Writing and appending

Python 
# Write (creates or overwrites)
with open("output.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("Second line\n")

# Append (adds to existing file)
with open("output.txt", "a") as f:
    f.write("Appended line\n")

# Write multiple lines at once
lines = ["line 1\n", "line 2\n", "line 3\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)   # does NOT add newlines automatically

# Write with print (convenience)
with open("output.txt", "w") as f:
    print("Line 1", file=f)       # adds newline automatically
    print("Line 2", file=f)

The context manager pattern

Why with matters: The with statement guarantees the file is closed when the block exits — even if an exception occurs. This is Python's equivalent of Java's try-with-resources. You'll see this pattern everywhere: database connections, network sockets, locks. Full deep-dive on Day 12.
Java — try-with-resources 
try (var writer = new FileWriter("out.txt")) {
    writer.write("Hello");
}   // auto-closed here
Python — with statement 
with open("out.txt", "w") as writer:
    writer.write("Hello")
# auto-closed here

# File modes:
# "r"  — read (default)
# "w"  — write (truncate)
# "a"  — append
# "x"  — create (fail if exists)
# "rb" — read binary
# "wb" — write binary

6. CSV Reading and Writing

The csv module is part of Python's standard library — no external dependency needed. It handles quoting, escaping, and delimiters automatically.

Reading CSV

Python 
import csv

# Basic CSV reader — returns lists
with open("users.csv") as f:
    reader = csv.reader(f)
    header = next(reader)      # skip header row
    for row in reader:
        print(row)             # ["Alice", "30", "NYC"]

# DictReader — each row is a dict (most useful)
with open("users.csv") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"], row["age"])
        # keys come from the header row automatically

Writing CSV

Python 
import csv

# Basic writer
with open("output.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["name", "age", "city"])       # header
    writer.writerow(["Alice", 30, "NYC"])
    writer.writerow(["Bob", 25, "London"])

# DictWriter — write from dicts
users = [
    {"name": "Alice", "age": 30, "city": "NYC"},
    {"name": "Bob",   "age": 25, "city": "London"},
]

with open("output.csv", "w", newline="") as f:
    fields = ["name", "age", "city"]
    writer = csv.DictWriter(f, fieldnames=fields)
    writer.writeheader()     # writes the header row
    writer.writerows(users)  # writes all data rows
Note: Always pass newline="" when opening CSV files for writing. Without it, you may get extra blank lines on Windows. This is a common gotcha.

Putting it together — practical file processing

Python — process a log file 
import re
from collections import Counter

# Read a log file, count error types
error_counts = Counter()

with open("app.log") as f:
    for line in f:
        match = re.search(r"ERROR: (\w+)", line)
        if match:
            error_counts[match.group(1)] += 1

# Write a summary report
with open("error_report.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["Error Type", "Count"])
    for error, count in error_counts.most_common():
        writer.writerow([error, count])

Wrap-up

  • Python strings have a rich set of built-in methods (split, join, strip, replace, startswith). They return new strings — strings are immutable.
  • String slicing (s[1:4], s[::-1]) works identically to list slicing. Use s[::-1] to reverse a string in one expression.
  • The re module handles regex. Use re.search() for first match, re.findall() for all matches, re.sub() for replacement. Always use raw strings (r"\d+").
  • File I/O with with open() is the idiomatic pattern — guaranteed cleanup like Java's try-with-resources but much less verbose.
  • The csv module's DictReader/DictWriter make CSV processing trivial. Each row becomes a dict keyed by header names.

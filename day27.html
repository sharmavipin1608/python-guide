Day 27 Phase 2 — Core Libraries

pytest

Python's testing story is cleaner than Java's. No test class required — just a function named test_ and a plain assert. Fixtures replace @BeforeEach setup, and parametrize replaces data providers.

Plan for the Day

What you'll cover

  • Writing test functions — no class, just assert
  • pytest fixtures — dependency injection for tests
  • @pytest.mark.parametrize — data-driven tests
  • Testing exceptions with pytest.raises()
  • Mocking with unittest.mock and pytest-mock
  • Running tests: filtering, verbose output, coverage

Java parallels

  • Test function → JUnit 5 @Test method
  • assert x == y → assertEquals(y, x)
  • Fixture → @BeforeEach setup / @AfterEach teardown
  • @parametrize → JUnit 5 @ParameterizedTest + @MethodSource
  • pytest.raises() → assertThrows()
  • unittest.mock.MagicMock → Mockito mock()
  • monkeypatch → no direct equivalent (closest: Mockito static mocking)

Writing Tests

Test functions — no class needed

A test is any function whose name starts with test_. Use plain Python assert — pytest rewrites assertions to show helpful failure messages without any special assertion methods.

Java — JUnit 5 
class BankAccountTest {
    @Test
    void deposit_increases_balance() {
        BankAccount acc = new BankAccount(100.0);
        acc.deposit(50.0);
        assertEquals(150.0, acc.getBalance());
    }

    @Test
    void withdraw_below_zero_throws() {
        BankAccount acc = new BankAccount(100.0);
        assertThrows(IllegalStateException.class,
            () -> acc.withdraw(200.0));
    }
}
Python — pytest 
# test_bank_account.py
import pytest
from bank_account import BankAccount

def test_deposit_increases_balance():
    acc = BankAccount(100.0)
    acc.deposit(50.0)
    assert acc.balance == 150.0   # plain assert — pytest shows a diff on failure

def test_withdraw_below_zero_raises():
    acc = BankAccount(100.0)
    with pytest.raises(ValueError) as exc_info:
        acc.withdraw(200.0)
    assert "insufficient" in str(exc_info.value).lower()

def test_repr():
    acc = BankAccount(100.0)
    assert repr(acc) == "BankAccount(balance=100.0)"

Running tests

Shell 
pip install pytest

pytest                          # run all tests in current dir
pytest tests/                   # run tests in a directory
pytest test_bank_account.py     # run a specific file
pytest test_bank_account.py::test_deposit_increases_balance  # specific test
pytest -v                       # verbose — show each test name
pytest -k "deposit"             # run only tests whose name contains "deposit"
pytest -x                       # stop at first failure
pytest --tb=short               # shorter traceback format

Fixtures

Fixtures are how pytest does dependency injection. A fixture is a function decorated with @pytest.fixture. Any test that names a fixture as a parameter automatically receives its return value. pytest handles setup and teardown automatically.

Java — JUnit 5 
class TaskManagerTest {
    TaskManager manager;

    @BeforeEach
    void setUp() {
        manager = new TaskManager(new InMemoryStorage());
    }

    @AfterEach
    void tearDown() {
        manager.clear();
    }

    @Test
    void add_task_increases_count() {
        manager.add("Buy milk");
        assertEquals(1, manager.count());
    }
}
Python — pytest fixtures 
import pytest
from task_manager import TaskManager
from storage import MemoryStorage

@pytest.fixture
def manager():
    """Provides a fresh TaskManager for each test."""
    return TaskManager(MemoryStorage())

@pytest.fixture
def manager_with_tasks(manager):
    """Fixture can depend on other fixtures."""
    manager.add("Buy milk")
    manager.add("Walk dog")
    return manager

def test_add_increases_count(manager):
    manager.add("Buy milk")
    assert manager.count() == 1

def test_complete_marks_done(manager_with_tasks):
    tasks = manager_with_tasks.all()
    manager_with_tasks.complete(tasks[0].id)
    assert tasks[0].done is True

Fixtures with setup and teardown

Python 
import pytest
import tempfile
import os

@pytest.fixture
def temp_dir():
    """Create a temp directory, yield it, clean up after."""
    with tempfile.TemporaryDirectory() as d:
        yield d   # test runs here
    # TemporaryDirectory context manager cleans up after yield

@pytest.fixture
def db_connection():
    """Open a DB connection, yield it, close after."""
    conn = create_connection(":memory:")  # in-memory SQLite
    conn.execute("CREATE TABLE tasks (id INTEGER, title TEXT)")
    yield conn
    conn.close()   # always runs, even if test fails

def test_writes_file(temp_dir):
    path = os.path.join(temp_dir, "output.txt")
    write_report(path)
    assert os.path.exists(path)

Fixture scope — control how often a fixture is created

import pytest

@pytest.fixture(scope="function")  # default — new instance per test

    # Set up once (e.g. start a server, load a large file)

    yield resource
    resource.stop()

@pytest.mark.parametrize("input_str, expected", [
    ("42",  42),
    ("-7",  -7),
    ("0",    0),
    ("  5 ", 5),   # whitespace should be stripped

def test_parse_int(input_str: str, expected: int):

@pytest.mark.parametrize("a, b, result", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),

Mocking

unittest.mock — patch external dependencies

Mocks replace external dependencies (HTTP calls, file system, databases) so tests are fast and isolated. Use patch() to replace a name during a test.

Python 
from unittest.mock import patch, MagicMock
import pytest

# Patch requests.get so no real HTTP calls are made
def test_fetch_user_returns_data():
    mock_response = MagicMock()
    mock_response.status_code = 200
    mock_response.json.return_value = {"login": "torvalds", "public_repos": 8}

    with patch("my_module.requests.get", return_value=mock_response) as mock_get:
        user = fetch_user("torvalds")

    assert user["login"] == "torvalds"
    mock_get.assert_called_once_with(
        "https://api.github.com/users/torvalds",
        timeout=10,
    )

# Using pytest-mock (pip install pytest-mock) — cleaner syntax
def test_fetch_user_404(mocker):
    mock_response = MagicMock()
    mock_response.status_code = 404
    mock_response.raise_for_status.side_effect = Exception("404 Not Found")

    mocker.patch("my_module.requests.get", return_value=mock_response)

    with pytest.raises(ValueError, match="not found"):
        fetch_user("nonexistent_user")

monkeypatch — pytest's built-in patcher

Python 
import pytest
import os

def get_config() -> dict:
    return {
        "db_host": os.environ.get("DB_HOST", "localhost"),
        "debug":   os.environ.get("DEBUG", "false") == "true",
    }

def test_reads_env_vars(monkeypatch):
    monkeypatch.setenv("DB_HOST", "test-db")
    monkeypatch.setenv("DEBUG", "true")

    config = get_config()
    assert config["db_host"] == "test-db"
    assert config["debug"] is True
    # env vars restored automatically after test

def test_patches_function(monkeypatch):
    monkeypatch.setattr("my_module.datetime", MagicMock(
        now=lambda: datetime(2024, 1, 1, 12, 0, 0)
    ))
    # now datetime.now() always returns 2024-01-01 12:00:00 in this test

Quick Reference Card

Basic test
def test_name(): assert result == expected

File must be named test_*.py. Function must start with test_.

Test exception
with pytest.raises(ValueError): risky_fn()

Use match="pattern" to also check the error message.

Fixture
@pytest.fixture / def my_thing(): return Thing()

Name the fixture as a test parameter to receive it.

Fixture with teardown
@pytest.fixture / def res(): setup(); yield obj; cleanup()

Code after yield always runs — even if the test fails.

Parametrize
@pytest.mark.parametrize("a,b", [(1,2),(3,4)])

Runs the test once per tuple. Each run appears as a separate test.

Mock
with patch("module.fn") as m: m.return_value = 42

Replaces module.fn with a mock for the duration of the with block.

Run specific tests
pytest -k "test_deposit" -v

-k filters by name substring. -v shows each test. -x stops on first fail.

Coverage
pytest --cov=src --cov-report=term-missing

Requires pip install pytest-cov. Shows which lines aren't tested.

Wrap-up

  • pytest needs no class, no import of assert methods, and no test runner configuration. A file named test_*.py with functions named test_* is a complete test suite. This is dramatically less boilerplate than JUnit.
  • Fixtures are how pytest handles setup and teardown. They're composable (fixtures can depend on fixtures), scoped (function/module/session), and automatically injected — just name the fixture as a parameter.
  • @pytest.mark.parametrize generates one test per input case — you see each as a separate pass/fail in the output. Use it instead of a loop inside a single test function.
  • Mock only external dependencies — HTTP, file I/O, databases, time, randomness. Don't mock your own classes; test them through their public interface. Over-mocking leads to tests that pass but don't catch real bugs.
  • Use monkeypatch for patching environment variables, module attributes, and file system calls — it automatically restores originals after each test, avoiding test-order dependencies.

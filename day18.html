datetime

The core types

Python's datetime module contains four main types: date (year/month/day), time (hour/minute/second/microsecond), datetime (date + time), and timedelta (duration). You'll mostly use datetime and timedelta.

Java 
LocalDate today = LocalDate.now();
LocalDateTime now = LocalDateTime.now();
LocalDateTime specific = LocalDateTime.of(2024, 3, 15, 10, 30, 0);
Duration duration = Duration.ofDays(7);
LocalDateTime future = now.plus(duration);
Python 
from datetime import datetime, date, time, timedelta

today = date.today()                         # 2024-03-15
now   = datetime.now()                       # 2024-03-15 10:30:45.123456
specific = datetime(2024, 3, 15, 10, 30, 0)  # year, month, day, h, m, s

# Arithmetic with timedelta
one_week  = timedelta(days=7)
next_week = now + one_week
yesterday = today - timedelta(days=1)
diff      = datetime(2024, 12, 31) - datetime.now()
print(f"{diff.days} days until year end")

# Extracting components
print(now.year, now.month, now.day)           # 2024 3 15
print(now.hour, now.minute, now.second)       # 10 30 45
print(now.weekday())   # 0=Monday … 6=Sunday
print(now.isoformat()) # '2024-03-15T10:30:45.123456'

Timezone-aware datetimes

A naive datetime has no timezone info — like a timestamp without a label. A aware datetime knows its timezone. Always use aware datetimes in backend systems that serve multiple timezones. Naive datetimes are a silent source of bugs.

Python 
from datetime import datetime, timezone, timedelta

# UTC — always use for storage and APIs
utc_now = datetime.now(tz=timezone.utc)
print(utc_now)              # 2024-03-15 10:30:45.123456+00:00
print(utc_now.tzinfo)       # UTC

# Create a fixed-offset timezone (e.g. IST = UTC+5:30)
IST = timezone(timedelta(hours=5, minutes=30))
ist_now = datetime.now(tz=IST)
print(ist_now)              # 2024-03-15 16:00:45.123456+05:30

# Convert between timezones
ist_from_utc = utc_now.astimezone(IST)
print(ist_from_utc)         # same moment, different representation

# Never mix naive and aware datetimes — raises TypeError
naive = datetime.now()
# naive + utc_now    # TypeError!

# Python 3.11+ shortcut
utc_now = datetime.now(timezone.utc)   # preferred

# For production: use the 'zoneinfo' module (Python 3.9+)
from zoneinfo import ZoneInfo
ny = datetime.now(tz=ZoneInfo("America/New_York"))
london = datetime.now(tz=ZoneInfo("Europe/London"))
print(ny.isoformat())   # handles DST automatically
Rule of thumb: Store all datetimes in UTC. Convert to local time only for display. This is the same convention as Java's ZonedDateTime / Spring Boot's default UTC storage.

Formatting and parsing

Java 
DateTimeFormatter fmt = DateTimeFormatter
    .ofPattern("yyyy-MM-dd HH:mm:ss");

String formatted = LocalDateTime.now().format(fmt);
LocalDateTime parsed = LocalDateTime.parse("2024-03-15 10:30:00", fmt);
Python 
from datetime import datetime

now = datetime.now()

# strftime — format datetime to string
print(now.strftime("%Y-%m-%d"))          # 2024-03-15
print(now.strftime("%d/%m/%Y %H:%M"))    # 15/03/2024 10:30
print(now.strftime("%B %d, %Y"))         # March 15, 2024
print(now.strftime("%A"))                # Friday

# strptime — parse string to datetime
dt = datetime.strptime("2024-03-15 10:30:00", "%Y-%m-%d %H:%M:%S")
dt = datetime.strptime("March 15, 2024", "%B %d, %Y")

# ISO format — use for APIs and JSON (no format string needed)
iso = now.isoformat()                    # '2024-03-15T10:30:45.123456'
back = datetime.fromisoformat(iso)       # round-trips perfectly

# Common format codes
# %Y=4-digit year, %m=month, %d=day
# %H=24h hour, %M=minute, %S=second
# %A=weekday name, %B=month name, %f=microseconds

json

loads / dumps — string serialization

json.dumps converts a Python object to a JSON string. json.loads parses a JSON string back to Python. The type mapping is: dict↔object, list↔array, str↔string, int/float↔number, True/False↔true/false, None↔null.

Java — Jackson 
ObjectMapper mapper = new ObjectMapper();

// Serialize
String json = mapper.writeValueAsString(myObject);

// Deserialize
MyClass obj = mapper.readValue(json, MyClass.class);
Map<?,?> map = mapper.readValue(json, Map.class);
Python 
import json

# Python → JSON string
data = {
    "name": "Alice",
    "age": 30,
    "active": True,
    "scores": [95, 87, 92],
    "address": None,
}

json_str = json.dumps(data)
print(json_str)
# {"name": "Alice", "age": 30, "active": true, ...}

# Pretty-print for readability
pretty = json.dumps(data, indent=2, sort_keys=True)
print(pretty)

# JSON string → Python
parsed = json.loads(json_str)
print(type(parsed))        # dict
print(parsed["name"])      # Alice
print(type(parsed["active"]))  # bool

load / dump — file-based serialization

Python 
import json
from pathlib import Path

data = {"users": [{"id": 1, "name": "Alice"}, {"id": 2, "name": "Bob"}]}

# Write to file
with open("users.json", "w") as f:
    json.dump(data, f, indent=2)

# Or with pathlib (simpler for small files)
Path("users.json").write_text(json.dumps(data, indent=2))

# Read from file
with open("users.json") as f:
    loaded = json.load(f)

# Or with pathlib
loaded = json.loads(Path("users.json").read_text())

Custom encoder — non-serializable types

By default, json.dumps raises TypeError for types it doesn't know: datetime, Decimal, set, custom classes. You handle this with a custom encoder.

Python 
import json
from datetime import datetime, date
from decimal import Decimal
from dataclasses import dataclass, asdict


class AppEncoder(json.JSONEncoder):
    """Custom JSON encoder for types not supported by default."""

    def default(self, obj):
        if isinstance(obj, datetime):
            return obj.isoformat()          # → "2024-03-15T10:30:00"
        if isinstance(obj, date):
            return obj.isoformat()          # → "2024-03-15"
        if isinstance(obj, Decimal):
            return float(obj)               # → 9.99
        if isinstance(obj, set):
            return sorted(obj)              # → list (sorted for determinism)
        # For dataclasses — use asdict()
        try:
            return asdict(obj)
        except TypeError:
            pass
        return super().default(obj)         # raises TypeError for unknown types


@dataclass
class Order:
    id: int
    amount: Decimal
    created_at: datetime
    tags: set[str]


order = Order(
    id=42,
    amount=Decimal("99.99"),
    created_at=datetime(2024, 3, 15, 10, 30),
    tags={"urgent", "vip"},
)

# Without custom encoder: TypeError
# json.dumps(order)  ← TypeError

# With custom encoder
json_str = json.dumps(order, cls=AppEncoder, indent=2)
print(json_str)
# {
#   "id": 42,
#   "amount": 99.99,
#   "created_at": "2024-03-15T10:30:00",
#   "tags": ["urgent", "vip"]
# }

# Alternative: use default= parameter for simple cases
json.dumps({"dt": datetime.now()}, default=str)  # converts anything to str

Custom decoder — parsing back to types

Python 
import json
from datetime import datetime

def decode_datetime(dct: dict) -> dict:
    """Convert ISO datetime strings back to datetime objects during parse."""
    for key, value in dct.items():
        if isinstance(value, str):
            try:
                dct[key] = datetime.fromisoformat(value)
            except ValueError:
                pass   # not a datetime — leave as string
    return dct


data = '{"name": "Alice", "created_at": "2024-03-15T10:30:00"}'
parsed = json.loads(data, object_hook=decode_datetime)
print(type(parsed["created_at"]))  # datetime — not string!
print(parsed["created_at"].year)   # 2024

Quick Reference Card

Current UTC time
datetime.now(timezone.utc)

Always use UTC for storage. Import timezone from datetime.

timedelta arithmetic
datetime.now() + timedelta(days=7, hours=3)

Add/subtract durations from datetimes. Supports days, seconds, microseconds, etc.

Format to string
dt.strftime("%Y-%m-%d %H:%M:%S")

Common codes: %Y year, %m month, %d day, %H hour, %M min

Parse from string
datetime.strptime("2024-03-15", "%Y-%m-%d")

Format must match exactly. For ISO strings use fromisoformat().

ISO format round-trip
dt.isoformat() / datetime.fromisoformat(s)

Best choice for JSON APIs and databases. Preserves timezone info.

JSON encode
json.dumps(data, indent=2)

Python → JSON string. Use json.dump(data, file) for file output.

JSON decode
json.loads(string) / json.load(file)

JSON string/file → Python dict/list/etc.

Custom encoder
json.dumps(obj, cls=MyEncoder)

Subclass json.JSONEncoder, override default(self, obj).

Quick non-serializable fix
json.dumps(obj, default=str)

Converts anything unknown to its string representation. Good for logging, not APIs.

Wrap-up

  • Always use timezone-aware datetimes in backend code. Naive datetimes (no timezone) silently cause bugs when servers change timezone or data crosses boundaries. datetime.now(timezone.utc) is the safe default.
  • datetime.isoformat() and datetime.fromisoformat() are your API serialization format — they round-trip perfectly and are universally understood. Use strftime/strptime only when dealing with legacy or user-facing format requirements.
  • json.dumps/json.loads handle basic Python types automatically. For datetime, Decimal, dataclasses, or custom objects, write a JSONEncoder subclass and pass it as cls=.
  • json.dumps(obj, default=str) is a quick workaround for logging and debugging — it converts anything non-serializable to its string repr. Don't use it in APIs where type fidelity matters on the receiving end.
  • Store times in UTC, compute in UTC, convert to local only at the presentation layer. This is the same rule as in Java's ZonedDateTime ecosystem — it prevents a whole class of timezone-related bugs.

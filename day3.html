Functions

Python functions are first-class objects. Learn default parameters, *args, **kwargs, multiple return values, lambdas, higher-order functions, and get your first taste of decorators.

Java parallels at a glance

1. Defining Functions

Java 
// Must declare return type and param types
public String greet(String name) {
    return "Hello, " + name + "!";
}

// void for no return value
public void printGreeting(String name) {
    System.out.println(greet(name));
}

// Call it
String msg = greet("Alice");
printGreeting("Bob");
Python 
# def keyword, no return type needed
def greet(name):
    return f"Hello, {name}!"

# Functions that don't return: implicitly return None
def print_greeting(name):
    print(greet(name))

# Call it
msg = greet("Alice")
print_greeting("Bob")

# type hints (optional, for tooling)
def greet_typed(name: str) -> str:
    return f"Hello, {name}!"
Docstrings: Python functions document themselves with a string literal as the first statement (a docstring). It's accessible via help() and is used by IDEs and tools.
Python — docstrings 
def calculate_tax(amount: float, rate: float) -> float:
    """
    Calculate tax on an amount.

    Args:
        amount: The pre-tax amount
        rate: Tax rate as a decimal (e.g. 0.08 for 8%)

    Returns:
        The tax amount (not the total)
    """
    return amount * rate

help(calculate_tax)   # prints the docstring
calculate_tax.__doc__ # access it as a string

2. Default Parameter Values

In Java, you'd create multiple overloaded methods to give parameters optional values. Python handles this in a single function definition.

Java — overloading 
// Three overloads for "optional" params
public String connect(String host) {
    return connect(host, 5432);
}

public String connect(String host, int port) {
    return connect(host, port, "mydb");
}

public String connect(String host, int port,
                       String db) {
    return host + ":" + port + "/" + db;
}
Python — default values 
# One function with defaults
def connect(host, port=5432, db="mydb"):
    return f"{host}:{port}/{db}"

# Call with just required arg
connect("localhost")
# "localhost:5432/mydb"

# Override one default
connect("localhost", db="orders")
# "localhost:5432/orders"

# Override all
connect("prod-db", 5433, "prod")

Positional vs keyword arguments

Python 
def create_user(name, age, role="user", active=True):
    return {"name": name, "age": age, "role": role, "active": active}

# Positional (order matters)
create_user("Alice", 30)

# Keyword (order doesn't matter)
create_user(age=30, name="Alice")

# Mix: positional first, then keyword
create_user("Alice", 30, role="admin")

# All keyword — most readable for many params
create_user(name="Alice", age=30, role="admin", active=False)

# You CAN'T: keyword before positional
# create_user(name="Alice", 30)  # SyntaxError
Mutable default argument trap — important! Never use a mutable object (list, dict) as a default value. This is a famous Python gotcha:
Python — the mutable default trap 
# WRONG: the list is created ONCE and shared across calls
def add_item_bad(item, items=[]):
    items.append(item)
    return items

add_item_bad("apple")   # ["apple"]
add_item_bad("banana")  # ["apple", "banana"] ← bug!

# CORRECT: use None as sentinel, create fresh object inside
def add_item_good(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items

add_item_good("apple")   # ["apple"]
add_item_good("banana")  # ["banana"] ← correct

3. *args — Variable Positional Arguments

*args lets a function accept any number of positional arguments, collecting them into a tuple. It's Python's equivalent of Java's varargs.

Java — varargs 
public int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) {
        total += n;
    }
    return total;
}

sum(1, 2, 3);         // 6
sum(1, 2, 3, 4, 5);   // 15
Python — *args 
def total(*numbers):   # 'numbers' is a tuple
    return sum(numbers)

total(1, 2, 3)         # 6
total(1, 2, 3, 4, 5)  # 15

# Can mix with regular params
# Regular params MUST come before *args
def log(level, *messages):
    for msg in messages:
        print(f"[{level}] {msg}")

log("INFO", "Server started", "Listening on :8080")

# Unpacking a list into *args with the * operator
nums = [1, 2, 3]
total(*nums)   # same as total(1, 2, 3)

4. **kwargs — Variable Keyword Arguments

**kwargs collects any extra keyword arguments into a dict. There's no direct Java equivalent — the closest approximation is passing a Map<String, Object>.

Python 
def create_profile(name, **attributes):
    profile = {"name": name}
    profile.update(attributes)  # merge the extra kwargs in
    return profile

# Any keyword args beyond 'name' go into 'attributes' dict
create_profile("Alice", age=30, city="NYC", role="admin")
# {"name": "Alice", "age": 30, "city": "NYC", "role": "admin"}

# Iterating over kwargs
def print_config(**settings):
    for key, value in settings.items():
        print(f"  {key}: {value}")

print_config(host="localhost", port=5432, timeout=30)

# Unpacking a dict into **kwargs with the ** operator
config = {"host": "localhost", "port": 5432}
print_config(**config)   # same as print_config(host=..., port=...)

Full signature order

When combining all parameter types, they must appear in this order: positional → *args → keyword-only → **kwargs.

Python — full parameter signature 
def full_example(
    required,             # required positional
    optional="default",   # optional positional with default
    *args,                # extra positionals → tuple
    keyword_only,         # after * = keyword-only (no positional)
    **kwargs              # extra keywords → dict
):
    print(required, optional, args, keyword_only, kwargs)

full_example("a", "b", "c", "d", keyword_only="kw", x=1, y=2)
# a  b  ('c', 'd')  kw  {'x': 1, 'y': 2}

# Keyword-only params are a great pattern for API design:
def send_email(to, subject, *, cc=None, bcc=None):
    # cc and bcc must be passed as keywords
    # Prevents mistakes like send_email("a@b.com", "Hi", "c@b.com")
    pass

5. Returning Multiple Values

Python functions can return multiple values by returning a tuple. The caller can unpack them directly.

Java — need a wrapper 
// Java 16+ record as return type
record MinMax(int min, int max) {}

public MinMax findMinMax(int[] nums) {
    int min = nums[0], max = nums[0];
    for (int n : nums) {
        if (n < min) min = n;
        if (n > max) max = n;
    }
    return new MinMax(min, max);
}

MinMax result = findMinMax(arr);
System.out.println(result.min() + " " + result.max());
Python — tuples 
def find_min_max(nums):
    return min(nums), max(nums)  # returns a tuple

# Tuple unpacking — most common usage
low, high = find_min_max([3, 1, 4, 1, 5, 9])
print(low, high)   # 1  9

# Can also capture as tuple
result = find_min_max([3, 1, 4])
print(result)       # (1, 4)
print(result[0])    # 1

# Ignore parts with _
low, _ = find_min_max([3, 1, 4])   # only care about min

6. Lambda Functions

Lambdas are anonymous single-expression functions. In Python they're limited to one expression — no statements, no multi-line logic. For anything complex, use a named def.

Java lambdas 
// Single param
Function<Integer, Integer> square = x -> x * x;

// Two params
BiFunction<Integer, Integer, Integer> add =
    (a, b) -> a + b;

// With block body
Function<String, String> trim =
    s -> {
        String result = s.strip();
        return result.toLowerCase();
    };

// Sorting with comparator
list.sort((a, b) -> a.length() - b.length());
Python lambdas 
# lambda params: expression
square = lambda x: x * x
add = lambda a, b: a + b

# Single expression only — no multi-line
# For complex logic, use def instead

# Common use: inline sort key
words = ["banana", "apple", "fig", "cherry"]
words.sort(key=lambda w: len(w))
# ["fig", "apple", "banana", "cherry"]

# Or with built-in sorted()
sorted_words = sorted(words, key=lambda w: w[-1])  # sort by last letter

# Sort dicts by a field
users = [{"name": "Bob", "age": 25}, {"name": "Alice", "age": 30}]
users.sort(key=lambda u: u["age"])

# Prefer operator.attrgetter/itemgetter for simple cases
from operator import itemgetter
users.sort(key=itemgetter("age"))  # same result, no lambda needed

7. Functions as First-Class Objects

In Python, functions are objects. You can pass them as arguments, return them from other functions, and store them in data structures. This is used constantly in Python frameworks and libraries.

Java — functional interfaces 
// Must use Function or custom @FunctionalInterface
Function<Integer, Integer> doubler = x -> x * 2;

public List<Integer> applyToAll(
    List<Integer> nums,
    Function<Integer, Integer> fn
) {
    return nums.stream()
        .map(fn)
        .collect(Collectors.toList());
}

applyToAll(List.of(1,2,3), doubler);
Python — functions as values 
# Functions are just objects — no wrapper needed
def double(x):
    return x * 2

def square(x):
    return x * x

# Pass function as argument
def apply_to_all(nums, fn):
    return [fn(n) for n in nums]

apply_to_all([1, 2, 3], double)   # [2, 4, 6]
apply_to_all([1, 2, 3], square)   # [1, 4, 9]

# Store functions in a dict — dispatch table
# (replaces if/elif chains or switch in some cases)
operations = {
    "double": double,
    "square": square,
    "negate": lambda x: -x,
}

op = operations["square"]
op(5)   # 25

Higher-order functions: map, filter, sorted

Python 
nums = [1, 2, 3, 4, 5]

# map() — apply function to each element (returns iterator)
doubled = list(map(double, nums))   # [2, 4, 6, 8, 10]

# filter() — keep elements where function returns True
evens = list(filter(lambda n: n % 2 == 0, nums))  # [2, 4]

# sorted() with key function
words = ["banana", "apple", "fig"]
sorted(words, key=len)                # ["fig", "apple", "banana"]
sorted(words, key=str.upper)          # alphabetical, case-insensitive

# Prefer list comprehensions over map/filter — more Pythonic:
doubled = [double(n) for n in nums]  # clearer
evens   = [n for n in nums if n % 2 == 0]  # clearer

Closures — functions that remember their environment

Python 
def make_multiplier(factor):
    # 'factor' is captured from the outer scope
    def multiply(x):
        return x * factor   # 'factor' lives on even after make_multiplier returns
    return multiply

triple = make_multiplier(3)
triple(10)   # 30

8. Intro to Decorators

A decorator is a function that wraps another function to extend its behavior without modifying the original. You've seen them as annotations in Java — @Override, @Transactional, @Cacheable. Python decorators are similar in intent but implemented differently: they're actual functions that wrap other functions.

Java — AOP / annotations 
// @Transactional wraps the method in a transaction
// without you writing transaction code in the method itself
@Transactional
public void saveUser(User user) {
    repo.save(user);
}

// @Cacheable caches the result
@Cacheable("users")
public User findById(Long id) {
    return repo.findById(id);
}
Python — decorators 
import time

# A simple timing decorator
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)   # call the original
        end = time.time()
        print(f"{func.__name__} took {end - start:.3f}s")
        return result
    return wrapper

# Apply decorator with @ syntax
@timer
def slow_operation():
    time.sleep(0.5)
    return "done"

slow_operation()   # prints "slow_operation took 0.500s"

# @timer is syntactic sugar for:
# slow_operation = timer(slow_operation)
Full decorator coverage is Day 11. Today just understand the concept: a decorator is a function that takes a function, wraps it in another function, and returns the wrapper. The @decorator syntax applies it cleanly.

Built-in decorators you'll use immediately

Python — built-in decorators 
from functools import lru_cache

# @lru_cache — memoize expensive function results
@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

fibonacci(100)   # fast — cached results reused

# @staticmethod — no self, no cls (like Java static method)
class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b

MathUtils.add(1, 2)  # no instance needed

# @classmethod — receives the class itself as first arg
class Counter:
    count = 0

    @classmethod
    def increment(cls):
        cls.count += 1

Putting it all together — a practical function design

Python — real-world function design 
import time
from functools import wraps

def retry(max_attempts=3, delay=1.0):
    """Decorator factory: retries a function on exception."""
    def decorator(func):
        @wraps(func)  # preserves func.__name__, __doc__
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_attempts:
                        raise
                    print(f"Attempt {attempt} failed: {e}. Retrying...")
                    time.sleep(delay)
        return wrapper
    return decorator

# Usage — reads like a configuration
@retry(max_attempts=3, delay=0.5)
def fetch_data(url):
    # might raise requests.RequestException
    pass

Wrap-up

  • Python functions replace Java method overloading via default parameter values — one function, not five overloads.
  • *args collects extra positional arguments into a tuple (like Java varargs). **kwargs collects extra keyword arguments into a dict (no Java equivalent).
  • Returning multiple values via tuple unpacking (x, y = f()) is idiomatic and avoids wrapper classes for simple cases.
  • Functions are first-class objects — pass them around, store them in dicts, return them from other functions. This is the foundation of decorators, callbacks, and functional patterns.
  • Decorators wrap functions to add behavior without modifying the original — like Spring AOP but simpler. You'll build your own on Day 11.

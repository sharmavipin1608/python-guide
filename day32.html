Day 32 Phase 3 — Backend + AI

Pydantic

Pydantic validates data at runtime using Python type hints. It powers FastAPI's request parsing, but it's also the right tool for config management, data transformation, and any boundary where you need guaranteed data shapes.

Plan for the Day

What you'll cover

  • BaseModel — defining schemas with type hints
  • Field() — validation constraints and metadata
  • Nested models and lists of models
  • Custom validators with @field_validator and @model_validator
  • Computed fields with @computed_field
  • Serialization: model_dump(), model_json(), aliases
  • BaseSettings — typed config from env vars

Java parallels

  • BaseModel → Bean Validation (@NotNull, @Size, etc.) on a POJO
  • Field(gt=0) → @Positive, @Min, @Max
  • @field_validator → @AssertTrue / custom ConstraintValidator
  • model_dump() → Jackson ObjectMapper.convertValue(obj, Map.class)
  • model_json() → ObjectMapper.writeValueAsString(obj)
  • BaseSettings → Spring @ConfigurationProperties
  • Field aliases → Jackson @JsonProperty("snake_case")

BaseModel — Defining Schemas

Java — Bean Validation 
public class CreateUserRequest {
    @NotBlank
    @Size(min=2, max=100)
    private String name;

    @NotNull @Email
    private String email;

    @Min(18) @Max(120)
    private Integer age;

    // getters, setters...
}
Python — Pydantic 
from pydantic import BaseModel, Field, EmailStr

# pip install pydantic[email] for EmailStr

class CreateUserRequest(BaseModel):
    name:  str   = Field(min_length=2, max_length=100)
    email: EmailStr                    # validates email format
    age:   int   = Field(ge=18, le=120)

# Pydantic validates and coerces on instantiation
user = CreateUserRequest(name="Alice", email="alice@example.com", age=30)
print(user.name)    # Alice
print(user.email)   # alice@example.com (validated)

# Validation errors are descriptive
try:
    bad = CreateUserRequest(name="A", email="not-an-email", age=15)
except ValueError as e:
    print(e)
# 3 validation errors:
#   name: String should have at least 2 characters
#   email: value is not a valid email address
#   age: Input should be greater than or equal to 18

Field() — validation constraints and metadata

Python 
from pydantic import BaseModel, Field
from decimal import Decimal

class Product(BaseModel):
    name:        str     = Field(min_length=1, max_length=200,
                                  description="Product display name")
    price:       Decimal = Field(gt=0, decimal_places=2,
                                  description="Price in USD, positive")
    sku:         str     = Field(pattern=r"^[A-Z]{2}-\d{4}$",
                                  examples=["AB-1234"],
                                  description="Format: XX-0000")
    quantity:    int     = Field(default=0, ge=0)
    tags:        list[str] = Field(default_factory=list, max_length=10)
    category_id: int | None = Field(default=None, gt=0)

p = Product(name="Widget", price="9.99", sku="WG-0001")
print(p.price)    # Decimal('9.99') — coerced from string
print(p.tags)     # []  — default factory

Nested Models

Python 
from pydantic import BaseModel, EmailStr
from datetime import datetime

class Address(BaseModel):
    street:  str
    city:    str
    country: str = "US"
    zip_code: str = Field(pattern=r"^\d{5}(-\d{4})?$")

class ContactInfo(BaseModel):
    email:   EmailStr
    phone:   str | None = None
    address: Address | None = None

class Order(BaseModel):
    id:          int
    customer:    ContactInfo
    items:       list[dict]
    total:       float
    created_at:  datetime

# Nested models are validated recursively
order = Order(
    id=42,
    customer={
        "email": "alice@example.com",
        "address": {
            "street": "123 Main St",
            "city": "Portland",
            "zip_code": "97201",
        },
    },
    items=[{"sku": "WG-0001", "qty": 2}],
    total=19.98,
    created_at="2024-03-15T10:30:00",   # auto-parsed to datetime
)

print(order.customer.address.city)   # Portland
print(type(order.created_at))        # datetime

Custom Validators

@field_validator — validate a single field

Python 
from pydantic import BaseModel, field_validator, Field
import re

class CreateUserRequest(BaseModel):
    username:  str = Field(min_length=3, max_length=30)
    password:  str = Field(min_length=8)
    email:     str

    @field_validator("username")
    @classmethod
    def username_alphanumeric(cls, v: str) -> str:
        """Username must contain only letters, numbers, underscores."""
        if not re.fullmatch(r"[a-z0-9_]+", v):
            raise ValueError("Username may only contain lowercase letters, numbers, and underscores")
        return v   # return the (possibly transformed) value

    @field_validator("password")
    @classmethod
    def password_strong(cls, v: str) -> str:
        """Must have at least one uppercase, one digit, one special char."""
        if not re.search(r"[A-Z]", v):
            raise ValueError("Password must contain at least one uppercase letter")
        if not re.search(r"\d", v):
            raise ValueError("Password must contain at least one digit")
        return v

    @field_validator("email")
    @classmethod
    def email_normalise(cls, v: str) -> str:
        """Lowercase and strip whitespace from email."""
        return v.strip().lower()

@model_validator — cross-field validation

Python 
from pydantic import BaseModel, model_validator
from datetime import date

class DateRange(BaseModel):
    start_date: date
    end_date:   date
    max_days:   int = 365

    @model_validator(mode="after")
    def end_after_start(self) -> "DateRange":
        """Cross-field: end must be after start, within max_days."""
        if self.end_date <= self.start_date:
            raise ValueError("end_date must be after start_date")
        delta = (self.end_date - self.start_date).days
        if delta > self.max_days:
            raise ValueError(f"Date range exceeds {self.max_days} days (got {delta})")
        return self

class PasswordChange(BaseModel):
    old_password: str
    new_password: str
    confirm:      str

    @model_validator(mode="after")
    def passwords_match(self) -> "PasswordChange":
        if self.new_password != self.confirm:
            raise ValueError("new_password and confirm do not match")
        if self.new_password == self.old_password:
            raise ValueError("new_password must differ from old_password")
        return self

Serialization and Aliases

Python 
from pydantic import BaseModel, Field, computed_field
from datetime import datetime

class UserResponse(BaseModel):
    id:         int
    full_name:  str = Field(serialization_alias="fullName")   # → JSON key "fullName"
    email:      str
    created_at: datetime

    # Computed field — derived from other fields, included in output
    @computed_field
    @property
    def display(self) -> str:
        return f"{self.full_name} <{self.email}>"

    model_config = {"populate_by_name": True}   # accept both field name and alias

user = UserResponse(id=1, full_name="Alice Smith", email="alice@example.com",
                    created_at=datetime.now())

# model_dump — to dict
print(user.model_dump())
# {'id': 1, 'full_name': 'Alice Smith', 'email': ..., 'created_at': ..., 'display': ...}

print(user.model_dump(by_alias=True))
# {'id': 1, 'fullName': 'Alice Smith', ...}   — uses alias

print(user.model_dump(exclude={"created_at"}))
# exclude sensitive or internal fields

# model_json — to JSON string
print(user.model_json(by_alias=True, indent=2))

# model_validate — parse from dict/JSON
raw = {"id": 2, "full_name": "Bob", "email": "bob@ex.com", "created_at": "2024-01-01T00:00:00"}
u2 = UserResponse.model_validate(raw)

# parse from JSON string
u3 = UserResponse.model_validate_json('{"id":3,"full_name":"Carol","email":"c@ex.com","created_at":"2024-01-01T00:00:00"}')

BaseSettings — Typed Config from Environment

BaseSettings is Pydantic's application config solution. Fields are populated from environment variables, .env files, or defaults — in that priority order. It replaces os.environ.get() calls scattered across your codebase with a single validated config object.

Java — @ConfigurationProperties 
@ConfigurationProperties(prefix = "app")
@Validated
public class AppConfig {
    @NotBlank
    private String dbUrl;
    @Min(1) @Max(100)
    private int maxConnections = 10;
    private boolean debug = false;
    // getters/setters...
}
Python — pydantic-settings 
# pip install pydantic-settings
from pydantic_settings import BaseSettings, SettingsConfigDict
from pydantic import Field, PostgresDsn

class Settings(BaseSettings):
    # Reads from env vars: DATABASE_URL, MAX_CONNECTIONS, DEBUG, SECRET_KEY
    database_url:    PostgresDsn = Field(description="Postgres connection string")
    max_connections: int         = Field(default=10, ge=1, le=100)
    debug:           bool        = False
    secret_key:      str         = Field(min_length=32)
    allowed_hosts:   list[str]   = ["localhost"]

    # Read from .env file automatically
    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        case_sensitive=False,   # DATABASE_URL and database_url both work
    )

# .env file:
# DATABASE_URL=postgresql://user:pass@localhost/mydb
# SECRET_KEY=my-very-long-secret-key-that-is-at-least-32-chars
# DEBUG=true

from functools import lru_cache

@lru_cache  # load once, reuse (singleton behaviour)
def get_settings() -> Settings:
    return Settings()

settings = get_settings()
print(settings.database_url)    # PostgresDsn validated
print(settings.max_connections) # int validated
print(settings.debug)           # bool coerced from "true"

Using settings in FastAPI

Python 
from fastapi import FastAPI, Depends
from typing import Annotated
from config import Settings, get_settings

app = FastAPI()
SettingsDep = Annotated[Settings, Depends(get_settings)]

@app.get("/config")
def show_config(settings: SettingsDep):
    # Only expose non-sensitive fields
    return {
        "debug": settings.debug,
        "max_connections": settings.max_connections,
        "allowed_hosts": settings.allowed_hosts,
    }

Quick Reference Card

Basic model
class M(BaseModel): name: str / price: float = 0.0

Validates on instantiation. Raises ValidationError on failure.

Field constraints
Field(gt=0, le=100, min_length=2, pattern=r"...")

gt/ge/lt/le for numbers, min_length/max_length for strings.

Field validator
@field_validator("name") @classmethod def check(cls, v): return v

Must return the (possibly transformed) value. Raise ValueError to fail.

Model validator
@model_validator(mode="after") def check(self): return self

Receives full model — use for cross-field validation.

Computed field
@computed_field @property def full(self) -> str: return f"{self.a} {self.b}"

Derived, read-only. Included in model_dump() output.

Serialize
obj.model_dump() / obj.model_json()

Add by_alias=True, exclude={"field"}, exclude_none=True as needed.

Parse
MyModel.model_validate(dict_or_obj)

Also: model_validate_json(json_str) for direct JSON parsing.

Settings
class S(BaseSettings): db_url: str / model_config = SettingsConfigDict(env_file=".env")

Auto-reads from env vars and .env file. Use @lru_cache for singleton.

Wrap-up

  • Pydantic validates at runtime, not compile time. Instantiating a model with invalid data raises a ValidationError immediately with structured error messages — no manual if/raise chains needed.
  • @field_validator runs on a single field and can both validate and transform the value (normalise email to lowercase, strip whitespace, parse a custom format). Always return the value.
  • @model_validator(mode="after") runs after all fields are validated and gives you access to the full model object — use it for cross-field rules like "end date must be after start date" or "confirm password must match password".
  • model_dump(exclude_none=True) is the idiom for PATCH payloads — only include fields that were actually provided. This pairs perfectly with SQLAlchemy's partial update pattern.
  • BaseSettings is a complete config solution: env vars, .env files, type coercion (strings → booleans, ints, lists), validation, and a typed object. It replaces every os.environ.get() call in your codebase.

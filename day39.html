Multi-turn Conversation

Python 
from openai import OpenAI
from openai.types.chat import ChatCompletionMessageParam

client = OpenAI()

def chat(messages: list[ChatCompletionMessageParam], user_input: str) -> str:
    """Add user message, get response, update history."""
    messages.append({"role": "user", "content": user_input})
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    assistant_message = response.choices[0].message
    messages.append(assistant_message)  # append AIMessage to history
    return assistant_message.content

history = [{"role": "system", "content": "You are a helpful Python tutor."}]
print(chat(history, "What is a generator?"))
print(chat(history, "Can you show me an example?"))   # remembers context

2. Streaming Responses

Streaming returns tokens as they're generated instead of waiting for the full response. This is essential for good UX in chat applications — users see output immediately rather than staring at a spinner for 10 seconds.

Python — basic streaming 
from openai import OpenAI

client = OpenAI()

# stream=True returns an iterator of chunks
stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Explain Python decorators step by step."}],
    stream=True,
)

# Each chunk contains a token (or empty delta at the end)
for chunk in stream:
    delta = chunk.choices[0].delta
    if delta.content:
        print(delta.content, end="", flush=True)
print()  # newline after streaming finishes

Streaming to a FastAPI SSE Endpoint

Python — FastAPI SSE stream 
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from openai import AsyncOpenAI
import asyncio

app = FastAPI()
client = AsyncOpenAI()   # async version for FastAPI

async def generate_stream(prompt: str):
    """Async generator that yields SSE-formatted tokens."""
    stream = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        stream=True,
    )
    async for chunk in stream:
        delta = chunk.choices[0].delta
        if delta.content:
            yield f"data: {delta.content}\n\n"  # SSE format
    yield "data: [DONE]\n\n"

@app.get("/chat/stream")
async def stream_chat(prompt: str):
    return StreamingResponse(
        generate_stream(prompt),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",      # disable nginx buffering
        },
    )

3. Function Calling / Tool Use

Function calling lets the LLM signal that it wants to call a function with specific arguments. You define the function schema; the LLM decides when and how to call it. This is more reliable than asking the LLM to output JSON in its text.

Python 
from openai import OpenAI
import json

client = OpenAI()

# Define the tool schema
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_task_count",
            "description": "Returns the number of tasks for a given status.",
            "parameters": {
                "type": "object",
                "properties": {
                    "status": {
                        "type": "string",
                        "enum": ["pending", "in_progress", "done"],
                        "description": "The task status to count",
                    }
                },
                "required": ["status"],
            },
        },
    }
]

# Real function that does the work
def get_task_count(status: str) -> int:
    # In reality: query your database
    counts = {"pending": 12, "in_progress": 3, "done": 47}
    return counts.get(status, 0)

messages = [{"role": "user", "content": "How many tasks are still pending?"}]

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    tools=tools,
    tool_choice="auto",    # LLM decides whether to call a tool
)

msg = response.choices[0].message

if msg.tool_calls:
    # LLM wants to call a function
    call = msg.tool_calls[0]
    args = json.loads(call.function.arguments)
    result = get_task_count(**args)    # {"status": "pending"} → get_task_count(status="pending")

    # Send result back to the LLM
    messages.append(msg)   # the LLM's tool-call message
    messages.append({
        "role": "tool",
        "tool_call_id": call.id,
        "content": str(result),
    })

    # Get the final answer
    final = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
    )
    print(final.choices[0].message.content)
    # "There are 12 pending tasks."

4. Structured Outputs

Structured outputs guarantee the response is valid JSON matching your schema. Unlike asking the LLM to "respond in JSON", this uses constrained sampling — it's 100% reliable.

Python — response_format with Pydantic 
from openai import OpenAI
from pydantic import BaseModel

client = OpenAI()

class TaskExtraction(BaseModel):
    title: str
    priority: str      # "low", "medium", "high"
    due_date: str | None
    tags: list[str]

# parse() is the Pydantic-aware method — returns a validated model
response = client.beta.chat.completions.parse(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "Extract task information from user input."},
        {"role": "user", "content": "I need to finish the API review by Thursday, it's urgent. Tag it as backend and review."},
    ],
    response_format=TaskExtraction,
)

task = response.choices[0].message.parsed
print(task.title)       # "Finish the API review"
print(task.priority)    # "high"
print(task.due_date)    # "Thursday"
print(task.tags)        # ["backend", "review"]
print(type(task))       #  — a real Pydantic model

5. Token Management

Every API call costs tokens — input + output. Understanding token limits and costs is important for production apps.

Python 
import tiktoken
from openai import OpenAI

client = OpenAI()

# Count tokens before sending (avoid exceeding context limits)
def count_tokens(text: str, model: str = "gpt-4o-mini") -> int:
    encoder = tiktoken.encoding_for_model(model)
    return len(encoder.encode(text))

# GPT-4o-mini context: 128k tokens (~96k words)
# GPT-4o context: 128k tokens
MAX_CONTEXT = 120_000   # leave buffer for output

def truncate_to_fit(text: str, model: str = "gpt-4o-mini") -> str:
    tokens = count_tokens(text, model)
    if tokens <= MAX_CONTEXT:
        return text
    # Truncate and warn
    encoder = tiktoken.encoding_for_model(model)
    encoded = encoder.encode(text)[:MAX_CONTEXT]
    return encoder.decode(encoded)

# Check usage after a call
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
    max_tokens=100,         # limit output tokens
)
usage = response.usage
print(f"Input: {usage.prompt_tokens}")
print(f"Output: {usage.completion_tokens}")
print(f"Total: {usage.total_tokens}")
# gpt-4o-mini: $0.15 per 1M input tokens, $0.60 per 1M output tokens

6. Async OpenAI Client

In FastAPI, use AsyncOpenAI — it's non-blocking and integrates cleanly with async def route handlers. Using the sync client in an async endpoint blocks the event loop.

Python — async FastAPI integration 
from fastapi import FastAPI, Depends
from openai import AsyncOpenAI
from functools import lru_cache
from pydantic import BaseModel

app = FastAPI()

@lru_cache
def get_openai_client() -> AsyncOpenAI:
    return AsyncOpenAI()   # singleton — reuses HTTP connection pool

class ChatRequest(BaseModel):
    message: str
    system_prompt: str = "You are a helpful Python tutor."

class ChatResponse(BaseModel):
    reply: str
    tokens_used: int

@app.post("/chat", response_model=ChatResponse)
async def chat(
    request: ChatRequest,
    client: AsyncOpenAI = Depends(get_openai_client),
):
    response = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": request.system_prompt},
            {"role": "user", "content": request.message},
        ],
        temperature=0,
        max_tokens=500,
    )
    return ChatResponse(
        reply=response.choices[0].message.content,
        tokens_used=response.usage.total_tokens,
    )

Wrap-up

  • The sync OpenAI() client is for scripts; the async AsyncOpenAI() is for FastAPI. Using sync in an async route handler blocks the entire event loop — don't do it.
  • Streaming (stream=True) dramatically improves perceived performance. For long responses, users see output immediately instead of waiting for the full completion.
  • Function calling / tool use is more reliable than asking the LLM to output JSON in its text. Use it whenever you need structured data from the LLM.
  • client.beta.chat.completions.parse() with a Pydantic model gives you 100% guaranteed valid, typed structured output. It's the right choice when you need specific fields from a response.
  • Use tiktoken to count tokens before sending large prompts. Exceeding the context limit gives you an API error; running close to it gives you truncated responses.

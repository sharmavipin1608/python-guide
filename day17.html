Day 17 Phase 2 — Core Libraries

pathlib + os + sys

Modern Python filesystem work uses pathlib.Path — an OOP approach that beats string concatenation entirely. os.environ and sys.argv handle the runtime environment your program runs in.

pathlib.Path

Constructing and navigating paths

pathlib.Path represents filesystem paths as objects. The / operator joins path segments — no string concatenation, no os.path.join() needed. Paths are immutable values.

Java 
Path base = Paths.get("/home/user/projects");
Path src  = base.resolve("myapp").resolve("src");
Path file = src.resolve("main.py");

System.out.println(file.getFileName());  // main.py
System.out.println(file.getParent());    // .../src
System.out.println(file.toString());     // full path
Python 
from pathlib import Path

# Construct a path
base = Path("/home/user/projects")
src  = base / "myapp" / "src"     # / operator joins segments
file = src / "main.py"

print(file.name)       # main.py
print(file.stem)       # main  (name without extension)
print(file.suffix)     # .py
print(file.parent)     # /home/user/projects/myapp/src
print(file.parts)      # ('/', 'home', 'user', 'projects', 'myapp', 'src', 'main.py')

# Special paths
here     = Path(__file__).parent    # directory of the current script
cwd      = Path.cwd()               # current working directory
home     = Path.home()              # /home/user or C:\Users\user

# Resolve — make absolute, resolve symlinks
p = Path("../config.json").resolve()
print(p)  # absolute path

Checking and inspecting paths

Python 
from pathlib import Path

p = Path("config.json")

p.exists()          # True/False — does it exist?
p.is_file()         # True if it's a regular file
p.is_dir()          # True if it's a directory
p.stat().st_size    # file size in bytes
p.stat().st_mtime   # last modified timestamp

# Safe pattern: check before reading
if p.exists() and p.is_file():
    content = p.read_text()
else:
    content = "{}"   # default

Reading and writing

Python 
from pathlib import Path

p = Path("data/output.txt")

# Writing — creates the file if it doesn't exist
p.write_text("Hello, World!\n", encoding="utf-8")

# Reading
content = p.read_text(encoding="utf-8")
print(content)

# Binary read/write
image = Path("photo.jpg").read_bytes()
Path("copy.jpg").write_bytes(image)

# Append — pathlib has no built-in append; use open()
with p.open("a") as f:
    f.write("Appended line\n")

Creating directories and listing files

Python 
from pathlib import Path

# Create directories
Path("logs/2024/jan").mkdir(parents=True, exist_ok=True)
# parents=True  — create all intermediate dirs (like mkdir -p)
# exist_ok=True — don't raise if already exists

# List directory contents
p = Path("src")
for child in p.iterdir():
    print(child.name, "dir" if child.is_dir() else "file")

# glob — find files matching a pattern
py_files = list(Path("src").glob("*.py"))          # direct children
all_py   = list(Path("src").rglob("*.py"))         # recursive
configs  = list(Path(".").glob("**/*.json"))        # same as rglob

# Practical: find all Python files and count lines
total_lines = sum(
    len(f.read_text().splitlines())
    for f in Path("src").rglob("*.py")
    if f.is_file()
)
print(f"Total lines: {total_lines}")

Renaming, moving, and deleting

Python 
from pathlib import Path

p = Path("old_name.txt")
p.write_text("content")

# Rename (move within same filesystem)
new_p = p.rename("new_name.txt")

# Move to different location
import shutil
shutil.move("new_name.txt", "archive/new_name.txt")

# Copy
shutil.copy2("source.txt", "destination.txt")   # preserves metadata

# Delete
Path("temp.txt").unlink(missing_ok=True)   # missing_ok: no error if absent
Path("empty_dir").rmdir()                  # only works on empty dirs
shutil.rmtree("full_dir")                  # remove tree recursively

os.environ — Environment Variables

Environment variables are the standard way to configure applications without hardcoding values. Python exposes them through os.environ, a dict-like object.

Java 
String host = System.getenv("DB_HOST");
String port = System.getenv().getOrDefault("DB_PORT", "5432");

if (host == null) {
    throw new IllegalStateException("DB_HOST not set");
}
Python 
import os

# Read — returns None if missing (like getenv in C)
host = os.environ.get("DB_HOST")

# Read with default
port = os.environ.get("DB_PORT", "5432")

# Read required — raises KeyError if missing
secret = os.environ["SECRET_KEY"]   # KeyError if not set

# Set (only affects current process, not parent shell)
os.environ["MY_VAR"] = "value"

# Check all env vars
for key, value in os.environ.items():
    print(f"{key}={value}")

# Delete
del os.environ["MY_VAR"]
os.environ.pop("MY_VAR", None)   # safe delete with default
Best practice: Use python-dotenv (Day 34) to load a .env file into os.environ for local development. For production, set env vars in your deployment platform (Docker, Kubernetes, cloud provider). Never hardcode secrets in source code.

sys — Runtime System Interface

sys.argv — command-line arguments

sys.argv is a list of strings: sys.argv[0] is always the script name, and subsequent elements are the arguments passed. For complex CLIs, use argparse or click (Day 47) instead of parsing sys.argv manually.

Python 
import sys

# Running: python script.py input.txt --verbose 42
# sys.argv == ['script.py', 'input.txt', '--verbose', '42']

script_name = sys.argv[0]       # 'script.py'
args        = sys.argv[1:]      # ['input.txt', '--verbose', '42']

if len(sys.argv) < 2:
    print(f"Usage: {script_name} ")
    sys.exit(1)   # non-zero exit code = error

input_file = sys.argv[1]
verbose    = "--verbose" in sys.argv

# Simple manual parsing
def parse_args() -> dict:
    args = sys.argv[1:]
    result = {"files": [], "verbose": False}
    i = 0
    while i < len(args):
        if args[i] == "--verbose":
            result["verbose"] = True
        else:
            result["files"].append(args[i])
        i += 1
    return result

sys.exit, sys.path, sys.stdin/stdout

Python 
import sys

# sys.exit — exit with a status code
# 0 = success, non-zero = error (convention)
sys.exit(0)     # clean exit
sys.exit(1)     # error exit
sys.exit("Error: file not found")  # string → printed to stderr, exit 1

# sys.path — where Python looks for modules
print(sys.path)
sys.path.insert(0, "/path/to/my/lib")   # add custom module location

# sys.stdin / stdout / stderr — file objects for standard streams
# Write to stderr (errors/warnings shouldn't pollute stdout)
print("Error: something went wrong", file=sys.stderr)
sys.stderr.write("Warning: deprecated\n")

# Read from stdin (e.g. piped input)
for line in sys.stdin:
    print(line.upper(), end="")

# Platform info
print(sys.platform)   # 'linux', 'darwin', 'win32'
print(sys.version)    # '3.12.0 (main, ...)'

os.path — legacy path operations

You'll see os.path in older code. Know what it does; prefer pathlib in new code.

Python 
import os.path

# os.path equivalents (all superseded by pathlib)
os.path.join("/home/user", "projects", "app.py")   # Path("/home/user") / "projects" / "app.py"
os.path.exists("config.json")                       # Path("config.json").exists()
os.path.basename("/home/user/file.txt")             # Path(...).name
os.path.dirname("/home/user/file.txt")              # Path(...).parent
os.path.splitext("file.txt")                        # Path(...).stem, Path(...).suffix
os.path.isfile("file.txt")                          # Path(...).is_file()
os.path.isdir("dir")                                # Path(...).is_dir()
os.path.abspath("../config")                        # Path("../config").resolve()

Quick Reference Card

Construct path
Path("/base") / "sub" / "file.txt"

Use / to join segments. Works on all platforms.

Path components
p.name / p.stem / p.suffix / p.parent

name=filename, stem=no ext, suffix=ext, parent=dir

Read / write text
p.read_text() / p.write_text("...")

One-liner file I/O. Add encoding="utf-8" explicitly.

Create dirs
Path("a/b/c").mkdir(parents=True, exist_ok=True)

Equivalent to mkdir -p. Safe to call repeatedly.

Find files
Path(".").glob("**/*.py")

Recursive glob. Returns a lazy iterator of Path objects.

Env var with default
os.environ.get("KEY", "default")

Never raises. Use os.environ["KEY"] when the var is required.

CLI args
sys.argv[1:]

sys.argv[0] is the script name. Rest are the user's arguments.

Error exit
sys.exit(1)

0 = success, non-zero = error. Checked by shells and CI systems.

Wrap-up

  • pathlib.Path is the modern way to work with files. Use the / operator to join segments instead of os.path.join(). Use read_text()/write_text() for simple file I/O instead of open()/read()/close().
  • Always use Path(__file__).parent to get the directory of the current script — never assume the working directory is where your script lives.
  • os.environ.get("KEY", "default") for optional config, os.environ["KEY"] for required config that should fail loudly if missing. Never hardcode secrets in code.
  • sys.argv[1:] for simple scripts. For anything with more than two arguments, use argparse (stdlib) or click (third-party, Day 47) — manual parsing of sys.argv doesn't scale.
  • Write errors to sys.stderr and set a non-zero exit code with sys.exit(1). This makes your script composable in shell pipelines and CI systems — the same contract as any Unix command.

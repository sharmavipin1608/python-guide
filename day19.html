Day 19 Phase 2 — Core Libraries

requests + httpx

Making HTTP requests in Python is one line. requests is the de-facto standard for synchronous HTTP. httpx is the modern drop-in that adds async support with the same clean API.

Plan for the Day

What you'll cover

  • requests.get / post / put / delete — the four verbs
  • Query params, headers, and JSON request bodies
  • Response: status codes, .json(), .text, .content
  • Error handling with raise_for_status()
  • Sessions — connection reuse, base headers, auth
  • httpx — same API, sync + async, HTTP/2 support

Java parallels

  • requests.get(url) → RestTemplate.getForObject(url, String.class)
  • requests.Session → RestTemplate instance (reused)
  • response.json() → ObjectMapper.readValue(body, Map.class)
  • raise_for_status() → HttpClientErrorException / HttpServerErrorException
  • httpx.AsyncClient → Spring's WebClient (reactive/async)

requests — Synchronous HTTP

Installation and basic GET

requests is a third-party library — install it with pip install requests. It's not in the standard library but is effectively universal in the Python ecosystem.

Java — RestTemplate 
RestTemplate rt = new RestTemplate();
String body = rt.getForObject(
    "https://api.github.com/users/torvalds",
    String.class
);
// Must parse JSON separately
Python 
import requests

response = requests.get("https://api.github.com/users/torvalds")

print(response.status_code)   # 200
print(response.headers["Content-Type"])  # application/json; charset=utf-8

# Parse JSON body directly
user = response.json()        # returns a dict
print(user["name"])           # Linus Torvalds
print(user["public_repos"])   # number of public repos

# Raw text and bytes
print(response.text)          # JSON as string
print(len(response.content))  # bytes

Query parameters, headers, and timeouts

Python 
import requests

# Query params — requests handles URL encoding
params = {"q": "python requests", "sort": "stars", "per_page": 5}
response = requests.get(
    "https://api.github.com/search/repositories",
    params=params,
    headers={"Accept": "application/vnd.github.v3+json"},
    timeout=10,   # seconds — ALWAYS set a timeout
)
# Actual URL: ...?q=python+requests&sort=stars&per_page=5

data = response.json()
for repo in data["items"]:
    print(f"{repo['full_name']}: ⭐{repo['stargazers_count']}")

POST, PUT, DELETE with JSON body

Python 
import requests

BASE = "https://jsonplaceholder.typicode.com"

# POST with JSON body — json= kwarg sets Content-Type automatically
new_post = {"title": "Hello", "body": "World", "userId": 1}
resp = requests.post(f"{BASE}/posts", json=new_post)
print(resp.status_code)   # 201
print(resp.json())        # {"id": 101, "title": "Hello", ...}

# PUT — replace a resource
updated = {"id": 1, "title": "Updated", "body": "New body", "userId": 1}
resp = requests.put(f"{BASE}/posts/1", json=updated)
print(resp.status_code)   # 200

# PATCH — partial update
resp = requests.patch(f"{BASE}/posts/1", json={"title": "Patched"})
print(resp.json())

# DELETE
resp = requests.delete(f"{BASE}/posts/1")
print(resp.status_code)   # 200

Error handling with raise_for_status()

requests does NOT raise on HTTP error codes by default — a 404 or 500 is just a response with a bad status. Call raise_for_status() to turn 4xx/5xx responses into exceptions.

Python 
import requests
from requests.exceptions import HTTPError, ConnectionError, Timeout

def fetch_user(username: str) -> dict:
    try:
        response = requests.get(
            f"https://api.github.com/users/{username}",
            timeout=10,
        )
        response.raise_for_status()   # raises HTTPError for 4xx/5xx
        return response.json()

    except HTTPError as e:
        if e.response.status_code == 404:
            raise ValueError(f"User '{username}' not found") from e
        raise   # re-raise unexpected HTTP errors

    except Timeout:
        raise TimeoutError(f"GitHub API timed out for user '{username}'")

    except ConnectionError:
        raise RuntimeError("Cannot connect to GitHub API")


# Clean call site
try:
    user = fetch_user("torvalds")
    print(user["name"])
except ValueError as e:
    print(f"Not found: {e}")

Sessions — connection reuse and base config

A Session reuses the underlying TCP connection for multiple requests to the same host (like Java's RestTemplate with a connection pool). It also lets you set base headers and auth once.

Python 
import requests
import os

TOKEN = os.environ.get("GITHUB_TOKEN", "")

# Session persists headers and auth across all requests
with requests.Session() as session:
    session.headers.update({
        "Authorization": f"Bearer {TOKEN}",
        "Accept": "application/vnd.github.v3+json",
        "User-Agent": "my-github-client/1.0",
    })

    # All requests through this session share headers
    user   = session.get("https://api.github.com/user").json()
    repos  = session.get("https://api.github.com/user/repos").json()
    issues = session.get(
        "https://api.github.com/repos/torvalds/linux/issues",
        params={"state": "open", "per_page": 10},
    ).json()

    print(f"Logged in as: {user['login']}")
    print(f"Your repos: {len(repos)}")
    print(f"Linux open issues (first 10): {len(issues)}")

httpx — Modern Drop-in Replacement

Sync API — identical to requests

httpx is a near-drop-in replacement for requests with the same API surface, but adds async support, HTTP/2, and better timeout handling. Install: pip install httpx.

Python 
import httpx

# Sync usage — identical to requests
with httpx.Client(
    base_url="https://api.github.com",
    headers={"Accept": "application/vnd.github.v3+json"},
    timeout=10.0,
) as client:
    resp = client.get("/users/torvalds")
    resp.raise_for_status()
    user = resp.json()
    print(user["name"])

    repos = client.get("/users/torvalds/repos", params={"per_page": 5})
    for r in repos.json():
        print(f"  {r['name']}: {r['stargazers_count']}⭐")

Async client — true non-blocking HTTP

The async client uses the same API but with async with and await. This lets you make dozens of HTTP calls concurrently without threads. (Full async coverage on Day 22-23.)

Python 
import asyncio
import httpx

async def fetch_users(usernames: list[str]) -> list[dict]:
    """Fetch multiple GitHub users concurrently."""
    async with httpx.AsyncClient(
        base_url="https://api.github.com",
        timeout=10.0,
    ) as client:
        tasks = [
            client.get(f"/users/{name}")
            for name in usernames
        ]
        responses = await asyncio.gather(*tasks)
        return [r.json() for r in responses if r.status_code == 200]


async def main():
    names = ["torvalds", "gvanrossum", "kennethreitz"]
    users = await fetch_users(names)
    for u in users:
        print(f"{u['login']}: {u['public_repos']} repos")

asyncio.run(main())
When to use which: Use requests for simple scripts and synchronous code — it's battle-tested and universally known. Use httpx when you need async (FastAPI test clients, async scraping) or HTTP/2. For new projects, httpx is the better default.

Authentication patterns

Python 
import requests

# Basic auth
resp = requests.get("https://api.example.com/data",
                    auth=("username", "password"))

# Bearer token (most REST APIs)
resp = requests.get(
    "https://api.example.com/data",
    headers={"Authorization": f"Bearer {token}"},
)

# API key as query param
resp = requests.get(
    "https://api.example.com/data",
    params={"api_key": api_key},
)

# Custom auth class (reusable)
class BearerAuth(requests.auth.AuthBase):
    def __init__(self, token: str):
        self.token = token
    def __call__(self, r):
        r.headers["Authorization"] = f"Bearer {self.token}"
        return r

resp = requests.get("https://api.example.com/data",
                    auth=BearerAuth(token))

Quick Reference Card

GET request
requests.get(url, params={}, headers={}, timeout=10)

Always set timeout — no default. Returns a Response object.

POST with JSON
requests.post(url, json=data)

json= serializes dict and sets Content-Type automatically.

Check for errors
response.raise_for_status()

Raises HTTPError for 4xx/5xx. Call before .json().

Parse response
response.json() / .text / .content / .status_code

.json() = parsed dict, .text = string, .content = bytes.

Session
with requests.Session() as s: s.headers.update({...})

Reuses connection, shares headers/auth across requests.

httpx sync
with httpx.Client(base_url=...) as c: c.get("/path")

Drop-in for requests. Supports base_url natively.

httpx async
async with httpx.AsyncClient() as c: await c.get(url)

Non-blocking HTTP. Use with asyncio.gather() for concurrent requests.

Error types
HTTPError / Timeout / ConnectionError

All from requests.exceptions. Always catch separately for correct handling.

Wrap-up

  • requests does not raise on bad HTTP status codes — it just gives you the response. Always call response.raise_for_status() before calling .json(), or you'll silently process error responses as if they were data.
  • Always set timeout. Without it, a hung server will hang your program indefinitely. Use 10 seconds for interactive scripts and adjust based on expected response times.
  • Use a Session (or httpx.Client) when making multiple requests to the same host — it reuses the TCP connection and lets you set base headers and auth once instead of on every call.
  • Pass json=data for POST bodies, not data=json.dumps(data). The json= kwarg serializes the dict and sets Content-Type: application/json automatically.
  • httpx is the better choice for new projects — same API as requests, async-capable, and HTTP/2 ready. The Day 21 project and Day 23 async scraper use it.

Day 15 Phase 2 — Core Libraries

collections module

Python's collections module gives you specialised container types that replace verbose Java patterns with single expressive lines — counting, grouping, queuing, and naming all become trivial.

Plan for the Day

What you'll cover

  • Counter — count occurrences in one call, replaces HashMap counting loops
  • defaultdict — dict with automatic default values, no more KeyError guards
  • OrderedDict — when insertion order needs to be explicit
  • deque — double-ended queue with O(1) append/pop on both ends
  • namedtuple — lightweight immutable data object with named fields

Java parallels

  • Counter → HashMap<T, Integer> with manual getOrDefault(k, 0) + 1
  • defaultdict → computeIfAbsent / getOrDefault
  • OrderedDict → LinkedHashMap
  • deque → ArrayDeque
  • namedtuple → Java Record (immutable) or a minimal POJO

Counter

Counting made trivial

Counter counts hashable elements. Pass any iterable and get a dict-like object mapping elements to counts. It supports arithmetic operations and the most-common query that would take 10+ lines in Java.

Java 
String[] words = text.split(" ");
Map<String, Integer> counts = new HashMap<>();
for (String w : words) {
    counts.put(w, counts.getOrDefault(w, 0) + 1);
}
// Finding top 3 requires sorting entries...
Python 
from collections import Counter

text = "the cat sat on the mat the cat"
counts = Counter(text.split())
# Counter({'the': 3, 'cat': 2, 'sat': 1, 'on': 1, 'mat': 1})

print(counts["the"])         # 3
print(counts["missing"])     # 0  — no KeyError!
print(counts.most_common(3)) # [('the', 3), ('cat', 2), ('sat', 1)]

# Count characters
char_count = Counter("mississippi")
print(char_count.most_common(3))  # [('s', 4), ('i', 4), ('p', 2)]

# Arithmetic on counters
a = Counter(["a", "b", "a", "c"])
b = Counter(["a", "b", "b", "d"])
print(a + b)   # Counter({'a': 3, 'b': 3, 'c': 1, 'd': 1})
print(a - b)   # Counter({'a': 1, 'c': 1})  — removes zero/neg
print(a & b)   # Counter({'a': 1, 'b': 1})  — min of each
print(a | b)   # Counter({'b': 2, 'a': 2, 'c': 1, 'd': 1})  — max

Practical: word frequency analysis

Python 
import re
from collections import Counter
from pathlib import Path

def word_frequency(filepath: str, top_n: int = 10) -> list[tuple[str, int]]:
    text = Path(filepath).read_text().lower()
    words = re.findall(r"\b[a-z]+\b", text)
    stopwords = {"the", "a", "an", "and", "or", "in", "of", "to", "is"}
    filtered = [w for w in words if w not in stopwords]
    return Counter(filtered).most_common(top_n)

# Returns: [('python', 42), ('class', 31), ...]

defaultdict

No more KeyError guards

A defaultdict automatically creates a value for missing keys using a factory function. The factory is any callable — list, int, set, or a lambda. This eliminates the defensive setdefault or if key not in d pattern.

Java 
Map<String, List<String>> groups = new HashMap<>();
for (String word : words) {
    String key = String.valueOf(word.charAt(0));
    groups.computeIfAbsent(key, k -> new ArrayList<>())
          .add(word);
}
Python 
from collections import defaultdict

# Group words by first letter
words = ["apple", "avocado", "banana", "blueberry", "cherry"]
groups: defaultdict[str, list[str]] = defaultdict(list)

for word in words:
    groups[word[0]].append(word)   # no KeyError, list auto-created

print(dict(groups))
# {'a': ['apple', 'avocado'], 'b': ['banana', 'blueberry'], 'c': ['cherry']}

# defaultdict(int) — counting without Counter
word_count: defaultdict[str, int] = defaultdict(int)
for word in words:
    word_count[word[0]] += 1   # missing keys start at 0

# defaultdict(set) — grouping unique values
seen: defaultdict[str, set] = defaultdict(set)
seen["errors"].add("timeout")
seen["errors"].add("timeout")   # duplicate — set deduplicates
print(len(seen["errors"]))      # 1
Note: defaultdict behaves like a regular dict in every other way — iteration, len(), .items() all work identically. The only difference is what happens on a missing key access.

deque

O(1) both ends

deque (double-ended queue) is the right structure when you need to append or pop from both ends efficiently. Python's list.pop(0) is O(n) — it shifts every element. deque.popleft() is O(1).

Java 
Deque<String> deque = new ArrayDeque<>();
deque.addFirst("front");
deque.addLast("back");
String first = deque.pollFirst();
String last  = deque.pollLast();
Python 
from collections import deque

dq = deque([1, 2, 3])
dq.appendleft(0)   # [0, 1, 2, 3]  — O(1)
dq.append(4)       # [0, 1, 2, 3, 4] — O(1)
dq.popleft()       # returns 0, O(1)
dq.pop()           # returns 4, O(1)

# Bounded deque — automatically discards oldest when full
recent = deque(maxlen=3)
for x in range(6):
    recent.append(x)
print(list(recent))  # [3, 4, 5] — only last 3 kept

# Rotate — move items from one end to the other
dq = deque([1, 2, 3, 4, 5])
dq.rotate(2)    # [4, 5, 1, 2, 3]
dq.rotate(-1)   # [5, 1, 2, 3, 4]

Real use: sliding window + BFS queue

Python 
from collections import deque

def sliding_window_max(nums: list[int], k: int) -> list[int]:
    """Find max in each sliding window of size k."""
    dq: deque[int] = deque()   # stores indices
    result = []
    for i, n in enumerate(nums):
        while dq and nums[dq[-1]] < n:
            dq.pop()
        dq.append(i)
        if dq[0] == i - k:
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result

print(sliding_window_max([1,3,-1,-3,5,3,6,7], 3))
# [3, 3, 5, 5, 6, 7]

namedtuple

Lightweight immutable records

namedtuple creates a tuple subclass with named fields. You get tuple immutability and unpacking, plus dot-access by name. It's lighter than a dataclass — no methods, no mutable state, just structured data.

Java — record 
public record Point(int x, int y) {}
public record Color(int r, int g, int b) {}

Point p = new Point(3, 4);
System.out.println(p.x());  // 3
// Can unpack? No.
Python 
from collections import namedtuple
from typing import NamedTuple

# Old style
Point = namedtuple("Point", ["x", "y"])

# Modern typed style (preferred)
class Point(NamedTuple):
    x: int
    y: int

class Color(NamedTuple):
    r: int
    g: int
    b: int
    alpha: float = 1.0   # default value

p = Point(3, 4)
print(p.x)       # 3 — named access
print(p[0])      # 3 — index access (it's still a tuple)
x, y = p         # tuple unpacking works

c = Color(255, 0, 0)
print(c)         # Color(r=255, g=0, b=0, alpha=1.0)

# Named tuples are hashable — can be dict keys or set members
palette = {Color(255, 0, 0), Color(0, 255, 0)}

# _replace — create a modified copy (immutable, so no in-place)
p2 = p._replace(y=10)
print(p2)   # Point(x=3, y=10)
print(p)    # Point(x=3, y=4)  — original unchanged

Wrap-up

  • Counter replaces every "count occurrences in a loop and store in a HashMap" pattern with a single call. Always reach for it when you need frequency analysis, voting tallies, or histogram data.
  • defaultdict eliminates defensive key-existence checks. Pass list, int, set, or any zero-argument callable as the factory. It's a direct replacement for computeIfAbsent in Java.
  • deque is the correct data structure for queues, sliding windows, and anything where you need O(1) operations at both ends. Never use list.pop(0) in a tight loop — it's O(n).
  • NamedTuple (typed class syntax) is the idiomatic choice for immutable value objects without methods. Lighter than @dataclass, supports tuple unpacking, and is hashable out of the box.
  • All these types live in the standard library — no installation required. Prefer them over reimplementing the same patterns with plain dict and list.

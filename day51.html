CI/CD for Python

A complete GitHub Actions pipeline: ruff linting, mypy type checking, pytest with coverage, and Docker image build + push on merge. The same pipeline pattern Java teams use with Maven/Gradle, adapted for Python.

Plan for the Day

What You'll Cover

  • GitHub Actions workflow structure ~10 min
  • Lint with ruff, type-check with mypy ~15 min
  • Run pytest with coverage reporting ~15 min
  • Build and push Docker image on merge ~15 min
  • Environment secrets and matrix builds ~10 min

Java Parallels

  • GitHub Actions workflow → Jenkinsfile or .github/workflows/*.yml in a Maven/Gradle project — same concept
  • ruff → Checkstyle / PMD — style and lint enforcement
  • mypy → javac type checking — catches type errors before runtime
  • pytest --cov → JaCoCo coverage in Maven Surefire reports
  • docker/build-push-action → same Docker build step used in Java CI pipelines

1. Full CI Workflow

This single workflow file runs on every push and pull request. It catches lint errors, type errors, and test failures before anything merges.

.github/workflows/ci.yml 
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    name: Lint + Type Check + Test
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12"]   # test on multiple versions

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: pip              # cache pip downloads between runs

      - name: Install dependencies
        run: pip install -e ".[dev]"

      - name: Lint with ruff
        run: |
          ruff check src/ tests/
          ruff format --check src/ tests/   # also check formatting

      - name: Type check with mypy
        run: mypy src/ --ignore-missing-imports

      - name: Run tests with coverage
        run: |
          pytest tests/ \
            --cov=src \
            --cov-report=xml \
            --cov-report=term-missing \
            --cov-fail-under=80

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          file: coverage.xml
          fail_ci_if_error: false   # don't fail CI if Codecov is down

2. Docker Build + Push on Merge

A separate job that only runs when code lands on main. It builds the Docker image, tags it with the commit SHA and latest, and pushes to GitHub Container Registry (ghcr.io) — free for public repos.

.github/workflows/docker.yml 
name: Docker Build + Push

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}   # owner/repo-name

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write    # needed to push to ghcr.io

    steps:
      - uses: actions/checkout@v4

      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}   # auto-provided, no setup needed

      - name: Extract metadata (tags, labels)
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=sha-          # ghcr.io/owner/repo:sha-abc1234
            type=raw,value=latest         # ghcr.io/owner/repo:latest

      - name: Build and push Docker image
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha            # GitHub Actions cache for Docker layers
          cache-to: type=gha,mode=max

3. Environment Secrets

Tests that hit external services (OpenAI, a real DB) need secrets. Never hardcode them — use GitHub Secrets, then reference them in the workflow as environment variables.

.github/workflows/ci.yml — with secrets 
    - name: Run tests with coverage
      env:
        OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        DATABASE_URL: ${{ secrets.TEST_DATABASE_URL }}
        ENVIRONMENT: testing
      run: pytest tests/ --cov=src --cov-report=xml
Skip expensive tests in CI: Mark tests that call real external APIs with @pytest.mark.slow and only run them on schedule or manually — not on every push.
Python — conditional test skipping 
import os
import pytest

# Skip if the API key isn't set (e.g. in forks without secrets)
skip_without_openai = pytest.mark.skipif(
    not os.getenv("OPENAI_API_KEY"),
    reason="OPENAI_API_KEY not set",
)

@skip_without_openai
def test_llm_response():
    from openai import OpenAI
    client = OpenAI()
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": "Say 'ok'"}],
    )
    assert "ok" in response.choices[0].message.content.lower()

4. ruff + mypy Configuration

pyproject.toml — ruff + mypy 
[tool.ruff]
line-length = 100
target-version = "py311"

[tool.ruff.lint]
select = [
    "E",    # pycodestyle errors
    "F",    # pyflakes (unused imports, undefined names)
    "I",    # isort (import ordering)
    "UP",   # pyupgrade (use modern Python syntax)
    "B",    # flake8-bugbear (common bugs)
    "SIM",  # flake8-simplify (simplifiable code)
]
ignore = ["E501"]   # line length handled by formatter

[tool.ruff.lint.isort]
known-first-party = ["mypackage"]

[tool.mypy]
python_version = "3.11"
strict = false          # start with false, enable gradually
ignore_missing_imports = true
check_untyped_defs = true
warn_unused_ignores = true
$ # Fix all auto-fixable issues at once
$ ruff check --fix src/ tests/
$ ruff format src/ tests/

$ # Run mypy
$ mypy src/ --ignore-missing-imports

5. Pre-commit Hooks

Pre-commit hooks run ruff and mypy locally before every commit — catching issues before they even reach CI. Much faster feedback loop than waiting for GitHub Actions.

$ pip install pre-commit
$ pre-commit install
.pre-commit-config.yaml 
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.4.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.10.0
    hooks:
      - id: mypy
        additional_dependencies: [pydantic, sqlalchemy]

  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.6.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-merge-conflict

Quick Reference

Trigger on PR
on: pull_request: branches: [main]

Runs CI on every PR targeting main

Matrix builds
strategy: matrix: python-version: ["3.11","3.12"]

Tests against multiple Python versions in parallel

Pip cache
uses: actions/setup-python@v5 with: cache: pip

Caches pip downloads — significantly speeds up CI

Lint + format check
ruff check src/ && ruff format --check src/

--check exits non-zero if formatting differs

Coverage threshold
pytest --cov-fail-under=80

CI fails if coverage drops below threshold

GitHub secret
env: MY_SECRET: ${{ secrets.MY_SECRET }}

Add secrets in repo Settings → Secrets → Actions

Docker layer cache
cache-from: type=gha

Reuses unchanged layers from previous builds — faster CI

Skip test conditionally
@pytest.mark.skipif(not os.getenv("KEY"), reason="...")

Skip tests that need secrets not available in forks

Wrap-up

  • Structure CI as two separate workflows: ci.yml runs on every push/PR (fast — lint, test, type-check); docker.yml runs only on merge to main (slower — build and push image).
  • Cache pip downloads with cache: pip in setup-python. Without it, every CI run reinstalls all packages — typically 30–60 seconds of wasted time per run.
  • ruff check --fix auto-fixes the majority of lint issues. Run it locally before pushing to avoid back-and-forth CI failures on style issues.
  • Never hardcode secrets in workflow files. Use GitHub Secrets and reference them with ${{ secrets.MY_SECRET }}. Secrets are masked in logs automatically.
  • Pre-commit hooks catch issues in seconds locally; GitHub Actions catches them in minutes in CI. Use both — local hooks for fast feedback, CI as the authoritative gate.

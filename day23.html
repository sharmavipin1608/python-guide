asyncio.Queue — Producer/Consumer

An asyncio.Queue is thread-safe within an async context and integrates with the event loop — producers await queue.put() and consumers await queue.get(). When the queue is full, producers suspend. When empty, consumers suspend.

Java 
BlockingQueue<String> q = new LinkedBlockingQueue<>(10);

// Producer thread
q.put("item");

// Consumer thread
String item = q.take();
Python 
import asyncio

async def producer(queue: asyncio.Queue, items: list) -> None:
    for item in items:
        await queue.put(item)
        print(f"Produced: {item}")
        await asyncio.sleep(0.1)
    await queue.put(None)   # sentinel to signal done

async def consumer(queue: asyncio.Queue, name: str) -> None:
    while True:
        item = await queue.get()
        if item is None:
            queue.task_done()
            break
        print(f"[{name}] consumed: {item}")
        await asyncio.sleep(0.2)   # simulate processing
        queue.task_done()

async def main():
    queue: asyncio.Queue[str | None] = asyncio.Queue(maxsize=5)
    items = [f"task-{i}" for i in range(10)]

    # Start producer and multiple consumers
    await asyncio.gather(
        producer(queue, items),
        consumer(queue, "worker-1"),
        consumer(queue, "worker-2"),
    )
    print("All done!")

asyncio.run(main())

Multiple consumers, one queue

Python 
import asyncio
import httpx

async def scrape_worker(
    name: str,
    queue: asyncio.Queue,
    client: httpx.AsyncClient,
    results: list,
) -> None:
    while True:
        url = await queue.get()
        if url is None:
            queue.task_done()
            break
        try:
            resp = await client.get(url, timeout=5.0)
            results.append({"url": url, "status": resp.status_code,
                            "size": len(resp.content)})
            print(f"[{name}] {url} → {resp.status_code}")
        except Exception as e:
            results.append({"url": url, "error": str(e)})
        finally:
            queue.task_done()

async def main():
    urls = [f"https://httpbin.org/get?n={i}" for i in range(20)]
    n_workers = 5
    queue: asyncio.Queue = asyncio.Queue()
    results = []

    for url in urls:
        await queue.put(url)
    for _ in range(n_workers):
        await queue.put(None)   # one sentinel per worker

    async with httpx.AsyncClient() as client:
        workers = [
            asyncio.create_task(scrape_worker(f"w{i}", queue, client, results))
            for i in range(n_workers)
        ]
        await asyncio.gather(*workers)

    print(f"Scraped {len(results)} URLs")

asyncio.run(main())

asyncio.Semaphore — Rate Limiting

A Semaphore limits how many coroutines can execute a section simultaneously. Use it to cap concurrent HTTP connections without restructuring your code into a queue.

Python 
import asyncio
import httpx

async def fetch_with_limit(
    sem: asyncio.Semaphore,
    client: httpx.AsyncClient,
    url: str,
) -> dict:
    async with sem:   # blocks if semaphore count is 0
        resp = await client.get(url, timeout=10.0)
        return {"url": url, "status": resp.status_code}

async def main():
    urls = [f"https://httpbin.org/get?n={i}" for i in range(50)]

    sem = asyncio.Semaphore(10)   # max 10 concurrent requests

    async with httpx.AsyncClient() as client:
        results = await asyncio.gather(
            *[fetch_with_limit(sem, client, url) for url in urls],
            return_exceptions=True,
        )

    ok = [r for r in results if isinstance(r, dict)]
    print(f"Completed: {len(ok)}/{len(urls)}")

asyncio.run(main())

Timeouts

asyncio.timeout (Python 3.11+)

Python 
import asyncio

async def slow_operation() -> str:
    await asyncio.sleep(5)
    return "done"

async def main():
    # asyncio.timeout — Python 3.11+
    try:
        async with asyncio.timeout(2.0):   # 2-second deadline
            result = await slow_operation()
    except TimeoutError:
        print("Operation timed out!")

    # asyncio.wait_for — works in Python 3.8+
    try:
        result = await asyncio.wait_for(slow_operation(), timeout=2.0)
    except asyncio.TimeoutError:
        print("wait_for timed out!")

asyncio.run(main())

Per-task timeout pattern

Python 
import asyncio
import httpx

async def fetch_with_timeout(
    client: httpx.AsyncClient,
    url: str,
    timeout: float = 3.0,
) -> dict | None:
    try:
        async with asyncio.timeout(timeout):
            resp = await client.get(url)
            return {"url": url, "status": resp.status_code}
    except (TimeoutError, httpx.RequestError) as e:
        print(f"Failed {url}: {type(e).__name__}")
        return None

async def main():
    urls = [
        "https://httpbin.org/delay/1",   # fast
        "https://httpbin.org/delay/5",   # will time out
        "https://httpbin.org/delay/2",   # borderline
    ]
    async with httpx.AsyncClient() as client:
        results = await asyncio.gather(
            *[fetch_with_timeout(client, url) for url in urls]
        )
    successful = [r for r in results if r is not None]
    print(f"{len(successful)}/{len(urls)} succeeded")

asyncio.run(main())

run_in_executor — Blocking Code in Async

Sometimes you must call a blocking library (a sync database driver, a legacy SDK) from async code. loop.run_in_executor() runs it in a thread pool without blocking the event loop.

Python 
import asyncio
import time
from concurrent.futures import ThreadPoolExecutor

def blocking_db_query(query: str) -> list[dict]:
    """Simulates a synchronous DB call that takes time."""
    time.sleep(1)   # blocks — we can't make this async
    return [{"id": 1, "data": query}]

async def async_db_query(query: str) -> list[dict]:
    """Wraps the blocking call so the event loop isn't blocked."""
    loop = asyncio.get_running_loop()
    # Run in a thread pool — event loop stays free
    result = await loop.run_in_executor(None, blocking_db_query, query)
    return result

async def main():
    # All three "queries" run concurrently via thread pool
    results = await asyncio.gather(
        async_db_query("SELECT * FROM users"),
        async_db_query("SELECT * FROM products"),
        async_db_query("SELECT * FROM orders"),
    )
    print(f"Got {len(results)} result sets")

asyncio.run(main())
When to use run_in_executor: Whenever you must call a synchronous blocking function from async code — legacy SDKs, requests (vs httpx), CPU-heavy computation, or any time.sleep()-style blocking call. The thread pool absorbs the blocking; your event loop stays responsive.

Practical: Async Web Scraper

Python 
"""
Async web scraper: fetch multiple pages concurrently,
extract titles, save results. Rate-limited with Semaphore.
"""
import asyncio
import re
from dataclasses import dataclass, field
from datetime import datetime, timezone

import httpx

CONCURRENCY = 5   # max simultaneous requests

@dataclass
class ScrapeResult:
    url: str
    status: int = 0
    title: str = ""
    error: str = ""
    fetched_at: datetime = field(default_factory=lambda: datetime.now(timezone.utc))

def extract_title(html: str) -> str:
    match = re.search(r"<title>(.+?)</title>", html, re.IGNORECASE | re.DOTALL)
    return match.group(1).strip() if match else "No title"

async def scrape_one(
    sem: asyncio.Semaphore,
    client: httpx.AsyncClient,
    url: str,
) -> ScrapeResult:
    async with sem:
        try:
            async with asyncio.timeout(10.0):
                resp = await client.get(url, follow_redirects=True)
                title = extract_title(resp.text)
                return ScrapeResult(url=url, status=resp.status_code, title=title)
        except TimeoutError:
            return ScrapeResult(url=url, error="timeout")
        except httpx.RequestError as e:
            return ScrapeResult(url=url, error=str(e))

async def scrape_all(urls: list[str]) -> list[ScrapeResult]:
    sem = asyncio.Semaphore(CONCURRENCY)
    async with httpx.AsyncClient(
        headers={"User-Agent": "async-scraper/1.0"},
        timeout=15.0,
    ) as client:
        return await asyncio.gather(
            *[scrape_one(sem, client, url) for url in urls],
            return_exceptions=False,
        )

async def main():
    urls = [
        "https://python.org",
        "https://httpbin.org/html",
        "https://example.com",
    ]

    results = await scrape_all(urls)

    print(f"\nScraped {len(results)} pages:\n")
    for r in results:
        if r.error:
            print(f"  ✗ {r.url[:50]:<50} ERROR: {r.error}")
        else:
            print(f"  ✓ [{r.status}] {r.url[:40]:<40} {r.title[:40]}")

asyncio.run(main())

Quick Reference Card

Async Queue
q = asyncio.Queue(maxsize=N); await q.put(x); x = await q.get()

Suspends producer when full, consumer when empty. Call q.task_done() after processing.

Semaphore
sem = asyncio.Semaphore(10); async with sem: ...

Limit to N concurrent operations. Use as async context manager.

Timeout (3.11+)
async with asyncio.timeout(5.0): await op()

Raises TimeoutError if the block takes longer than the deadline.

wait_for (3.8+)
await asyncio.wait_for(coro(), timeout=5.0)

Raises asyncio.TimeoutError. Works on any Python 3.8+ version.

Run blocking code
loop = asyncio.get_running_loop(); await loop.run_in_executor(None, fn, arg)

None = default thread pool. Keeps the event loop responsive.

Cancel a task
task = asyncio.create_task(coro()); task.cancel()

Injects CancelledError at the next await. Catch it for cleanup.

Async context manager
async with httpx.AsyncClient() as c: ...

Calls __aenter__/__aexit__. Use for clients, DB sessions, locks.

Async iterator
async for item in async_gen(): ...

Like a regular iterator but each step can await. Used in async generators and streams.

Wrap-up

  • asyncio.Queue decouples producers and consumers without threads. Use it when you want to control backpressure — a bounded queue (with maxsize) automatically slows producers when consumers can't keep up.
  • asyncio.Semaphore is the cleanest way to rate-limit concurrent operations when you're using asyncio.gather(). It's simpler than restructuring into a queue for cases where all tasks are known upfront.
  • Always add per-operation timeouts when making network calls in async code. Use asyncio.timeout() (Python 3.11+) or asyncio.wait_for() for compatibility — never rely solely on the HTTP library's timeout.
  • run_in_executor(None, fn, *args) is the escape hatch for synchronous blocking code. The None uses the default thread pool. This is how you safely call legacy synchronous libraries from async code.
  • The scraper pattern (Semaphore + gather + httpx + timeout + dataclass result) is a complete, production-ready template. Adjust the concurrency limit based on target server policy and your connection pool size.

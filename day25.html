Pandas Basics

Pandas brings labeled, structured tabular data to Python. Think of a DataFrame as a typed, programmable spreadsheet — or a SQL result set you can reshape, filter, aggregate, and join in code.

  • Selecting columns and rows: df["col"], df.loc, df.iloc
  • Filtering rows with boolean indexing

  • groupby and aggregate functions

  • DataFrame → a typed ResultSet you can transform in code

Series and DataFrame

Series — a labeled 1D array

A Series is a one-dimensional labeled array — like a NumPy array but with an index. Each value has a label, not just a position number. It's a single column of data.

Python 
import pandas as pd

# Create from list — auto-indexed 0, 1, 2...
prices = pd.Series([9.99, 14.99, 7.49, 19.99])
print(prices)
# 0     9.99
# 1    14.99
# 2     7.49
# 3    19.99
# dtype: float64

# Create with named index
scores = pd.Series({"Alice": 92, "Bob": 85, "Carol": 78})
print(scores["Alice"])   # 92
print(scores.mean())     # 85.0
print(scores > 80)       # Alice: True, Bob: True, Carol: False

DataFrame — a labeled 2D table

Python 
import pandas as pd

# Create from dict of lists — most common
data = {
    "name":   ["Alice", "Bob", "Carol", "Dave"],
    "dept":   ["Eng", "Sales", "Eng", "HR"],
    "salary": [95_000, 60_000, 88_000, 72_000],
    "years":  [5, 3, 7, 2],
}
df = pd.DataFrame(data)
print(df)
#     name   dept  salary  years
# 0  Alice    Eng   95000      5
# 1    Bob  Sales   60000      3
# 2  Carol    Eng   88000      7
# 3   Dave     HR   72000      2

# Basic inspection
print(df.shape)       # (4, 4) — rows, columns
print(df.dtypes)      # name: object, salary: int64 ...
print(df.describe())  # stats for numeric columns
print(df.head(2))     # first 2 rows
print(df.tail(2))     # last 2 rows
print(df.info())      # column types + null counts

Reading and Selecting Data

Loading from CSV

Python 
import pandas as pd

# Read CSV — handles headers, types, encoding automatically
df = pd.read_csv("employees.csv")

# Common options
df = pd.read_csv(
    "sales.csv",
    parse_dates=["date"],          # auto-parse date columns
    dtype={"amount": float},       # enforce column types
    na_values=["N/A", "—", ""],    # treat these as NaN
    encoding="utf-8",
    sep=",",                        # delimiter (default comma)
)

# Read just a few columns — faster for large files
df = pd.read_csv("large.csv", usecols=["id", "name", "amount"])

Selecting columns and rows

Python 
import pandas as pd

df = pd.DataFrame({
    "name":   ["Alice", "Bob", "Carol", "Dave"],
    "dept":   ["Eng", "Sales", "Eng", "HR"],
    "salary": [95_000, 60_000, 88_000, 72_000],
    "years":  [5, 3, 7, 2],
})

# Select a single column → Series
names = df["name"]

# Select multiple columns → DataFrame
subset = df[["name", "salary"]]

# .loc — label-based (row label, column label)
print(df.loc[0, "name"])         # "Alice" — row 0, column "name"
print(df.loc[0:2, "name":"salary"])  # rows 0-2, cols name to salary (inclusive!)

# .iloc — integer-position based (like list indexing)
print(df.iloc[0, 0])             # "Alice" — first row, first column
print(df.iloc[0:2, 0:2])         # rows 0-1, cols 0-1 (exclusive end!)
print(df.iloc[:, -1])            # last column: years
loc vs iloc: df.loc[0:2] includes row 2 (label-based, inclusive). df.iloc[0:2] excludes row 2 (position-based, like Python slicing). This inconsistency trips everyone up — know which one you're using.

Boolean filtering

Java — SQL equivalent 
// SELECT * FROM employees
// WHERE salary > 80000 AND dept = 'Eng'
employees.stream()
    .filter(e -> e.salary > 80_000
             && e.dept.equals("Eng"))
    .collect(toList());
Python 
import pandas as pd

# Single condition
high_earners = df[df["salary"] > 80_000]

# Multiple conditions — use & (and), | (or), ~ (not)
eng_high = df[(df["dept"] == "Eng") & (df["salary"] > 80_000)]

# .query() — SQL-like string syntax
eng_high = df.query("dept == 'Eng' and salary > 80_000")

# isin — filter by list of values
tech_depts = df[df["dept"].isin(["Eng", "Design"])]

# str methods on string columns
alices = df[df["name"].str.startswith("A")]
contains_a = df[df["name"].str.contains("a", case=False)]

Transforming Data

Adding and modifying columns

Python 
import pandas as pd

df = pd.DataFrame({
    "name":   ["Alice", "Bob", "Carol", "Dave"],
    "dept":   ["Eng", "Sales", "Eng", "HR"],
    "salary": [95_000, 60_000, 88_000, 72_000],
    "years":  [5, 3, 7, 2],
})

# New column from arithmetic
df["monthly"] = df["salary"] / 12

# New column from condition
df["senior"] = df["years"] >= 5

# New column from string operation
df["name_upper"] = df["name"].str.upper()

# apply() — apply any function row-by-row or element-by-element
df["band"] = df["salary"].apply(
    lambda s: "high" if s > 90_000 else "mid" if s > 70_000 else "low"
)

# rename columns
df = df.rename(columns={"salary": "annual_salary", "years": "tenure"})

# drop columns
df = df.drop(columns=["monthly"])

# sort
df_sorted = df.sort_values("annual_salary", ascending=False)
print(df_sorted[["name", "annual_salary", "band"]])

groupby and Aggregation

groupby works just like SQL's GROUP BY — no sorting required unlike itertools.groupby.

Java — SQL 
// SELECT dept, AVG(salary), COUNT(*)
// FROM employees GROUP BY dept
employees.stream()
    .collect(Collectors.groupingBy(
        e -> e.dept,
        Collectors.averagingDouble(e -> e.salary)
    ));
Python 
import pandas as pd

df = pd.DataFrame({
    "name":   ["Alice", "Bob", "Carol", "Dave", "Eve"],
    "dept":   ["Eng", "Sales", "Eng", "HR", "Sales"],
    "salary": [95_000, 60_000, 88_000, 72_000, 65_000],
    "years":  [5, 3, 7, 2, 4],
})

# Group by one column
by_dept = df.groupby("dept")

# Single aggregation
print(by_dept["salary"].mean())
# dept
# Eng      91500.0
# HR       72000.0
# Sales    62500.0

# Multiple aggregations
summary = by_dept["salary"].agg(["mean", "min", "max", "count"])
print(summary)

# Named aggregations — control output column names
summary = by_dept.agg(
    avg_salary=("salary", "mean"),
    headcount=("name",   "count"),
    avg_tenure=("years",  "mean"),
)
print(summary)

# Group by multiple columns
by_dept_seniority = df.groupby(["dept", df["years"] >= 5])["salary"].mean()

merge — SQL-Style Joins

Python 
import pandas as pd

employees = pd.DataFrame({
    "id":   [1, 2, 3, 4],
    "name": ["Alice", "Bob", "Carol", "Dave"],
    "dept_id": [10, 20, 10, 30],
})

departments = pd.DataFrame({
    "dept_id": [10, 20, 30],
    "dept_name": ["Engineering", "Sales", "HR"],
    "budget": [500_000, 200_000, 150_000],
})

# INNER JOIN — only matching rows
inner = pd.merge(employees, departments, on="dept_id")
print(inner[["name", "dept_name"]])

# LEFT JOIN — all employees, NaN for unmatched dept
left = pd.merge(employees, departments, on="dept_id", how="left")

# RIGHT JOIN / OUTER JOIN
right = pd.merge(employees, departments, on="dept_id", how="right")
outer = pd.merge(employees, departments, on="dept_id", how="outer")

# Join on different column names
pd.merge(employees, departments,
         left_on="dept_id", right_on="dept_id")

# Concatenate DataFrames vertically (like SQL UNION)
q1 = pd.DataFrame({"month": ["Jan", "Feb"], "sales": [100, 120]})
q2 = pd.DataFrame({"month": ["Mar", "Apr"], "sales": [130, 110]})
all_data = pd.concat([q1, q2], ignore_index=True)

Quick Reference Card

Create DataFrame
pd.DataFrame({"col": [1,2,3]})

From dict of lists. Also: pd.read_csv("file.csv")

Select column
df["name"] / df[["name","salary"]]

Single bracket → Series. Double bracket → DataFrame.

Label-based select
df.loc[0:2, "name":"salary"]

Both ends inclusive. Use for named indices.

Position-based select
df.iloc[0:2, 0:3]

Exclusive end (like Python slicing). Use for positional access.

Filter rows
df[df["salary"] > 80_000]

Use & for AND, | for OR. Or use df.query("salary > 80000").

groupby
df.groupby("dept")["salary"].agg(["mean","count"])

Works like SQL GROUP BY. No pre-sorting needed.

merge / join
pd.merge(df1, df2, on="id", how="inner")

how: "inner", "left", "right", "outer". Like SQL JOIN.

Export
df.to_csv("out.csv", index=False)

Also: to_json(), to_excel(), to_parquet().

Wrap-up

  • A DataFrame is a labeled 2D table — columns have names, rows have an index. Think of it as a SQL result set that you can reshape, filter, and aggregate without writing SQL.
  • df.loc is label-based (inclusive on both ends). df.iloc is position-based (exclusive end, like Python slicing). Know which you're using — they behave differently at the boundaries.
  • Boolean filtering (df[df["col"] > value]) is the primary way to query rows. Use & and | for multiple conditions, always wrapped in parentheses. Use df.query() for readability with complex conditions.
  • groupby is Pandas' GROUP BY — it works on unsorted data (unlike Python's itertools.groupby) and supports multiple aggregations with agg(). Named aggregations keep your output column names clean.
  • pd.merge() is SQL JOIN. The how parameter controls whether unmatched rows are kept. Always specify how explicitly — don't rely on the default "inner" being what you want.

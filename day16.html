Day 16 Phase 2 — Core Libraries

itertools + functools

Two modules that complete Python's functional toolkit. itertools gives you lazy combinators for building data pipelines. functools gives you higher-order function tools for composition and partial application.

Plan for the Day

What you'll cover

  • itertools: chain, islice, cycle, repeat
  • itertools: product, permutations, combinations
  • itertools.groupby — group consecutive elements
  • functools.reduce — fold a sequence into a single value
  • functools.partial — pre-fill function arguments
  • operator module — function versions of operators

Java parallels

  • itertools.chain → Stream.concat()
  • itertools.islice → stream.limit(n).skip(m)
  • itertools.groupby → Collectors.groupingBy() (but requires sorted input)
  • functools.reduce → stream.reduce(identity, accumulator)
  • functools.partial → no direct equivalent; closest is a lambda or method reference
  • itertools.product → nested for-loops or flatMap

itertools — Lazy Sequence Tools

chain, islice, cycle, repeat

All itertools functions return iterators — they produce values lazily. Compose them to build memory-efficient data pipelines without intermediate lists.

Python 
import itertools

# chain — concatenate iterables without building a list
a = [1, 2, 3]
b = [4, 5, 6]
c = (7, 8, 9)
print(list(itertools.chain(a, b, c)))   # [1, 2, 3, 4, 5, 6, 7, 8, 9]

# chain.from_iterable — flatten one level of nesting
nested = [[1, 2], [3, 4], [5, 6]]
print(list(itertools.chain.from_iterable(nested)))  # [1, 2, 3, 4, 5, 6]

# islice — lazy slicing of any iterator
gen = (x**2 for x in range(1_000_000))
first_five = list(itertools.islice(gen, 5))      # [0, 1, 4, 9, 16]
next_five  = list(itertools.islice(gen, 5, 10))  # [25, 36, 49, 64, 81]

# cycle — repeat an iterable forever
colors = itertools.cycle(["red", "green", "blue"])
labels = list(itertools.islice(colors, 7))
print(labels)  # ['red', 'green', 'blue', 'red', 'green', 'blue', 'red']

# repeat — repeat a value N times (or forever)
print(list(itertools.repeat(0, 5)))    # [0, 0, 0, 0, 0]
print(list(map(pow, range(5), itertools.repeat(2))))  # [0,1,4,9,16]

product, permutations, combinations

These replace nested loops for combinatorial problems. All return iterators — use list() to materialise or iterate directly.

Python 
import itertools

# product — Cartesian product (replaces nested for-loops)
suits = ["♠", "♥", "♦", "♣"]
ranks = ["A", "K", "Q", "J"]
cards = list(itertools.product(ranks, suits))
print(cards[:4])
# [('A', '♠'), ('A', '♥'), ('A', '♦'), ('A', '♣')]

# product with repeat — e.g. all 2-digit binary numbers
binary = list(itertools.product([0, 1], repeat=2))
print(binary)   # [(0,0), (0,1), (1,0), (1,1)]

# permutations — ordered arrangements
print(list(itertools.permutations("ABC", 2)))
# [('A','B'),('A','C'),('B','A'),('B','C'),('C','A'),('C','B')]

# combinations — unordered selections (no repeats)
print(list(itertools.combinations("ABCD", 2)))
# [('A','B'),('A','C'),('A','D'),('B','C'),('B','D'),('C','D')]

# combinations_with_replacement — allows repeats
print(list(itertools.combinations_with_replacement("AB", 2)))
# [('A','A'),('A','B'),('B','B')]

groupby — group consecutive elements

groupby groups consecutive elements that share the same key. Important: the input must be sorted by the key — groupby only groups adjacent matching elements, like SQL GROUP BY on sorted data.

Java — Stream.collect 
// Java groups unsorted data — itertools.groupby doesn't!
Map<String, List<String>> grouped =
    words.stream()
         .collect(Collectors.groupingBy(
             w -> String.valueOf(w.charAt(0))
         ));
Python — itertools.groupby 
import itertools

words = ["apple", "avocado", "banana", "blueberry", "cherry", "cranberry"]
# MUST be sorted by the grouping key first!
words.sort(key=lambda w: w[0])

for letter, group in itertools.groupby(words, key=lambda w: w[0]):
    print(f"{letter}: {list(group)}")
# a: ['apple', 'avocado']
# b: ['banana', 'blueberry']
# c: ['cherry', 'cranberry']

# Practical: run-length encoding
data = "AAABBBCCDDDDEE"
rle = [(k, len(list(g))) for k, g in itertools.groupby(data)]
print(rle)
# [('A', 3), ('B', 3), ('C', 2), ('D', 4), ('E', 2)]

accumulate, starmap, takewhile, dropwhile

Python 
import itertools, operator

# accumulate — running totals (like a scan/fold that keeps intermediates)
sales = [100, 200, 150, 300, 250]
running = list(itertools.accumulate(sales))
print(running)  # [100, 300, 450, 750, 1000]

running_max = list(itertools.accumulate(sales, max))
print(running_max)  # [100, 200, 200, 300, 300]

# starmap — map with argument unpacking
pairs = [(2, 3), (4, 5), (6, 7)]
print(list(itertools.starmap(pow, pairs)))   # [8, 1024, 279936]
print(list(itertools.starmap(operator.add, pairs)))  # [5, 9, 13]

# takewhile / dropwhile — conditional slicing
nums = [1, 3, 5, 4, 2, 6, 8]
print(list(itertools.takewhile(lambda x: x < 5, nums)))  # [1, 3]
print(list(itertools.dropwhile(lambda x: x < 5, nums)))  # [5, 4, 2, 6, 8]

functools — Higher-Order Function Tools

reduce — fold a sequence

reduce applies a function cumulatively to reduce a sequence to a single value. It's the general form of sum(), max(), etc. — use it when no built-in aggregator fits.

Java 
int product = IntStream.of(1, 2, 3, 4, 5)
    .reduce(1, (a, b) -> a * b);   // 120

String joined = Stream.of("a", "b", "c")
    .reduce("", (a, b) -> a + b);  // "abc"
Python 
from functools import reduce
import operator

nums = [1, 2, 3, 4, 5]

product = reduce(lambda a, b: a * b, nums)   # 120
product = reduce(operator.mul, nums)          # same, faster

# With initial value (safe on empty sequences)
total = reduce(operator.add, [], 0)   # 0 — no error

# Flatten a list of lists
nested = [[1, 2], [3, 4], [5, 6]]
flat = reduce(operator.add, nested)   # [1, 2, 3, 4, 5, 6]

# Build a pipeline of functions
def compose(*fns):
    """Compose functions right-to-left: compose(f, g)(x) == f(g(x))"""
    return reduce(lambda f, g: lambda x: f(g(x)), fns)

double = lambda x: x * 2
inc    = lambda x: x + 1
double_then_inc = compose(inc, double)
print(double_then_inc(5))   # 11

partial — pre-fill arguments

partial creates a new callable with some arguments already filled in. It's the Python equivalent of binding arguments in functional programming — useful for adapting a function's signature for callbacks, map(), or configuration.

Python 
from functools import partial

def power(base: int, exp: int) -> int:
    return base ** exp

square = partial(power, exp=2)   # exp is pre-filled
cube   = partial(power, exp=3)

print(square(4))   # 16
print(cube(3))     # 27
print(list(map(square, [1, 2, 3, 4, 5])))  # [1, 4, 9, 16, 25]

# Practical: configuring a logger with a fixed prefix
import logging

def log_with_prefix(prefix: str, level: int, msg: str) -> None:
    logging.log(level, f"[{prefix}] {msg}")

warn_db  = partial(log_with_prefix, "DB",  logging.WARNING)
info_api = partial(log_with_prefix, "API", logging.INFO)

warn_db("connection timeout")   # [DB] connection timeout
info_api("request received")    # [API] request received

# partial also works with positional args
int_from_binary = partial(int, base=2)
print(int_from_binary("1010"))   # 10
print(int_from_binary("1111"))   # 15

operator module — function versions of operators

The operator module exposes Python's built-in operators as regular functions — useful when you need to pass an operator to reduce, map, or sorted.

Python 
import operator
from functools import reduce

# operator functions mirror their operator symbols
print(operator.add(3, 4))     # 7  — same as 3 + 4
print(operator.mul(3, 4))     # 12 — same as 3 * 4
print(operator.lt(3, 4))      # True — same as 3 < 4
print(operator.not_(True))    # False — same as not True

# attrgetter — get an attribute by name (useful for sorted key)
from operator import attrgetter, itemgetter

class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

employees = [Employee("Bob", 80_000), Employee("Alice", 95_000)]
by_salary = sorted(employees, key=attrgetter("salary"))
print([e.name for e in by_salary])  # ['Bob', 'Alice']

# itemgetter — get dict/tuple item by key (faster than lambda)
records = [{"name": "Bob", "score": 70}, {"name": "Alice", "score": 90}]
by_score = sorted(records, key=itemgetter("score"), reverse=True)
print([r["name"] for r in by_score])   # ['Alice', 'Bob']

# Much faster than: key=lambda r: r["score"]

Quick Reference Card

chain
itertools.chain(a, b, c)

Concatenate iterables lazily. chain.from_iterable(nested) flattens one level.

islice
islice(gen, start, stop)

Lazy slice of any iterator. Essential for taking N items from a generator.

product
product(a, b) / product(a, repeat=2)

Cartesian product. Replaces nested for-loops.

groupby
groupby(sorted_data, key=fn)

Group consecutive matching elements. Input MUST be sorted by key first.

accumulate
accumulate(data, operator.add)

Running totals / scan. Default is addition. Pass any binary function.

takewhile / dropwhile
takewhile(pred, it) / dropwhile(pred, it)

Take/skip elements while predicate is true, then stop/start.

reduce
reduce(operator.mul, [1,2,3,4])

Fold a sequence to one value. Provide initial value for safety on empty input.

partial
partial(fn, arg1, kwarg=val)

Pre-fill arguments. Returns a new callable with fewer required parameters.

attrgetter / itemgetter
sorted(objs, key=attrgetter("name"))

Faster than lambda x: x.name or lambda x: x["key"].

Wrap-up

  • All itertools functions are lazy — they return iterators, not lists. This means you can compose them into long pipelines without any intermediate memory cost. Wrap in list() only when you need all values at once.
  • groupby is not like SQL's GROUP BY — it only groups consecutive identical keys. Always sort by your key first, or use defaultdict if your data isn't pre-sorted.
  • reduce is the general fold. Use built-ins (sum, max, min, any, all) when they exist — they're faster and more readable. Reach for reduce only when no built-in fits.
  • partial is cleaner than wrapping in a lambda when you're partially applying a function. partial(int, base=2) is more descriptive and reusable than lambda x: int(x, 2).
  • operator.attrgetter and operator.itemgetter are faster than equivalent lambdas for sorting keys. Prefer them in performance-sensitive code and when the lambda would be trivial.

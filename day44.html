1. How Python Creates Classes

In Python, everything is an object — including classes. When you write class Foo:, Python calls the metaclass (by default type) to create the class object. type is both a function and a class.

Python — type() as a class factory 
# The usual way
class Point:
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y
    def distance(self) -> float:
        return (self.x**2 + self.y**2) ** 0.5

# Exactly equivalent — type(name, bases, namespace)
Point = type(
    "Point",
    (object,),
    {
        "__init__": lambda self, x, y: (setattr(self, "x", x), setattr(self, "y", y)),
        "distance": lambda self: (self.x**2 + self.y**2) ** 0.5,
    }
)

p = Point(3, 4)
print(p.distance())    # 5.0

# Every class has a __class__ and a __class__.__class__
print(type(p))             # 
print(type(Point))         #   ← Point's metaclass is type
print(type(type))          #   ← type is its own metaclass
The metaclass call sequence when Python sees class Foo(Base): ...:
1. Collect the class body into a namespace dict
2. Determine the metaclass (default: type)
3. Call metaclass(name, bases, namespace) → this returns the class object
4. Bind the result to the name Foo

2. Writing a Custom Metaclass

A metaclass is a class that inherits from type. You override __new__ or __init__ to customize class creation.

Python — singleton via metaclass 
class SingletonMeta(type):
    """Metaclass that makes every class using it a singleton."""
    _instances: dict = {}

    def __call__(cls, *args, **kwargs):
        # __call__ is invoked when you do MyClass(...)
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

class DatabaseConnection(metaclass=SingletonMeta):
    def __init__(self, url: str):
        self.url = url
        self.connected = False

    def connect(self):
        self.connected = True

db1 = DatabaseConnection("postgresql://localhost/mydb")
db2 = DatabaseConnection("postgresql://other/db")   # ignored — same instance returned
print(db1 is db2)      # True
print(db2.url)         # "postgresql://localhost/mydb"

Auto-registering Subclasses

Python — plugin registry via metaclass 
class PluginMeta(type):
    """Automatically registers every subclass in a registry."""
    registry: dict[str, type] = {}

    def __init__(cls, name, bases, namespace):
        super().__init__(name, bases, namespace)
        if bases:    # don't register the base class itself
            PluginMeta.registry[name] = cls

class BaseExporter(metaclass=PluginMeta):
    def export(self, data: list) -> str:
        raise NotImplementedError

class CsvExporter(BaseExporter):
    def export(self, data: list) -> str:
        return ",".join(str(x) for x in data)

class JsonExporter(BaseExporter):
    def export(self, data: list) -> str:
        import json; return json.dumps(data)

# After defining the classes, the registry is populated automatically
print(PluginMeta.registry)
# {"CsvExporter": , "JsonExporter": }

# Factory function using the registry
def get_exporter(name: str) -> BaseExporter:
    cls = PluginMeta.registry.get(name)
    if not cls:
        raise ValueError(f"Unknown exporter: {name}")
    return cls()

__init_subclass__ — The Lighter Alternative

Python 
# For many metaclass use cases, __init_subclass__ is simpler
class BaseHandler:
    _registry: dict[str, type] = {}

    def __init_subclass__(cls, route: str, **kwargs):
        super().__init_subclass__(**kwargs)
        BaseHandler._registry[route] = cls
        print(f"Registered handler {cls.__name__} for route '{route}'")

class HomeHandler(BaseHandler, route="/"):
    def handle(self): return "Home page"

class AboutHandler(BaseHandler, route="/about"):
    def handle(self): return "About page"

# Registered automatically at class definition time
print(BaseHandler._registry)
# {"/": HomeHandler, "/about": AboutHandler}

# Prefer __init_subclass__ over metaclasses when you don't need
# to intercept class creation itself (just subclass registration)

3. Real-world Metaclass: ORM-style

SQLAlchemy's declarative base, Django's models, and Python's @dataclass all use metaclasses (or __init_subclass__) under the hood. Here's a simplified ORM field-declaration system to show the pattern.

Python — mini ORM metaclass 
class Field:
    def __init__(self, field_type: type, required: bool = True):
        self.field_type = field_type
        self.required = required

class ModelMeta(type):
    def __new__(mcs, name, bases, namespace):
        fields = {
            k: v for k, v in namespace.items()
            if isinstance(v, Field)
        }
        namespace["_fields"] = fields
        # Generate __init__ that validates types
        def __init__(self, **kwargs):
            for fname, fdef in self._fields.items():
                value = kwargs.get(fname)
                if fdef.required and value is None:
                    raise ValueError(f"{fname} is required")
                if value is not None and not isinstance(value, fdef.field_type):
                    raise TypeError(f"{fname} must be {fdef.field_type.__name__}")
                setattr(self, fname, value)
        namespace["__init__"] = __init__
        return super().__new__(mcs, name, bases, namespace)

class Model(metaclass=ModelMeta):
    pass

class User(Model):
    id = Field(int)
    name = Field(str)
    email = Field(str)

u = User(id=1, name="Vipin", email="v@example.com")
print(u.name)      # "Vipin"
# User(id="bad")  → TypeError: id must be int

4. __slots__ — Memory-Efficient Objects

By default, Python stores instance attributes in a __dict__ — a hash table. For objects you create millions of, this overhead adds up. __slots__ replaces the dict with a fixed set of descriptors, reducing memory by 40–60%.

Python — __slots__ basics 
class Point:
    """Regular class — each instance has a __dict__."""
    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

class PointSlots:
    """Slotted class — no __dict__, fixed attributes only."""
    __slots__ = ("x", "y")

    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

p1 = Point(1.0, 2.0)
p2 = PointSlots(1.0, 2.0)

# Regular class has a __dict__
print(p1.__dict__)              # {"x": 1.0, "y": 2.0}

# Slotted class does not
# print(p2.__dict__)            # AttributeError!

# Can't add arbitrary attributes to slotted instances
p1.z = 3.0                      # works — dict is flexible
# p2.z = 3.0                    # AttributeError: z not in __slots__

Memory and Speed Benchmark

Python 
import sys
import timeit

# Memory comparison
p_regular = Point(1.0, 2.0)
p_slotted = PointSlots(1.0, 2.0)

print(sys.getsizeof(p_regular))         # ~48 bytes (just the object header)
print(sys.getsizeof(p_regular.__dict__))# ~232 bytes (the dict overhead)
print(sys.getsizeof(p_slotted))         # ~56 bytes (slots are inline)
# Total regular: ~280 bytes vs slotted: ~56 bytes — 5x difference

# Speed comparison — attribute access
regular_time = timeit.timeit(
    "p.x", setup="from __main__ import Point; p = Point(1.0, 2.0)", number=10_000_000
)
slotted_time = timeit.timeit(
    "p.x", setup="from __main__ import PointSlots; p = PointSlots(1.0, 2.0)", number=10_000_000
)
print(f"Regular: {regular_time:.3f}s")
print(f"Slotted: {slotted_time:.3f}s")
# Slotted is typically 20-30% faster for attribute access

__slots__ with Inheritance

Python 
class Animal:
    __slots__ = ("name", "age")
    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

class Dog(Animal):
    # Add slots for Dog-specific attributes
    # Don't redeclare parent slots — they're inherited
    __slots__ = ("breed",)

    def __init__(self, name: str, age: int, breed: str):
        super().__init__(name, age)
        self.breed = breed

d = Dog("Rex", 3, "Labrador")
print(d.name, d.breed)    # works — both parent and child slots available

# WARNING: if ANY class in the hierarchy doesn't define __slots__,
# a __dict__ is added back and the memory savings are lost.
class BadBase:
    pass   # no __slots__

class BadDog(BadBase):
    __slots__ = ("breed",)   # useless — BadBase added __dict__ back

__slots__ with dataclass

Python 3.10+ 
from dataclasses import dataclass

# slots=True generates __slots__ automatically — best of both worlds
@dataclass(slots=True, frozen=True)
class Coordinate:
    lat: float
    lon: float
    altitude: float = 0.0

c = Coordinate(lat=37.7749, lon=-122.4194)
print(c.lat)
# c.lat = 1.0  → FrozenInstanceError (frozen=True)
# c.extra = 1  → AttributeError (slots=True)
# Memory-efficient + immutable + auto-generated __init__/__repr__/__eq__

Wrap-up

  • type is Python's built-in metaclass. Every class is an instance of type. Metaclasses let you customize what happens when a class is defined — not when it's instantiated.
  • Most metaclass use cases can be replaced with __init_subclass__, which is simpler and more readable. Reach for metaclasses only when you need to intercept or transform the class namespace itself.
  • __slots__ removes the per-instance __dict__ and replaces it with fixed descriptors. For classes with millions of instances, this is a 3–5× memory reduction with a 20–30% attribute access speedup.
  • If any class in an inheritance chain doesn't define __slots__, a __dict__ is silently added back. You lose all the memory savings. Apply __slots__ consistently throughout the hierarchy.
  • @dataclass(slots=True) (Python 3.10+) is the pragmatic choice — you get slot efficiency without manually writing __slots__ or a custom __init__.

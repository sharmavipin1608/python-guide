Project Phase 1 — Foundation

Refactor CLI with OOP

Take the Day 7 task manager and rebuild it properly — a Task dataclass, a TaskManager class, a logging decorator, and type hints throughout. Same features, dramatically cleaner code.

Project Spec

What to build

  • A Task dataclass with id, title, description, done, created_at fields
  • A TaskManager class that owns the task list and all operations
  • JSON persistence via a JsonStorage class with a context manager interface
  • A @logged decorator that records each action with a timestamp
  • Full type hints on every function and method
  • Same CLI menu as Day 7: add, list, complete, delete, quit

Stretch goals

  • Add a TaskFilter class with method chaining: TaskFilter(tasks).pending().created_before(date).all()
  • Add due dates to Task using datetime and filter overdue tasks
  • Add a @validate_input decorator that prevents empty task titles

Architecture Overview

The refactor separates concerns that were mixed together in Day 7. Each class has one job:

models.py — Task dataclass (data only, no logic)
storage.py — JsonStorage class (file I/O only)
manager.py — TaskManager class (business logic)
decorators.py — @logged, @validate_input decorators
cli.py — CLI class (user interaction only)
main.py — entry point, wires everything together
Day 7 vs Day 14: Day 7 had all logic in functions scattered across two files. Day 14 uses classes so each concern is encapsulated — adding features only touches the relevant class, not the whole codebase.

models.py — The Task Dataclass

📄 models.py
Python 
from dataclasses import dataclass, field
from datetime import datetime


@dataclass
class Task:
    """Represents a single task. Immutable id and created_at."""
    id: int
    title: str
    description: str = ""
    done: bool = False
    created_at: datetime = field(default_factory=datetime.now)

    def complete(self) -> "Task":
        """Return a new Task with done=True (keeps original intact)."""
        # We can't mutate in place cleanly if we want to keep history,
        # but for simplicity here we just set done directly
        self.done = True
        return self

    def to_dict(self) -> dict:
        """Serialize to JSON-safe dict."""
        return {
            "id": self.id,
            "title": self.title,
            "description": self.description,
            "done": self.done,
            "created_at": self.created_at.isoformat(),
        }

    @classmethod
    def from_dict(cls, data: dict) -> "Task":
        """Deserialize from dict (e.g. loaded from JSON)."""
        return cls(
            id=data["id"],
            title=data["title"],
            description=data.get("description", ""),
            done=data.get("done", False),
            created_at=datetime.fromisoformat(data["created_at"]),
        )

    def __str__(self) -> str:
        status = "✓" if self.done else "○"
        return f"[{status}] #{self.id} {self.title}"

storage.py — JSON Persistence

📄 storage.py
Python 
import json
from pathlib import Path
from contextlib import contextmanager
from typing import Generator

from models import Task


class JsonStorage:
    """Handles reading and writing tasks to a JSON file."""

    def __init__(self, path: str | Path = "tasks.json"):
        self.path = Path(path)

    def load(self) -> list[Task]:
        """Load all tasks from file. Returns empty list if file missing."""
        if not self.path.exists():
            return []
        try:
            data = json.loads(self.path.read_text())
            return [Task.from_dict(d) for d in data]
        except (json.JSONDecodeError, KeyError) as e:
            print(f"Warning: could not load tasks ({e}), starting fresh.")
            return []

    def save(self, tasks: list[Task]) -> None:
        """Write all tasks to file."""
        self.path.write_text(
            json.dumps([t.to_dict() for t in tasks], indent=2)
        )

    @contextmanager
    def session(self) -> Generator[list[Task], None, None]:
        """Load tasks, yield them for modification, save on exit."""
        tasks = self.load()
        try:
            yield tasks
        finally:
            self.save(tasks)

decorators.py — Logging + Validation

📄 decorators.py
Python 
import functools
from datetime import datetime
from typing import Callable, TypeVar, Any

F = TypeVar("F", bound=Callable[..., Any])


def logged(func: F) -> F:
    """Log method calls with timestamp and arguments."""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        ts = datetime.now().strftime("%H:%M:%S")
        # args[0] is self — skip it for cleaner log output
        call_args = ", ".join(
            [repr(a) for a in args[1:]] +
            [f"{k}={v!r}" for k, v in kwargs.items()]
        )
        print(f"[{ts}] {func.__name__}({call_args})")
        return func(*args, **kwargs)
    return wrapper  # type: ignore[return-value]


def validate_title(func: F) -> F:
    """Reject empty or whitespace-only task titles."""
    @functools.wraps(func)
    def wrapper(self, title: str, *args, **kwargs):
        if not title or not title.strip():
            print("Error: task title cannot be empty.")
            return None
        return func(self, title.strip(), *args, **kwargs)
    return wrapper  # type: ignore[return-value]

manager.py — Business Logic

📄 manager.py
Python 
from models import Task
from storage import JsonStorage
from decorators import logged, validate_title


class TaskManager:
    """Owns the task list and all business operations."""

    def __init__(self, storage: JsonStorage):
        self._storage = storage
        self._tasks: list[Task] = storage.load()

    def _next_id(self) -> int:
        return max((t.id for t in self._tasks), default=0) + 1

    @logged
    @validate_title
    def add(self, title: str, description: str = "") -> Task:
        """Create and store a new task."""
        task = Task(id=self._next_id(), title=title, description=description)
        self._tasks.append(task)
        self._storage.save(self._tasks)
        return task

    @logged
    def complete(self, task_id: int) -> bool:
        """Mark a task as done. Returns True if found."""
        for task in self._tasks:
            if task.id == task_id:
                task.complete()
                self._storage.save(self._tasks)
                return True
        print(f"Task #{task_id} not found.")
        return False

    @logged
    def delete(self, task_id: int) -> bool:
        """Remove a task. Returns True if found."""
        original = len(self._tasks)
        self._tasks = [t for t in self._tasks if t.id != task_id]
        if len(self._tasks) < original:
            self._storage.save(self._tasks)
            return True
        print(f"Task #{task_id} not found.")
        return False

    def all(self) -> list[Task]:
        return list(self._tasks)

    def pending(self) -> list[Task]:
        return [t for t in self._tasks if not t.done]

    def completed(self) -> list[Task]:
        return [t for t in self._tasks if t.done]

cli.py — User Interface

📄 cli.py
Python 
from manager import TaskManager


class CLI:
    """Handles all user interaction. Knows nothing about storage."""

    def __init__(self, manager: TaskManager):
        self._mgr = manager

    def run(self) -> None:
        print("\n=== Task Manager (OOP Edition) ===")
        actions = {
            "1": self._add,
            "2": self._list,
            "3": self._complete,
            "4": self._delete,
            "5": self._quit,
        }
        while True:
            self._print_menu()
            choice = input("Choice: ").strip()
            action = actions.get(choice)
            if action:
                action()
            else:
                print("Invalid choice.")

    def _print_menu(self) -> None:
        pending = len(self._mgr.pending())
        print(f"\n[{pending} pending]")
        print("1) Add task   2) List tasks   3) Complete   4) Delete   5) Quit")

    def _add(self) -> None:
        title = input("Title: ").strip()
        desc  = input("Description (optional): ").strip()
        task  = self._mgr.add(title, desc)
        if task:
            print(f"Added: {task}")

    def _list(self) -> None:
        tasks = self._mgr.all()
        if not tasks:
            print("No tasks yet.")
            return
        print(f"\n{'ID':>3}  {'Status':<8} {'Title'}")
        print("-" * 40)
        for t in tasks:
            status = "Done" if t.done else "Pending"
            print(f"{t.id:>3}  {status:<8} {t.title}")

    def _complete(self) -> None:
        try:
            task_id = int(input("Task ID to complete: "))
            if self._mgr.complete(task_id):
                print(f"Task #{task_id} marked as done.")
        except ValueError:
            print("Please enter a valid number.")

    def _delete(self) -> None:
        try:
            task_id = int(input("Task ID to delete: "))
            if self._mgr.delete(task_id):
                print(f"Task #{task_id} deleted.")
        except ValueError:
            print("Please enter a valid number.")

    def _quit(self) -> None:
        print("Goodbye!")
        raise SystemExit(0)

Key Design Decisions

Dependency injection — not global state

TaskManager receives its JsonStorage via the constructor rather than importing and calling it directly. This means you can swap in a different storage backend (in-memory, SQLite, remote API) without changing TaskManager at all — and you can pass a fake storage in tests.

Java parallel: This is constructor injection — the same pattern as @Autowired in Spring, but without a framework. Python doesn't need a DI container for simple cases; passing dependencies as constructor args is idiomatic.

Decorator stacking on methods

The add method stacks two decorators. Remember: bottom-up application means validate_title wraps the raw method first, then logged wraps that. On call: logged runs first (prints the log), then passes control to validate_title, which validates before calling the real add.

Python 
@logged          # outermost — runs first on call
@validate_title  # innermost — runs second, before add body
def add(self, title: str, description: str = "") -> Task:
    ...

Stretch: TaskFilter with method chaining

Python 
from datetime import datetime
from models import Task


class TaskFilter:
    """Fluent interface for filtering tasks."""

    def __init__(self, tasks: list[Task]):
        self._tasks = list(tasks)

    def pending(self) -> "TaskFilter":
        self._tasks = [t for t in self._tasks if not t.done]
        return self   # return self for chaining

    def completed(self) -> "TaskFilter":
        self._tasks = [t for t in self._tasks if t.done]
        return self

    def created_before(self, dt: datetime) -> "TaskFilter":
        self._tasks = [t for t in self._tasks if t.created_at < dt]
        return self

    def with_title(self, substring: str) -> "TaskFilter":
        self._tasks = [t for t in self._tasks if substring.lower() in t.title.lower()]
        return self

    def all(self) -> list[Task]:
        return list(self._tasks)

    def count(self) -> int:
        return len(self._tasks)


# Usage
from manager import TaskManager
from storage import JsonStorage

manager = TaskManager(JsonStorage())

results = (
    TaskFilter(manager.all())
    .pending()
    .with_title("buy")
    .all()
)
print(f"Pending 'buy' tasks: {len(results)}")

Quick Reference Card

Dataclass with from_dict
@classmethod / def from_dict(cls, d): return cls(...)

Factory classmethod for deserialization — replaces a static parse method.

Context manager storage
@contextmanager / def session(): tasks = load(); yield tasks; save(tasks)

Load-yield-save pattern guarantees persistence even on exception.

Decorator on method
@logged / def add(self, title): ...

Works the same as on functions. args[0] is self.

Constructor injection
def __init__(self, storage: JsonStorage): self._storage = storage

Dependencies come in via constructor — testable, swappable, no global state.

Method chaining
def filter(self): ...; return self

Return self from each method to enable fluent chaining.

Entry point
if __name__ == "__main__": main()

Wire dependencies here — keep main.py thin, classes reusable.

Wrap-up

  • Splitting into models, storage, manager, decorators, and cli gives each class a single, clear responsibility. Adding a new feature (due dates, priorities) touches exactly one class — not the whole codebase.
  • Constructor injection makes TaskManager independently testable. You can pass a fake or in-memory Storage object in tests — no file system required.
  • The @dataclass + to_dict/from_dict pattern is the standard Python approach to serializable data objects. It replaces both the Lombok boilerplate and the manual Jackson mapping in Java.
  • Method chaining (return self) is the Python equivalent of Java's builder pattern — clean, readable, no builder class required.
  • Decorators on class methods work the same as on functions — args[0] is self. Stack them for composable cross-cutting concerns (logging, validation, timing) without touching the method body.

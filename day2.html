Day 02 Phase 1 — Foundation

Control Flow + Comprehensions

Python's if/elif/else, for loops over anything, while loops with their unique else clause, and list comprehensions — the idiom that replaces 90% of Java's for-loop-plus-add patterns.

Java parallels at a glance

  • Python uses indentation (4 spaces) to define blocks — no curly braces
  • elif ≈ Java's else if
  • match/case ≈ Java's switch expression (but much more powerful)
  • Python's for x in collection ≈ Java's enhanced for-each
  • List comprehensions ≈ Java streams: list.stream().filter(...).map(...).collect(...) — but terser

1. if / elif / else

Python uses indentation to define blocks — not curly braces. The standard is 4 spaces. This is enforced by the language, not just style.

Java 
int score = 85;

if (score >= 90) {
    System.out.println("A");
} else if (score >= 80) {
    System.out.println("B");
} else if (score >= 70) {
    System.out.println("C");
} else {
    System.out.println("F");
}
Python 
score = 85

if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
else:
    print("F")
The colon matters: Every if, elif, else, for, while, def, and class header ends with a colon. Forgetting it is the #1 syntax error for newcomers.

Comparison and logical operators

Java 
x == y    // equality
x != y    // not equal
x && y    // logical AND
x || y    // logical OR
!x        // logical NOT

// Checking multiple conditions
if (x > 0 && x < 10) { }
Python 
x == y    # equality
x != y    # not equal
x and y   # logical AND (spelled out)
x or y    # logical OR  (spelled out)
not x     # logical NOT (spelled out)

# Chained comparisons — no Java equivalent
if 0 < x < 10:      # reads like math
    pass

# Same as: if x > 0 and x < 10
# but more Pythonic

Membership and identity operators

Python 
fruits = ["apple", "banana", "cherry"]

# 'in' operator — like Java's .contains()
if "apple" in fruits:
    print("found it")

if "mango" not in fruits:
    print("not in list")

# 'is' — identity (same object in memory)
# Use for None checks, NOT for value equality
x = None
if x is None:       # correct
    pass
if x is not None:   # correct
    pass

# Don't use 'is' for int/str comparisons
# (works by coincidence for small ints due to caching)
a = 1000
b = 1000
a == b   # True  (value equality — use this)
a is b   # False (different objects in memory)

match / case (Python 3.10+)

Python didn't have a switch statement until 3.10, when match/case was introduced. It's more powerful than Java's switch — it supports structural pattern matching.

Java switch expression 
String day = "MON";

String type = switch (day) {
    case "MON", "TUE", "WED",
         "THU", "FRI" -> "Weekday";
    case "SAT", "SUN" -> "Weekend";
    default -> "Unknown";
};
Python match/case 
day = "MON"

match day:
    case "MON" | "TUE" | "WED" | "THU" | "FRI":
        result = "Weekday"
    case "SAT" | "SUN":
        result = "Weekend"
    case _:          # _ is the wildcard (like default)
        result = "Unknown"
match/case goes much further: It can destructure dicts, lists, and dataclasses — like Kotlin's when or Rust's match. You'll see this in depth on Day 13 with dataclasses.

2. for Loops

Python's for loop iterates directly over items — not over an index counter. This eliminates the classic Java for (int i = 0; i < list.size(); i++) pattern.

Java 
List<String> names = List.of("Alice","Bob","Carol");

// Enhanced for-each
for (String name : names) {
    System.out.println(name);
}

// Need index? Old style:
for (int i = 0; i < names.size(); i++) {
    System.out.println(i + ": " + names.get(i));
}
Python 
names = ["Alice", "Bob", "Carol"]

# Basic iteration
for name in names:
    print(name)

# Need index? Use enumerate()
for i, name in enumerate(names):
    print(f"{i}: {name}")

# enumerate starts at 0 by default
# start at 1:
for i, name in enumerate(names, start=1):
    print(f"{i}: {name}")

range() — like Java's index-based for loop

Java 
for (int i = 0; i < 5; i++) { }
for (int i = 0; i < 10; i += 2) { }
for (int i = 10; i > 0; i--) { }
Python 
for i in range(5):       # 0, 1, 2, 3, 4
    pass

for i in range(0, 10, 2):   # 0, 2, 4, 6, 8 (step)
    pass

for i in range(10, 0, -1):  # 10, 9, ..., 1 (reverse)
    pass

# range is lazy — no list created in memory
# for large loops this matters

zip() — iterate two collections in parallel

Java 
List<String> names = List.of("Alice", "Bob");
List<Integer> scores = List.of(95, 82);

for (int i = 0; i < names.size(); i++) {
    System.out.println(
        names.get(i) + ": " + scores.get(i));
}
Python 
names = ["Alice", "Bob"]
scores = [95, 82]

# zip pairs them up
for name, score in zip(names, scores):
    print(f"{name}: {score}")

# zip stops at shortest list
# Use itertools.zip_longest to pad instead

Iterating over dicts

Python 
config = {"host": "localhost", "port": 5432, "db": "mydb"}

# Just keys (default)
for key in config:
    print(key)

# Keys explicitly
for key in config.keys():
    print(key)

# Values
for value in config.values():
    print(value)

# Both — like Java's entrySet()
for key, value in config.items():
    print(f"{key} = {value}")

3. while, break, continue, and Loop else

Java 
int n = 10;
while (n > 0) {
    System.out.println(n);
    n--;
}

// break and continue work the same
for (int i = 0; i < 10; i++) {
    if (i == 3) continue;
    if (i == 7) break;
    System.out.println(i);
}
Python 
n = 10
while n > 0:
    print(n)
    n -= 1    # no -- operator in Python

# break and continue work the same
for i in range(10):
    if i == 3:
        continue    # skip 3
    if i == 7:
        break       # stop at 7
    print(i)

Loop else — Python's unique feature

Both for and while loops can have an else block. The else runs if the loop completed without hitting a break. This is genuinely useful for search patterns.

Java — search pattern 
List<Integer> nums = List.of(1, 3, 5, 7);
boolean found = false;

for (int n : nums) {
    if (n % 2 == 0) {
        System.out.println("Found even: " + n);
        found = true;
        break;
    }
}

if (!found) {
    System.out.println("No evens found");
}
Python — loop else 
nums = [1, 3, 5, 7]

for n in nums:
    if n % 2 == 0:
        print(f"Found even: {n}")
        break
else:
    # Runs only if loop ended WITHOUT break
    print("No evens found")

# No flag variable needed!
When to use loop else: Whenever you'd normally use a boolean flag variable like found = False to track whether a loop found what it was looking for. The else replaces the flag entirely.

4. List Comprehensions

List comprehensions are the most important Python idiom to internalize as a Java developer. They replace the common pattern of creating an empty list, looping, and conditionally appending.

Basic syntax

Java — stream equivalent 
List<Integer> nums = List.of(1,2,3,4,5);

// Map: square each number
List<Integer> squares = nums.stream()
    .map(n -> n * n)
    .collect(Collectors.toList());

// Filter: keep evens only
List<Integer> evens = nums.stream()
    .filter(n -> n % 2 == 0)
    .collect(Collectors.toList());

// Map + Filter
List<Integer> evenSquares = nums.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * n)
    .collect(Collectors.toList());
Python — list comprehensions 
nums = [1, 2, 3, 4, 5]

# Map: [expression for item in iterable]
squares = [n * n for n in nums]
# [1, 4, 9, 16, 25]

# Filter: [expression for item in iterable if condition]
evens = [n for n in nums if n % 2 == 0]
# [2, 4]

# Map + Filter combined
even_squares = [n * n for n in nums if n % 2 == 0]
# [4, 16]

Real-world examples

Python 
names = ["  alice  ", "BOB", "  Carol"]

# Clean and normalize a list of names
clean = [name.strip().title() for name in names]
# ["Alice", "Bob", "Carol"]

# Extract a field from a list of dicts
users = [
    {"name": "Alice", "age": 30, "active": True},
    {"name": "Bob",   "age": 25, "active": False},
    {"name": "Carol", "age": 35, "active": True},
]

# Get names of active users
active_names = [u["name"] for u in users if u["active"]]
# ["Alice", "Carol"]

# Transform: get (name, age) tuples
name_ages = [(u["name"], u["age"]) for u in users]
# [("Alice", 30), ("Bob", 25), ("Carol", 35)]

# Flatten a nested list
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flat = [x for row in matrix for x in row]
# [1, 2, 3, 4, 5, 6, 7, 8, 9]
Performance note: List comprehensions are faster than equivalent for-loop-plus-append in Python because the interpreter can optimize them. They're also more readable once you get used to the syntax. Prefer them over list(map(...)) or list(filter(...)) in most cases.

Nested list comprehensions

Python 
# Multiplication table: 3x3
table = [[i * j for j in range(1, 4)] for i in range(1, 4)]
# [[1, 2, 3], [2, 4, 6], [3, 6, 9]]

# When it gets complex, a regular for loop is clearer
# Don't sacrifice readability for terseness

5. Dict and Set Comprehensions

The comprehension syntax works for dicts and sets too.

Python — dict comprehension 
names = ["Alice", "Bob", "Carol"]
scores = [95, 82, 91]

# Build a dict from two lists
score_map = {name: score for name, score in zip(names, scores)}
# {"Alice": 95, "Bob": 82, "Carol": 91}

# Invert a dict (swap keys and values)
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
# {1: "a", 2: "b", 3: "c"}

# Filter a dict — keep entries where score > 85
high_scorers = {k: v for k, v in score_map.items() if v > 85}
# {"Alice": 95, "Carol": 91}

# Transform values
uppercased = {k.upper(): v for k, v in original.items()}
# {"A": 1, "B": 2, "C": 3}
Python — set comprehension 
words = ["apple", "banana", "avocado", "blueberry", "cherry"]

# Unique first letters
first_letters = {w[0] for w in words}
# {'a', 'b', 'c'}  — order not guaranteed (it's a set)

# Long words (deduped automatically)
long_words = {w for w in words if len(w) > 6}
# {'avocado', 'blueberry', 'banana', 'cherry'}

6. Ternary Expression

Java 
int age = 20;

// Ternary: condition ? value_if_true : value_if_false
String label = age >= 18 ? "adult" : "minor";

// Null coalescing (Java 8+)
String name = getName(); // might be null
String display = name != null ? name : "Anonymous";
Python 
age = 20

# Ternary: value_if_true if condition else value_if_false
label = "adult" if age >= 18 else "minor"

# "or" as null coalescing (for falsy values)
name = get_name()  # might return None or ""
display = name or "Anonymous"

# Careful: 'or' triggers on any falsy value (0, "", [], None)
# For strict None-only check, use:
display = name if name is not None else "Anonymous"

Putting it all together

Python — real example 
students = [
    {"name": "Alice", "grade": 92, "active": True},
    {"name": "Bob",   "grade": 74, "active": True},
    {"name": "Carol", "grade": 88, "active": False},
    {"name": "Dave",  "grade": 55, "active": True},
]

# Get letter grades for active students, sorted by name
results = [
    {
        "name": s["name"],
        "letter": (
            "A" if s["grade"] >= 90 else
            "B" if s["grade"] >= 80 else
            "C" if s["grade"] >= 70 else
            "F"
        )
    }
    for s in students
    if s["active"]
]

for r in results:
    print(f"{r['name']}: {r['letter']}")

for loops

while condition:

List comprehension

Dict comprehension

Set comprehension
{expr for x in xs}

Use {} — but empty set is set() not {}

Ternary

Membership

x not in collection
x is None / x is not None

Use is/is not for None; == for values

Wrap-up

  • Python uses indentation (4 spaces) to define blocks — no curly braces. The colon ends every block header.
  • and, or, not are words, not symbols. Chained comparisons like 0 < x < 10 work natively.
  • Python's for iterates directly over items. Use enumerate() for index+item, zip() for parallel iteration.
  • Loop else runs when a loop finishes without a break — useful for search patterns without a flag variable.
  • List comprehensions replace Java's for-loop-plus-add and most stream operations. Learn the syntax: [expr for x in xs if cond].
  • Dict and set comprehensions work the same way — build any collection in one readable expression.

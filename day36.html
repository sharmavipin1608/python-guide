1. What Is LangChain?

LangChain is a framework for building applications powered by large language models. It provides abstractions for the common patterns: prompting, chaining multiple LLM calls, connecting to external data, and giving LLMs tools to use.

When to use LangChain: Use it when you need more than a single LLM call — RAG pipelines, agent loops, multi-step reasoning, or routing between different models. For a simple "call GPT, return the result" use case, the raw OpenAI SDK (Day 39) is simpler.

2. LLM Wrappers

LangChain wraps different LLM providers behind a consistent interface. Swap out OpenAI for Ollama (local) or Anthropic without changing the rest of your code.

Python — OpenAI 
from langchain_openai import ChatOpenAI

# Chat model (GPT-4o, GPT-4, etc.)
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0,          # 0 = deterministic, 1 = creative
    max_tokens=500,
)

# Simple invocation
response = llm.invoke("What is the GIL in Python?")
print(response.content)     # AIMessage → .content for the text
Python — Ollama (local models) 
from langchain_ollama import ChatOllama

# Runs against your local Ollama instance (ollama pull llama3.2)
llm = ChatOllama(
    model="llama3.2",
    temperature=0,
)

response = llm.invoke("Explain Python generators in one paragraph.")
print(response.content)

# Same interface — just swap the LLM object, everything else stays identical

Message Types

Python 
from langchain_core.messages import HumanMessage, SystemMessage, AIMessage

# Build a conversation manually
messages = [
    SystemMessage(content="You are a Python expert helping a Java developer."),
    HumanMessage(content="What's the equivalent of HashMap in Python?"),
]

response = llm.invoke(messages)
print(response.content)      # "The equivalent of HashMap in Python is a dict..."

# Continue the conversation
messages.append(response)    # append the AIMessage
messages.append(HumanMessage(content="And what about TreeMap?"))
response2 = llm.invoke(messages)

3. Prompt Templates

A PromptTemplate is a reusable prompt with placeholders — like a parameterized SQL query. It keeps prompt text out of your business logic and makes prompts testable.

Python 
from langchain_core.prompts import ChatPromptTemplate

# Define template with {placeholders}
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a Python expert. Explain concepts clearly with Java parallels."),
    ("human", "Explain {concept} in Python. Compare it to {java_equivalent} in Java."),
])

# Format the prompt — produces a list of messages
messages = prompt.format_messages(
    concept="decorators",
    java_equivalent="AOP @Around advice",
)
print(messages[1].content)
# "Explain decorators in Python. Compare it to AOP @Around advice in Java."

# Invoke directly
response = llm.invoke(messages)

Few-Shot Prompting

Python 
from langchain_core.prompts import FewShotChatMessagePromptTemplate

examples = [
    {"input": "HashMap", "output": "dict"},
    {"input": "ArrayList", "output": "list"},
    {"input": "Optional", "output": "str | None"},
]

example_prompt = ChatPromptTemplate.from_messages([
    ("human", "Java: {input}"),
    ("ai", "Python: {output}"),
])

few_shot_prompt = FewShotChatMessagePromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
)

final_prompt = ChatPromptTemplate.from_messages([
    ("system", "Convert Java types to Python equivalents."),
    few_shot_prompt,
    ("human", "Java: {input}"),
])

chain = final_prompt | llm
response = chain.invoke({"input": "Set"})
print(response.content)    # "Python: set[str]"

4. Chains with LCEL

LangChain Expression Language (LCEL) uses the pipe operator | to compose steps. Each step's output becomes the next step's input — like a Unix pipeline or Java stream operations.

Python — basic chain 
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# prompt | llm | parser — each | passes output to the next step
chain = (
    ChatPromptTemplate.from_template(
        "Summarize this Python concept for a Java developer: {concept}"
    )
    | llm
    | StrOutputParser()     # extracts .content string from AIMessage
)

result = chain.invoke({"concept": "context managers"})
print(result)  # plain string, not AIMessage

Sequential Chain

Python 
from langchain_core.runnables import RunnablePassthrough

# Step 1: generate a Python code snippet
code_prompt = ChatPromptTemplate.from_template(
    "Write a short Python example of {concept}. Code only, no explanation."
)

# Step 2: review that code
review_prompt = ChatPromptTemplate.from_template(
    "Review this Python code and list any issues:\n\n{code}"
)

parser = StrOutputParser()

# Chain: generate code, then review it
generate_chain = code_prompt | llm | parser

review_chain = (
    {"code": generate_chain}       # pass generated code as 'code' key
    | review_prompt
    | llm
    | parser
)

result = review_chain.invoke({"concept": "async generators"})
print(result)

Streaming

Python 
# Stream tokens as they arrive — important for long responses
chain = (
    ChatPromptTemplate.from_template("Explain {topic} in detail.")
    | llm
    | StrOutputParser()
)

for chunk in chain.stream({"topic": "Python's GIL"}):
    print(chunk, end="", flush=True)    # print each token as it arrives
print()  # newline at end

5. Output Parsers

LLMs return text. Output parsers turn that text into structured Python objects — a critical step when you need the result to drive further logic.

Pydantic Output Parser

Python 
from langchain_core.output_parsers import PydanticOutputParser
from langchain_core.prompts import ChatPromptTemplate
from pydantic import BaseModel, Field

class CodeReview(BaseModel):
    issues: list[str] = Field(description="List of issues found in the code")
    severity: str = Field(description="overall severity: low, medium, or high")
    suggestion: str = Field(description="One-sentence improvement suggestion")

parser = PydanticOutputParser(pydantic_object=CodeReview)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a code reviewer. Respond in the requested format."),
    ("human", "Review this code:\n\n{code}\n\n{format_instructions}"),
]).partial(format_instructions=parser.get_format_instructions())

chain = prompt | llm | parser

result = chain.invoke({"code": "def add(a, b): return a+b"})
print(result.issues)      # ['No type hints', 'No docstring']
print(result.severity)    # 'low'
print(type(result))       #

JSON Output Parser

Python 
from langchain_core.output_parsers import JsonOutputParser

# Use with_structured_output for cleaner JSON (OpenAI function calling)
llm_json = ChatOpenAI(model="gpt-4o-mini", temperature=0)

class TaskExtraction(BaseModel):
    title: str
    priority: str
    due_date: str | None

structured_llm = llm_json.with_structured_output(TaskExtraction)

result = structured_llm.invoke(
    "Extract task info: 'Finish the API by Friday, it's urgent'"
)
print(result.title)      # "Finish the API"
print(result.priority)   # "high"
print(result.due_date)   # "Friday"

Wrap-up

  • LangChain's LLM wrappers (ChatOpenAI, ChatOllama) share the same interface — you can swap models without changing prompt or chain code.
  • LCEL's pipe operator | is LangChain's most important concept: prompt | llm | parser reads left-to-right and each step's output feeds the next.
  • StrOutputParser() is usually the last step — it converts the AIMessage object to a plain string. Without it you get an object, not text.
  • Use llm.with_structured_output(PydanticModel) when you need a structured response. It uses OpenAI function calling under the hood — far more reliable than asking the LLM to format JSON.
  • Keep prompts in PromptTemplate objects, not f-strings. This makes them testable, reusable, and separates prompt engineering from application logic.

asyncio Basics

Async lets you handle thousands of I/O operations concurrently on a single thread — no GIL concerns, no thread overhead. One event loop, many suspended coroutines, blazing throughput.

Why Async?

When your program makes a network request or reads a file, it spends most of the time waiting — the CPU sits idle. Threads solve this by switching threads while one waits. Async solves it better: one thread runs an event loop that juggles many suspended operations without thread-switching overhead.

Thread model: 1 thread per request → 1000 requests = 1000 threads (expensive)
Async model:   1 event loop → 1000 requests = 1000 suspended coroutines (cheap)

Coroutine A sends HTTP request → suspends (await) → event loop runs B and C
Coroutine B reads file → suspends (await) → event loop runs A's response handler
Coroutine C queries DB → suspends (await) → all make progress concurrently

async def and await

Coroutines — functions that can pause

A function with async def is a coroutine function. Calling it returns a coroutine object — it doesn't execute the body yet, just like calling a generator function returns a generator. You need to await it or run it with asyncio.run().

Java — CompletableFuture 
CompletableFuture<String> fetchAsync(String url) {
    return CompletableFuture.supplyAsync(() -> {
        // blocking call runs in ForkJoinPool thread
        return http.get(url);
    });
}

CompletableFuture<String> result = fetchAsync(url);
String data = result.get(); // blocks
Python 
import asyncio

# async def makes this a coroutine function
async def fetch(url: str) -> str:
    # await suspends THIS coroutine, letting others run
    await asyncio.sleep(1)   # simulate network delay
    return f"data from {url}"

# Calling fetch() does NOT run it — returns a coroutine object
coro = fetch("https://example.com")
print(type(coro))   # <class 'coroutine'>

# asyncio.run() creates an event loop and runs the coroutine
result = asyncio.run(fetch("https://example.com"))
print(result)   # "data from https://example.com"

# You can only await inside an async function
async def main():
    result = await fetch("https://example.com")
    print(result)

asyncio.run(main())
Async is infectious. Once you await something, your function must be async def. And the caller of your async def must also await it, so it must also be async def. This bubbles all the way up to asyncio.run() at the top level.

asyncio.sleep() vs time.sleep()

This is the most important gotcha. time.sleep() blocks the entire thread — the event loop freezes and nothing else can run. asyncio.sleep() suspends the current coroutine and yields control back to the event loop.

Python 
import asyncio
import time

# WRONG: time.sleep blocks the entire event loop
async def bad_worker(name: str, delay: float):
    print(f"{name} starting")
    time.sleep(delay)     # blocks — no other coroutine runs!
    print(f"{name} done")

# CORRECT: asyncio.sleep suspends just this coroutine
async def good_worker(name: str, delay: float):
    print(f"{name} starting")
    await asyncio.sleep(delay)   # suspends, event loop runs others
    print(f"{name} done")

async def main():
    start = time.perf_counter()
    # These appear to run sequentially with time.sleep:
    # await asyncio.gather(bad_worker("A", 1), bad_worker("B", 1))

    # These run concurrently with asyncio.sleep:
    await asyncio.gather(
        good_worker("A", 1.0),
        good_worker("B", 1.0),
        good_worker("C", 1.0),
    )
    print(f"Total: {time.perf_counter() - start:.2f}s")  # ~1.0s not 3.0s

asyncio.run(main())

asyncio.gather() and create_task()

gather — run coroutines concurrently

asyncio.gather() schedules multiple coroutines to run concurrently and waits for all of them. It returns a list of results in the same order as the input coroutines.

Java 
CompletableFuture<String> f1 = fetchAsync(url1);
CompletableFuture<String> f2 = fetchAsync(url2);
CompletableFuture<String> f3 = fetchAsync(url3);

CompletableFuture.allOf(f1, f2, f3).join();
String r1 = f1.get(), r2 = f2.get(), r3 = f3.get();
Python 
import asyncio
import httpx

async def fetch(client: httpx.AsyncClient, url: str) -> dict:
    resp = await client.get(url)
    resp.raise_for_status()
    return resp.json()

async def main():
    urls = [
        "https://jsonplaceholder.typicode.com/posts/1",
        "https://jsonplaceholder.typicode.com/posts/2",
        "https://jsonplaceholder.typicode.com/posts/3",
    ]

    async with httpx.AsyncClient() as client:
        # gather runs all three concurrently
        results = await asyncio.gather(
            *[fetch(client, url) for url in urls]
        )

    for r in results:
        print(r["title"])

asyncio.run(main())

Error handling in gather

Python 
import asyncio

async def might_fail(n: int) -> int:
    await asyncio.sleep(0.1)
    if n == 2:
        raise ValueError(f"Task {n} failed!")
    return n * 10

async def main():
    # Default: first exception cancels remaining and re-raises
    try:
        results = await asyncio.gather(
            might_fail(1),
            might_fail(2),   # raises
            might_fail(3),
        )
    except ValueError as e:
        print(f"One task failed: {e}")

    # return_exceptions=True — collect all results, exceptions included
    results = await asyncio.gather(
        might_fail(1),
        might_fail(2),
        might_fail(3),
        return_exceptions=True,
    )
    for i, r in enumerate(results):
        if isinstance(r, Exception):
            print(f"Task {i+1} failed: {r}")
        else:
            print(f"Task {i+1} succeeded: {r}")

asyncio.run(main())

create_task() — fire-and-forget

create_task() schedules a coroutine immediately without waiting for it. Use it when you want to start background work and continue doing other things — the event loop will run the task alongside your current coroutine.

Python 
import asyncio

async def background_job(name: str) -> None:
    print(f"[bg] {name} started")
    await asyncio.sleep(2)
    print(f"[bg] {name} finished")

async def main():
    # Start background task — doesn't block
    task = asyncio.create_task(background_job("cleanup"))

    # Continue doing other work
    print("Doing main work...")
    await asyncio.sleep(1)
    print("Main work done.")

    # Wait for the background task to finish
    await task   # or just let the event loop clean up

asyncio.run(main())
# Output:
# [bg] cleanup started
# Doing main work...
# Main work done.
# [bg] cleanup finished

Practical: Async HTTP with httpx

Sequential vs concurrent fetching

Python 
import asyncio
import httpx
import time

URLS = [f"https://jsonplaceholder.typicode.com/posts/{i}" for i in range(1, 11)]

async def fetch_one(client: httpx.AsyncClient, url: str) -> dict:
    resp = await client.get(url, timeout=10.0)
    resp.raise_for_status()
    return {"url": url, "title": resp.json()["title"]}

async def fetch_sequential() -> list[dict]:
    async with httpx.AsyncClient() as client:
        results = []
        for url in URLS:
            results.append(await fetch_one(client, url))
        return results

async def fetch_concurrent() -> list[dict]:
    async with httpx.AsyncClient() as client:
        return await asyncio.gather(
            *[fetch_one(client, url) for url in URLS]
        )

async def main():
    start = time.perf_counter()
    seq = await fetch_sequential()
    print(f"Sequential: {time.perf_counter() - start:.2f}s ({len(seq)} posts)")

    start = time.perf_counter()
    con = await fetch_concurrent()
    print(f"Concurrent: {time.perf_counter() - start:.2f}s ({len(con)} posts)")
    # Concurrent is ~10x faster — all 10 requests in-flight at once

asyncio.run(main())
Install httpx: pip install httpx. Use httpx.AsyncClient — not requests — for async code. requests is synchronous only and will block the event loop.

Quick Reference Card

Define coroutine
async def fn(): await something()

Any function with async def. Must be awaited or run with asyncio.run().

Entry point
asyncio.run(main())

Creates an event loop, runs the coroutine, closes the loop. Call once at top level.

Concurrent execution
results = await asyncio.gather(coro1(), coro2(), coro3())

Runs all concurrently, returns list of results in order.

Background task
task = asyncio.create_task(coro())

Schedules immediately, doesn't block. Await later to get result/exceptions.

Non-blocking sleep
await asyncio.sleep(1.0)

Suspends current coroutine. Never use time.sleep() in async code.

Suppress exceptions in gather
await asyncio.gather(*coros, return_exceptions=True)

Exceptions are returned as values instead of raised. Check with isinstance(r, Exception).

Async HTTP
async with httpx.AsyncClient() as c: await c.get(url)

Use httpx.AsyncClient, not requests, inside async code.

Async context manager
async with resource() as r: ...

Like with but calls __aenter__/__aexit__ — used by httpx.AsyncClient, DB connections, etc.

Wrap-up

  • async def defines a coroutine function. Calling it returns a coroutine object — the body doesn't run until you await it or schedule it with asyncio.run() / create_task().
  • Never use time.sleep() inside async code — it blocks the entire event loop, defeating the purpose. Always use await asyncio.sleep(). The same rule applies to any blocking call: use async-aware alternatives.
  • asyncio.gather() is the primary tool for running multiple coroutines concurrently. Use return_exceptions=True when you need all results even if some fail — otherwise the first exception short-circuits everything.
  • Async is best for I/O-bound work with many concurrent operations. For CPU-bound work, async doesn't help — still need multiprocessing. For moderate concurrency, ThreadPoolExecutor is often simpler.
  • Async is "infectious": once one function is async def, its callers must also be async def. Design your async boundary carefully — convert at the edges (CLI entry point, route handler) and keep the async boundary as wide as needed.

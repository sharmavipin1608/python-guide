1. try / except / else / finally

Python's error handling is similar to Java's try/catch but with a twist: the else clause runs only when no exception occurred. This lets you separate "happy path" code from error handling cleanly.

Basic exception handling

Java 
try {
    int result = Integer.parseInt(input);
    System.out.println("Parsed: " + result);
} catch (NumberFormatException e) {
    System.out.println("Invalid number: " + e.getMessage());
} finally {
    System.out.println("Always runs");
}
Python 
try:
    result = int(input_str)
    print(f"Parsed: {result}")
except ValueError as e:
    print(f"Invalid number: {e}")
finally:
    print("Always runs")

Catching specific exception types

Python 
# Catch multiple specific types
try:
    value = my_dict[key]
    result = 10 / value
except KeyError:
    print(f"Key '{key}' not found")
except ZeroDivisionError:
    print("Cannot divide by zero")
except (TypeError, ValueError) as e:
    # Catch multiple types in one block
    print(f"Bad value: {e}")

# Access the exception object with 'as'
try:
    open("nonexistent.txt")
except FileNotFoundError as e:
    print(f"File error: {e}")
    print(f"Filename: {e.filename}")   # exception attributes
Never use bare except: — it catches everything including KeyboardInterrupt and SystemExit, making it impossible to Ctrl+C your program. At minimum, catch Exception which excludes those. Better: catch the specific exceptions you expect.

The else clause — Python's unique advantage

Python 
# else runs ONLY if no exception was raised in try
try:
    f = open("config.json")
except FileNotFoundError:
    print("Config file missing, using defaults")
    config = {}
else:
    # Only runs if open() succeeded
    # Errors HERE are NOT caught by the except above
    config = json.load(f)
    f.close()
    print("Config loaded successfully")
finally:
    # Always runs — cleanup
    print("Config loading complete")
Why else matters: Without else, you'd put the success code inside try. But then any exception from that code gets caught too, masking bugs. else keeps the try block narrow — only the code that might throw the expected exception goes in try.

The full pattern

Python — complete error handling 
def read_config(path):
    """Read and parse a JSON config file."""
    try:
        f = open(path)                       # might raise FileNotFoundError
    except FileNotFoundError:
        print(f"Config not found: {path}")
        return {}                            # return default
    except PermissionError:
        print(f"No permission to read: {path}")
        return {}
    else:
        # File opened successfully — parse it
        import json
        try:
            return json.load(f)              # might raise json.JSONDecodeError
        except json.JSONDecodeError as e:
            print(f"Invalid JSON: {e}")
            return {}
    finally:
        # Cleanup — this runs no matter what
        print("Config read attempt complete")

2. Raising Exceptions

Java 
public void setAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException(
            "Age cannot be negative: " + age);
    }
    this.age = age;
}

// Re-throwing
try {
    riskyOperation();
} catch (IOException e) {
    logger.error("Failed", e);
    throw e;   // re-throw
}
Python 
def set_age(age):
    if age < 0:
        raise ValueError(f"Age cannot be negative: {age}")
    return age

# Re-raising the current exception
try:
    risky_operation()
except OSError as e:
    logger.error(f"Failed: {e}")
    raise          # re-raise without arguments

# Raise a different exception, keeping the original as cause
try:
    config = json.loads(raw_data)
except json.JSONDecodeError as e:
    raise ValueError("Invalid config format") from e
    # 'from e' chains the exceptions (like Java's cause)

Common built-in exceptions

Python — exceptions you'll see most often 
# ValueError — wrong value for the type
int("abc")                  # ValueError

# TypeError — wrong type for an operation

# KeyError — dict key not found
{}["missing"]               # KeyError

# IndexError — list index out of range
[1, 2, 3][10]               # IndexError

# AttributeError — object doesn't have that attribute
"hello".nonexistent()       # AttributeError

# FileNotFoundError — file doesn't exist

# ZeroDivisionError

# ImportError / ModuleNotFoundError

3. Custom Exception Classes

Just like in Java, you create custom exceptions by subclassing Exception. In Python, it's much less boilerplate.

Java 
public class InsufficientFundsException
        extends Exception {
    private final double balance;
    private final double amount;

    public InsufficientFundsException(
            double balance, double amount) {
        super(String.format(
            "Cannot withdraw %.2f, balance: %.2f",
            amount, balance));
        this.balance = balance;
        this.amount = amount;
    }
}
Python 
class InsufficientFundsError(Exception):
    """Raised when a withdrawal exceeds the balance."""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(
            f"Cannot withdraw {amount:.2f}, "
            f"balance: {balance:.2f}"
        )
Python — using custom exceptions 
# Define a hierarchy for your application
class AppError(Exception):
    """Base exception for our application."""
    pass

class ValidationError(AppError):
    """Raised when input validation fails."""
    pass

class NotFoundError(AppError):
    """Raised when a resource is not found."""
    pass

# Usage
def get_user(user_id):
    if not isinstance(user_id, int):
        raise ValidationError(f"user_id must be int, got {type(user_id)}")
    user = db.find(user_id)
    if user is None:
        raise NotFoundError(f"User {user_id} not found")
    return user

# Catching the hierarchy
try:
    user = get_user(input_id)
except NotFoundError as e:
    print(f"404: {e}")
except AppError as e:
    # Catches ValidationError and any other AppError subclass
    print(f"App error: {e}")
Convention: Python exceptions end in Error (not Exception), e.g., ValueError, TypeError. Follow this naming when creating your own.

4. Importing Modules

Python's import system is simpler than Java's. A module is just a .py file. A package is a directory with an __init__.py file. No classpath to wrestle with.

Import styles

Java 
// Import specific class
import java.util.ArrayList;

// Import everything (discouraged)
import java.util.*;

// Static import
import static java.lang.Math.PI;

// Usage
ArrayList<String> list = new ArrayList<>();
double area = PI * r * r;
Python 
# Import the whole module
import json
data = json.loads('{"a": 1}')

# Import specific things
from pathlib import Path
p = Path("data.txt")

# Import with alias
import numpy as np
arr = np.array([1, 2, 3])

# Import multiple things
from os.path import join, exists

# Import everything (avoid — pollutes namespace)
from math import *   # don't do this in real code
Best practice: Prefer import module or from module import specific_thing. Avoid from module import * — it makes it hard to tell where names come from and can cause name collisions.

Standard library modules you'll use often

5. Creating Your Own Module

Any .py file is a module. You can import functions, classes, and variables from it. No boilerplate — just write the code and import it.

Python — mathutils.py 
# mathutils.py — this is a module

PI = 3.14159

def circle_area(radius):
    """Calculate the area of a circle."""
    return PI * radius ** 2

def circle_circumference(radius):
    """Calculate the circumference of a circle."""
    return 2 * PI * radius
Python — main.py (using the module) 
# main.py — import from your module
import mathutils

area = mathutils.circle_area(5)
print(f"Area: {area:.2f}")

# Or import specific things
from mathutils import circle_area, PI
print(f"PI is {PI}")
print(f"Area: {circle_area(10):.2f}")

The __name__ == '__main__' pattern

In Java, the entry point is public static void main(String[] args). Python doesn't require a special method — it runs files top to bottom. But you need a way to distinguish "running this file directly" from "importing this file as a module."

Python — mathutils.py with entry point guard 
# mathutils.py

PI = 3.14159

def circle_area(radius):
    return PI * radius ** 2

def circle_circumference(radius):
    return 2 * PI * radius

# This block only runs when executing this file directly
# NOT when importing it as a module
if __name__ == "__main__":
    # Test code, demos, or CLI entry point
    print(f"Area of r=5: {circle_area(5):.2f}")
    print(f"Circumference of r=5: {circle_circumference(5):.2f}")
How it works: When Python runs a file directly, it sets __name__ to "__main__". When the file is imported, __name__ is set to the module name (e.g., "mathutils"). This check prevents test/demo code from running on import.

6. Packages and Project Structure

A package is a directory containing an __init__.py file (which can be empty). It lets you organize modules into a hierarchy, like Java packages.

File structure 
myproject/
├── main.py
└── utils/
    ├── __init__.py      ← makes utils/ a package
    ├── math_helpers.py
    └── string_helpers.py
Python — importing from packages 
# From main.py
from utils.math_helpers import circle_area
from utils.string_helpers import clean_input

# Or import the submodule
import utils.math_helpers as math_utils
math_utils.circle_area(5)

# __init__.py can re-export for convenience
# utils/__init__.py:
from .math_helpers import circle_area
from .string_helpers import clean_input

# Then from main.py you can do:
from utils import circle_area, clean_input

Wrap-up

  • try/except is Python's try/catch. The else block (runs only if no exception) is a Python-unique feature that keeps your try blocks narrow and intention-revealing.
  • Always catch specific exceptions. Never use bare except: — it swallows KeyboardInterrupt and SystemExit.
  • Use raise to throw exceptions, raise ... from e to chain them. Custom exceptions should subclass Exception and end in Error.
  • Any .py file is a module. Use import or from x import y. Avoid from x import * in real code.
  • The if __name__ == "__main__": guard is Python's entry point pattern. It prevents demo/test code from running when the file is imported as a module.
